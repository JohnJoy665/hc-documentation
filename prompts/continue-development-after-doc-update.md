# Continue Development After Documentation Update

Прочитай `AGENTS.md`.

Работай в режиме `development mode`.

Ранее задача была остановлена, потому что в документации не хватало данных.

После этого пользователь добавил новые материалы в `INBOX/`, и `documentation mode` обновил:

- `docs/`
- `rag/`
- `index/`

## Что нужно сделать

1. Вернуться к исходной разработческой задаче.
2. Повторно изучить:

   - `index/map.md`
   - `index/glossary.md`
   - `index/open_questions.md`
   - `docs/`
   - `rag/`

3. Проверить, закрыты ли ранее найденные пробелы.
4. Если знаний теперь хватает — предложить план и код.
5. Если знаний все еще не хватает — снова остановиться и точно указать, каких данных не хватает.

## Важное ограничение

Серверный код WebTutor не является обычным JavaScript.

Не используй JavaScript/Node.js API без подтверждения в локальной документации.

```markdown
- ...
```

```markdown
- ...

## Следующий шаг

Добавьте недостающие материалы в `INBOX/` и снова запустите `prompts/process-inbox.md`.
```
