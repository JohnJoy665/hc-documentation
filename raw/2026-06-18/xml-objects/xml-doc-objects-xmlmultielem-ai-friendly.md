# XmlMultiElem

## 1. Назначение

`XmlMultiElem` — виртуальный объект для работы с множественными XML-элементами, то есть с элементами, которые в XMD-форме описаны с атрибутом `MULTIPLE="1"`.

Объект появляется при обращении к полю множественного элемента через точечную нотацию.

```js
candidate.profession_id
```

Если `profession_id` описан в форме как множественный элемент, выражение `candidate.profession_id` возвращает не один `XmlElem`, а объект `XmlMultiElem`.

`XmlMultiElem` ведет себя как массив фактических элементов `XmlElem` с одинаковым именем среди дочерних элементов родителя.

Пример:

```xml
<candidate>
  <profession_id>2054</profession_id>
  <profession_id>2402</profession_id>
  <profession_id>2110</profession_id>
</candidate>
```

В таком случае:

```js
candidate.profession_id
```

соответствует набору всех элементов `profession_id` внутри `candidate`.

## 2. Связанные объекты

| Объект | Роль |
|---|---|
| `XmlDoc` | XML-документ целиком. |
| `XmlElem` | Один XML-элемент документа. |
| `XmlMultiElem` | Виртуальный объект-массив для группы одноименных множественных элементов. |

## 3. Модель поведения

`XmlMultiElem` не является физическим XML-узлом. Это виртуальная обертка над набором дочерних элементов родительского элемента.

Например, если есть поле:

```js
candidate.profession_id
```

то `candidate` — родительский `XmlElem`, а `profession_id` — имя множественного дочернего элемента.

Операции `Add()`, `Clear()`, `DeleteByValue()` и другие изменяют фактические дочерние элементы родителя.

## 4. Главные правила для AI/Codex

1. Не считать `XmlMultiElem` одиночным значением. Это массивоподобный объект.
2. Для добавления значения использовать `Add().Value = ...` или `ObtainByValue(...)`.
3. Для полной замены списка значений использовать `SetValues(values)`.
4. Для очистки всех значений использовать `Clear()`, а не ручной цикл, если нет особой логики.
5. `DeleteByValue(value)` выбрасывает ошибку, если элемента с таким значением нет. Перед ним безопаснее проверять `ByValueExists(value)`.
6. `DeleteAll()` — устаревшее имя `Clear()`. В новом коде предпочтительно использовать `Clear()`.
7. Для универсального кода, который должен работать и с одиночным `XmlElem`, и с множественным `XmlMultiElem`, использовать свойство `Instances`.

## 5. Быстрые сценарии

### 5.1. Добавить значение

```js
candidate.profession_id.Add().Value = 2054;
```

### 5.2. Добавить значение, только если его еще нет

```js
candidate.profession_id.ObtainByValue( 2054 );
```

или явно:

```js
if ( ! candidate.profession_id.ByValueExists( 2054 ) )
{
    candidate.profession_id.Add().Value = 2054;
}
```

### 5.3. Удалить все значения

```js
candidate.profession_id.Clear();
```

### 5.4. Безопасно удалить значение

```js
if ( candidate.profession_id.ByValueExists( 2054 ) )
{
    candidate.profession_id.DeleteByValue( 2054 );
}
```

### 5.5. Полностью заменить список значений

```js
candidate.profession_id.SetValues( [2054, 2402, 2110] );
```

### 5.6. Получить XQuery-константу из списка значений

```js
candidate.profession_id.XQueryLiteral
```

Может вернуть строку:

```xquery
(2054,2402,2110)
```

## 6. Свойства объекта

| Свойство | Тип / результат | Кратко |
|---|---:|---|
| `Doc` | `XmlDoc` | Документ, к которому относится объект. |
| `ForeignArray` | `Object` | Массив, указанный в `FOREIGN-ARRAY` текущего элемента. |
| `FormElem` | `XmlElem` формы | Элемент формы текущего элемента. |
| `HasValue` | `Bool` | Есть ли хотя бы один фактический множественный элемент. |
| `Instances` | `Array` | Массив всех фактических элементов `XmlElem`. |
| `IsMultiElem` | `Bool` | Признак множественного элемента. См. предупреждение ниже. |
| `Name` | `String` | Имя соответствующего множественного элемента. |
| `Parent` | `XmlElem` | Родительский элемент. |
| `XQueryLiteral` | `String` | XQuery-константа в виде последовательности значений. |

## 7. Методы объекта

| Метод | Назначение |
|---|---|
| `Add()` | Добавляет новый множественный элемент и возвращает его как `XmlElem`. |
| `ByValueExists(value)` | Проверяет, есть ли элемент с заданным значением. |
| `Clear()` | Удаляет все соответствующие множественные элементы. |
| `DeleteAll()` | Устаревший синоним `Clear()`. |
| `DeleteByValue(value)` | Удаляет первый элемент с заданным значением. Если не найден — ошибка. |
| `GetByKey(keyValue[, keyName])` | Возвращает элемент с заданным ключом. Если не найден — ошибка. |
| `ObtainByKey(keyValue[, keyName])` | Находит элемент по ключу или создает новый. |
| `ObtainByKeySorted(keyValue[, keyName])` | Находит элемент по ключу или вставляет новый в позицию по возрастанию ключа. |
| `ObtainByValue(value)` | Находит элемент по значению или создает новый. |
| `SetValues(values)` | Устанавливает новый список значений, затирая старый. |

# 8. Методы подробно

## 8.1. `XmlMultiElem.SetValues()`

Устанавливает список значений множественного элемента.

Все предыдущие значения затираются.

### Синтаксис

```js
SetValues( values )
```

### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `values` | абстрактный массив | Новый список значений. |

### Результат

Нет данных в исходном описании.

### Пример

```js
candidate.profession_id.SetValues( [2054, 2402, 2110] );
```

### AI notes

Использовать, когда нужно именно заменить весь набор значений, а не добавить одно значение.

---

## 8.2. `XmlMultiElem.Add()`

Добавляет новый соответствующий множественный элемент к родительскому элементу и возвращает ссылку на него.

### Синтаксис

```js
Add()
```

### Результат

`XmlElem` — новый добавленный элемент.

### Пример

```js
candidate.profession_id.Add().Value = 2054;
```

### AI notes

После `Add()` часто нужно явно установить `.Value`, потому что `Add()` возвращает объект `XmlElem`, а не само значение.

---

## 8.3. `XmlMultiElem.Clear()`

Удаляет все соответствующие множественные элементы.

### Синтаксис

```js
Clear()
```

### Результат

Нет.

### Пример

```js
candidate.profession_id.Clear();
```

### AI notes

Предпочтительный метод для удаления всех значений множественного поля.

---

## 8.4. `XmlMultiElem.DeleteAll()`

Удаляет все соответствующие множественные элементы.

Устаревшее название метода `Clear()`.

### Синтаксис

```js
DeleteAll()
```

### Результат

Нет.

### Рекомендуемая замена

```js
candidate.profession_id.Clear();
```

### AI notes

В новом коде не использовать `DeleteAll()`, если нет необходимости поддерживать старый стиль. Использовать `Clear()`.

---

## 8.5. `XmlMultiElem.DeleteByValue()`

Удаляет соответствующий множественный элемент с заданным значением.

Если таких элементов несколько, удаляет первый.

Если элемент с таким значением отсутствует, возвращает ошибку.

### Синтаксис

```js
DeleteByValue( val )
```

### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `val` | Any | Значение элемента. |

### Результат

Нет.

### Пример

```js
candidate.profession_id.DeleteByValue( 2054 );
```

### Безопасный шаблон

```js
if ( candidate.profession_id.ByValueExists( 2054 ) )
{
    candidate.profession_id.DeleteByValue( 2054 );
}
```

### AI notes

Так как метод возвращает ошибку при отсутствии значения, Codex должен предпочитать проверку через `ByValueExists()` перед удалением, если отсутствие значения допустимо.

---

## 8.6. `XmlMultiElem.ByValueExists()`

Проверяет, существует ли соответствующий множественный элемент с заданным значением.

### Синтаксис

```js
ByValueExists( value )
```

### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `value` | Any | Заданное значение. |

### Результат

`Bool`.

### Пример

```js
if ( candidate.profession_id.ByValueExists( 2054 ) )
{
    // значение есть
}
```

### AI notes

Использовать для безопасной проверки перед `DeleteByValue()` или перед ручным `Add()`.

---

## 8.7. `XmlMultiElem.GetByKey()`

Находит соответствующий множественный элемент с заданным значением ключевого поля.

Если такой элемент не найден, возвращает ошибку.

### Синтаксис

```js
GetByKey( keyValue )
GetByKey( keyValue, keyName )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `keyValue` | Any | Да | Значение ключа. |
| `keyName` | String | Нет | Имя элемента, являющегося ключом. Если не указано, используется первичный ключ. |

### Результат

`XmlElem`.

### AI notes

Использовать только если отсутствие элемента является ошибочной ситуацией. Если элемент может отсутствовать и его нужно создать — использовать `ObtainByKey()`.

---

## 8.8. `XmlMultiElem.ObtainByKey()`

Ищет соответствующий множественный элемент с заданным ключевым элементом.

Если не находит, добавляет новый дочерний элемент и присваивает его ключевому полю заданное значение.

Возвращает ранее существовавший или вновь созданный дочерний элемент.

### Синтаксис

```js
ObtainByKey( keyValue )
ObtainByKey( keyValue, keyName )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `keyValue` | Any | Да | Значение ключа. |
| `keyName` | String | Нет | Имя элемента, являющегося ключом. Если не указано, используется первичный ключ. |

### Результат

`XmlElem`.

### AI notes

Удобный метод для idempotent-логики: “получить или создать”.

---

## 8.9. `XmlMultiElem.ObtainByValue()`

Ищет соответствующий множественный элемент с заданным значением.

Если не находит, добавляет новый элемент и присваивает ему заданное значение.

### Синтаксис

```js
ObtainByValue( value )
```

### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `value` | Any | Значение элемента. |

### Результат

`XmlElem`.

### Пример

```js
candidate.profession_id.ObtainByValue( 2054 );
```

### AI notes

Предпочтительный способ добавить значение без дублирования, если элемент с таким значением уже может существовать.

---

## 8.10. `XmlMultiElem.ObtainByKeySorted()`

Находит элемент, относящийся к данному объекту, по ключу.

Если элемент отсутствует, добавляет его в соответствующую позицию по возрастанию ключа.

### Синтаксис

```js
ObtainByKeySorted( keyValue )
ObtainByKeySorted( keyValue, keyName )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `keyValue` | Any | Да | Значение ключа. |
| `keyName` | String | Нет | Имя элемента, являющегося ключом. Если не указано, используется первичный ключ. |

### Результат

`XmlElem`.

### AI notes

Использовать, когда порядок элементов должен сохраняться по возрастанию ключа.

# 9. Свойства подробно

## 9.1. `XmlMultiElem.Doc`

Возвращает документ, к которому относится данный объект.

### Результат

`XmlDoc`.

---

## 9.2. `XmlMultiElem.ForeignArray`

Возвращает массив, указанный в атрибуте `FOREIGN-ARRAY` текущего элемента.

### Результат

`Object`.

### AI notes

Использовать, если множественное поле ссылается на внешний справочник/каталог через форму.

---

## 9.3. `XmlMultiElem.FormElem`

Возвращает ссылку на элемент формы текущего элемента.

Ссылка возвращается независимо от того, был документ открыт по форме или нет:

- если элемент был создан по форме, возвращается ссылка на элемент формы, по которой он был открыт;
- если нет, возвращается динамический элемент формы, созданный специально для данного элемента.

---

## 9.4. `XmlMultiElem.HasValue`

Возвращает `true`, если существует хотя бы один соответствующий множественный элемент.

### Результат

`Bool`.

### Пример из исходного описания

```js
if ( candidate.profession_id.HasValue() )
{
}
```

### AI notes

В списке свойств `HasValue` указан как свойство, но в примере исходного описания вызывается как метод: `HasValue()`.

Для Codex важно не делать уверенного вывода о синтаксисе без проверки на конкретной версии платформы. В предыдущих объектах `HasValue` описан как атрибут. Предпочтительная форма по общей модели свойств:

```js
if ( candidate.profession_id.HasValue )
{
}
```

Если в существующем проекте используется `HasValue()`, сохранять стиль проекта.

---

## 9.5. `XmlMultiElem.Instances`

Возвращает массив всех соответствующих множественных элементов.

### Результат

`Array`.

Сам по себе этот метод/атрибут не несет особого смысла, поскольку объект `XmlMultiElem` сам является массивом. Но вместе с одноименным свойством объекта `XmlElem` позволяет писать универсальный код для одиночных и множественных элементов.

### Пример

```js
array = candidate.profession_id.Instances;
```

Возвращает массив всех элементов `profession_id`.

```js
array = candidate.location_id.Instances;
```

Для одиночного `XmlElem` возвращает массив из одного элемента `location_id`.

### Универсальный шаблон

```js
for ( elem in someElemOrMultiElem.Instances )
{
    // обработка elem как XmlElem
}
```

## 9.6. `XmlMultiElem.IsMultiElem`

Исходное описание содержит противоречивый фрагмент:

> `XmlMultiElem.IsMultiElem` всегда возвращает false. Аналогичный метод объекта `XmlElem` всегда возвращает false. Это позволяет по конструкции вида `candidate.profession_id.IsMultiElem` определять, был ли элемент `profession_id` описан c атрибутом `MULTIPLE="1"`.

### Важное предупреждение

В этом месте исходный текст, вероятно, содержит ошибку или опечатку.

По смыслу и по аналогии с описанием `XmlElem.IsMultiElem`:

- `XmlElem.IsMultiElem` всегда возвращает `false`;
- `XmlMultiElem.IsMultiElem` должен позволять определить, что элемент множественный, то есть логически ожидается `true`.

Однако в исходном фрагменте явно написано `false`.

### AI rule

Codex не должен уверенно утверждать поведение `XmlMultiElem.IsMultiElem` без проверки на конкретной версии WebTutor/WebSoft HCM.

Для безопасной документации фиксируем:

- источник содержит противоречие;
- конструкция `candidate.profession_id.IsMultiElem` предназначена для определения, описан ли элемент с `MULTIPLE="1"`;
- фактическое возвращаемое значение нужно проверить на целевой версии платформы.

# 10. Практические паттерны

## 10.1. Синхронизация множественного поля со списком значений

```js
candidate.profession_id.SetValues( professionIDs );
```

Использовать, если старые значения должны быть полностью заменены.

## 10.2. Добавление без дублей

```js
for ( professionID in professionIDs )
{
    candidate.profession_id.ObtainByValue( professionID );
}
```

## 10.3. Удаление значения без ошибки при отсутствии

```js
if ( candidate.profession_id.ByValueExists( professionID ) )
{
    candidate.profession_id.DeleteByValue( professionID );
}
```

## 10.4. Очистка перед заполнением

```js
candidate.profession_id.Clear();

for ( professionID in professionIDs )
{
    candidate.profession_id.Add().Value = professionID;
}
```

Если порядок значений важен и список уже подготовлен, чаще проще использовать `SetValues()`.

## 10.5. Универсная обработка одиночных и множественных элементов

```js
for ( elem in field.Instances )
{
    // elem всегда обрабатывается как XmlElem
}
```

Этот подход работает, если `field` может быть как обычным `XmlElem`, так и `XmlMultiElem`.

# 11. Отличия похожих методов

## `Clear()` vs `DeleteAll()`

| Метод | Статус | Что делает |
|---|---|---|
| `Clear()` | Актуальный | Удаляет все значения. |
| `DeleteAll()` | Устаревший | То же самое, синоним `Clear()`. |

## `Add()` vs `ObtainByValue()`

| Метод | Поведение |
|---|---|
| `Add()` | Всегда добавляет новый элемент. Может создать дубль. |
| `ObtainByValue(value)` | Возвращает существующий элемент с таким значением или создает новый. Удобен для защиты от дублей. |

## `GetByKey()` vs `ObtainByKey()`

| Метод | Если элемент найден | Если элемент не найден |
|---|---|---|
| `GetByKey()` | Возвращает `XmlElem`. | Ошибка. |
| `ObtainByKey()` | Возвращает `XmlElem`. | Создает новый элемент и возвращает его. |
| `ObtainByKeySorted()` | Возвращает `XmlElem`. | Создает новый элемент в позиции по возрастанию ключа. |

## `DeleteByValue()` vs `Clear()`

| Метод | Что удаляет |
|---|---|
| `DeleteByValue(value)` | Первый элемент с заданным значением. |
| `Clear()` | Все элементы данного множественного поля. |

# 12. Ошибки и безопасные альтернативы

| Операция | Риск | Безопасная альтернатива |
|---|---|---|
| `DeleteByValue(value)` | Ошибка, если значения нет. | Проверить `ByValueExists(value)`. |
| `GetByKey(key)` | Ошибка, если элемента нет. | Использовать `ObtainByKey(key)`, если элемент можно создать. |
| `Add().Value = value` | Может создать дубль. | Использовать `ObtainByValue(value)`. |
| `DeleteAll()` | Устаревшее имя. | Использовать `Clear()`. |
| Использование `IsMultiElem` без проверки | В исходнике противоречие. | Проверить поведение на версии платформы или использовать структуру формы. |

# 13. Минимальная шпаргалка для Codex

```js
// Добавить значение, допускаются дубли
candidate.profession_id.Add().Value = 2054;

// Добавить значение без дубля
candidate.profession_id.ObtainByValue( 2054 );

// Проверить наличие значения
if ( candidate.profession_id.ByValueExists( 2054 ) )
{
}

// Удалить одно значение безопасно
if ( candidate.profession_id.ByValueExists( 2054 ) )
{
    candidate.profession_id.DeleteByValue( 2054 );
}

// Удалить все значения
candidate.profession_id.Clear();

// Полностью заменить список
candidate.profession_id.SetValues( [2054, 2402, 2110] );
```

# 14. Непроверенные / спорные места исходного описания

Этот раздел важен для последующей нормализации документации.

## 14.1. Опечатка `MILTIPLE`

В исходном тексте указано `MILTIPLE`, но в документации и XMD-формах ожидаемый атрибут — `MULTIPLE`.

## 14.2. `XmlMultiElem.IsMultiElem`

Исходник говорит, что свойство всегда возвращает `false`, но одновременно утверждает, что оно нужно для определения `MULTIPLE="1"`. Это противоречие.

До проверки на целевой версии платформы не использовать это свойство как единственный надежный источник истины.

## 14.3. `XmlMultiElem.Parent`

В исходнике результат указан как `Объект XmlDoc`, но по смыслу должен быть родительский `XmlElem`.

## 14.4. `HasValue` как свойство или метод

В списке оно указано как свойство `HasValue`, но пример показывает вызов `HasValue()`.

Для единообразия с `XmlElem.HasValue` предпочтительно рассматривать его как свойство, если проектный код не показывает обратное.
