# SP-XML: объект `XmlElem`

`XmlElem` — объект, соответствующий конкретному XML-узлу/элементу в открытом XML-документе. Элемент имеет собственное значение и набор дочерних элементов.

Этот файл предназначен для включения в AI-friendly документацию WebTutor/WebSoft HCM. Материал переработан из исходного справочного текста без добавления неподтвержденных API.

## Кратко

- `XmlElem` представляет один элемент XML-документа.
- Элемент может быть создан по форме XMD или быть динамическим, то есть существовать без формы.
- К дочерним элементам можно обращаться как к свойствам объекта: `elem.xxx`.
- Элемент также обладает свойствами массива: можно получать список дочерних элементов стандартными приемами работы с массивами, включая индексирование `elem[i]`.
- Для множественных элементов используется отдельный объект `XmlMultiElem`; у одиночного `XmlElem` свойство `IsMultiElem` всегда возвращает `false`.

## Важные правила для AI-агента

1. Не считать SP-XML полноценным JavaScript. Синтаксис похож, но поведение объектов, XML-элементов, каталогов и форм специфично для платформы.
2. Перед изменением XML-структуры различать:
   - элемент, созданный по форме;
   - динамический элемент;
   - элемент с атрибутом `MULTIPLE`.
3. Для безопасного доступа предпочитать методы с префиксом `Opt...` или `...Exists`, если отсутствие элемента допустимо.
4. Методы без `Opt` часто возвращают ошибку, если элемент/атрибут не найден.
5. Динамические атрибуты (`AddAttr`, `SetAttr`, `DeleteAttr`, `Attr`, `AttrNames`) применимы только к динамическим элементам, если в описании метода не указано иное.
6. Если нужно полностью синхронизировать элемент с XML-данными из строки или URL, перед `LoadData()` / `LoadDataFromUrl()` нужно вызвать `Clear()`, потому что эти методы подгружают только присутствующие в исходном XML данные.
7. Хранить ссылку на изменяемый `XmlElem` в `Session` опасно: платформа автоматически устанавливает `ReadOnly = true`, чтобы предотвратить небезопасные одновременные изменения из разных потоков.

## Частые паттерны

### Безопасно получить дочерний элемент

```js
child = elem.OptChild( "name" );
if ( child != undefined )
{
    value = child.Value;
}
```

### Получить дочерний элемент по ключу без ошибки

```js
item = list.GetOptChildByKey( keyValue );
if ( item != undefined )
{
    // элемент найден
}
```

### Найти или создать дочерний элемент по ключу

```js
item = list.ObtainChildByKey( keyValue );
```

### Добавить значение в простой массив

```js
entries.AddChild().Value = 10;
```

### Полностью заменить содержимое перед загрузкой XML-фрагмента

```js
elem.Clear();
elem.LoadData( xmlData );
```

### Удалить дочерний элемент по ключу, не падая при отсутствии

```js
list.DeleteOptChildByKey( keyValue );
```

### Экспортировать XML текущего элемента

```js
xml = elem.GetXml( "tabs=1,DocHeader=1" );
```

## Ошибка vs `undefined`

| Сценарий | Метод/свойство | Поведение при отсутствии |
|---|---|---|
| Получить обязательный дочерний элемент | `Child()` | в исходном списке есть, подробное описание в фрагменте отсутствует |
| Получить необязательный дочерний элемент | `OptChild()` | возвращает `undefined` |
| Получить путь | `EvalPath()` | возвращает ошибку при неверном пути |
| Получить необязательный путь | `EvalOptPath()` | возвращает `undefined` |
| Получить дочерний элемент по ключу | `GetChildByKey()` | возвращает ошибку |
| Получить дочерний элемент по ключу безопасно | `GetOptChildByKey()` | возвращает `undefined` |
| Удалить дочерний элемент по ключу | `DeleteChildByKey()` | возвращает ошибку, если не найден |
| Удалить дочерний элемент по ключу безопасно | `DeleteOptChildByKey()` | ничего не делает, если не найден |
| Удалить атрибут динамического элемента | `DeleteAttr()` | возвращает ошибку, если атрибута нет |
| Удалить атрибут безопасно | `DeleteOptAttr()` | ничего не делает, если атрибута нет |
| Получить связанный внешний элемент | `ForeignElem` | возвращает ошибку, если не найден |
| Получить связанный внешний элемент безопасно | `OptForeignElem` | возвращает `undefined` |

## Подробное описание методов и свойств

Ниже сохранены все описания, присутствовавшие в исходном фрагменте, но приведены к единому Markdown-формату.


### `AddAttr`


**Тип:** Метод


Добавляет атрибут к текущему элементу. Метод работает только для динамических элементов.

    

**Аргумент**

name   - имя атрибута (String).

value   - значение атрибута (String).

 

**Результат:**

нет


### `AddChild`


**Тип:** Метод


Добавляет дочерний элемент и возвращает указатель на него.

Если текущий элемент создан по форме, то он должен быть простым массивом. При этом аргументы для вызова функции не требуются.

Если текущий элемент является динамическим (т.е. построенным без формы), то добавляется дочерний динамический элемент с именем и типом, указанных в качестве аргументов.   

 

**Аргументы**

name - имя дочернего элемента (String). Необязательный аргумент

type - тип дочернего элемента (String). Необязательный аргумент

 

**Результат:**

Объект XmlElem


### `AddChildElem`


**Тип:** Метод


Добавляет уже созданный элемент в качестве в качестве дочернего по отношению к текущему элементу . Аргументом может быть элемент, уже созданный каким-либо образом (например, через функцию CrateElem()).

 

**Аргумент**

elem - созданный заранее элемент (Объект XmlElem)

 

**Результат:**

нет


### `AddDynamicChild`


**Тип:** Метод


Добавляет динамический дочерний элемент к текущему элементу и возвращает ссылку на внов созданный элемент. Текущий элемент при этом не обязан быть динамическим.

    

**Аргументы**

name   - имя элемента (String).

type   - тип элемента (String). Необязательный аргумент. По умолчанию создается обычная структура (составной элемент).

 

**Результат:**

Объект XmlElem


### `AssignElem`


**Тип:** Метод


Копирует в текущий элемент данные из другого элемента, включая дочерние элементы. Значения всех совпадающим по имени элементов копируются, элементы с атрибутом MULTIPLE при этом синхронизируются по количеству. Если присваиваемый и текущий элементы были созданы по разным формам - присваивются значения только по совпадающим полям.

    

**Аргументы**

srcElem   - присваиваемый элемент (Объект XmlElem).


### `AssignExtraElem`


**Тип:** Метод


Действует аналогично AssignElem(), но текущему элементу присваиваются значения только по тем полям, которые в текущем элементе не заполнены.

 

**Аргументы**

srcElem - присваиваемый элемент (object).

 

**Результат:**

нет


### `Attr`


**Тип:** Метод


Возвращает специальный псевдо-объект, позволяющий одновременно читать и записывать значение атрибута динамического элемента (например в SOURCE).

Значение псевдо-объекта по умолчанию является значением атрибута, если атрибут с заданным именем присутствует, и пустая строка, если такого атрибута нет.

При попытке установить непустое значение псевдо-объекта запишется соответствующее значение атрибута. При попытке установить пустое значенеи псевдо-объекта атрибут удалится.

**Синтаксис**
Attr( attrName )

Attr( attrName, attrType )

**Аргументы**
attrNamestring

Имя атрибута

attrTypestringoptional

**Тип атрибута** (поддерживается "string" либо "bool"). Для "bool" функция вернет true или false вместо строки.

**Пример**
<EDIT SOURCE="Ps.Attr( 'SHOW-CONDITION' )"/>
<CHECK SOURCE="Ps.Attr( 'READ-ONLY', 'bool' )"/>


### `Child`


**Тип:** Метод


Метод указан в исходном списке методов `XmlElem`, но в предоставленном фрагменте подробное описание отсутствует. Не использовать без дополнительной проверки в официальной документации или исходных XMD/WT-файлах.


### `ChildByKeyExists`


**Тип:** Метод


Проверяет, существует ли дочерний элемент с заданным значением ключевого поля.

**Смотри также** ChildByKeyExistsRec().

    

**Аргументы**

keyValue   - значение ключа (Any).

keyName   - имя элемента, являющегося ключом (String). Необязательный аргумент. Если имя ключа не указано, используется первичный ключ.

 

**Результат:**

Bool


### `ChildByKeyExistsRec`


**Тип:** Метод


Специализированный аналог также ChildByKeyExists() с поддержкой рекурсивных массивов.

Позволяет искать только по первиному ключу.

 

**Аргументы**

keyValue   - значение ключа (Any).

 

**Результат:**

Bool


### `ChildByValueExists`


**Тип:** Метод


Проверяет, существует ли дочерний элемент с заданым значением.

    

**Аргументы**

elemVal  - значение элемента (Any).

 

**Результат:**

Bool


### `ChildExists`


**Тип:** Метод


Проверяет, существует ли дочерний элемент с заданым именем.

    

**Аргументы**

name   - имя элемента (String).

 

**Результат:**

Bool


### `ChildValue`


**Тип:** Метод


Если аргумент - обычный дочерний элемент, то метод просто возвращает его значение. Если аргумент - метод или атрибут с таким наименованием, то метод вычисляет его и возвращает вычисленное значение.

SetInnerXml( xmlStr )

**Аргументы**
xmlStrstring

**Смотри также**
SetXml()


### `SetOptAttrValue`


**Тип:** Метод


Устанавливает значение атрибута с заданным именем, если значение атрибута отличается от его требуемого значения по умолчанию. Метод работает только для динамических элементов.

Если атрибут с заданным именем существует, то, если новое значение не совпадает со значением по умолчанию, устанавливает новое значение атрибута. Если новое значение совпадает со значением по умолчанию, атрибут удаляется.

Если атрибут не существует, то он добавляется, если новое значение не совпадает со значением по умолчанию.

    

**Аргументы**

attrName   - имя атрибута (String)

attValue   - значение атрибута (String)

defaultVal   - значение атрибута по умолчанию (String). Необязательный аргумент. По умолчанию - пустая строка.

 

**Результат:**

нет

**Аргументы**

order1   - направление сортировки: '+' или '-' (String).

...

 

**Результат:**

нет

### `UpdateValues`

**Результат:**

нет


### `AttrNames`

Элемент должен быть динамическим, для элемента, созданного по форме, AttrNames вернет пустой массив.


### `BaseMultipleElem`

Идет "по цепочке" элементов вверх, и возвращает первый элемент с атрибутом MULTIPLE="1". Если такого элемента нет, возвращается ошибка.

Объект XmlElem


### `ChildIndex`


**Тип:** Свойство


Возвращает индекс текущего элемента внутри родительского элемента, начиная с 0. Если родительского элемента нет - возвращается ошибка.


### `ChildNum`


**Тип:** Свойство


Возвращает количество дочерних элементов.

    

**Тип атрибута**

Integer


### `Doc`


**Тип:** Свойство


Возвращает документ, в состав которого входит текущий элемент. Если документа нет - возвращает ошибку.

 

**Тип атрибута**

Объект XmlElem


### `ExternalDispName`


**Тип:** Свойство


Возвращает основное отображаемое значение элемента для внешнего использования, например, для внешних ссылок. Обычно используется для элементов каталога. Для простых элементов совпадет с PrimaryDispName, если элемент иерархический - возвращает всю цепочку имен по иерархии.

 

**Тип атрибута**

String


### `ForeignArray`


**Тип:** Свойство


Возвращает целевой массив (вычисляет выражение, описанное в атрибуте FOREIGN-ARRAY). Если атрибут FOREIGN-ARRAY в форме элемент не описан, возвращается ошибка.

    

 

**Тип атрибута**:

Объект


### `ForeignArrayCodeStr`


**Тип:** Свойство


Возвращает значение атрибута FOREIGEN-ARRAY в том виде, в котором оно был описан в форме.

 

**Тип атрибута**

String


### `ForeignDispName`


**Тип:** Свойство


Возвращает внешнее первичное отображаемое значение того объекта, на который ссылается атрибут FOREIGN-ELEM.

 

Для обычных элементов конструкция

person.org_id.ForeignDispName

эквивалентна

person.org_id.ForeignElem.ExternalDispName

 

Для элементов типа MULTIPLE результат составляется из всех значений одиночных элементов, разделенных через запятую.

Таки образом конструкция

candidate.profession_id.ForeignDispName

эквивалентна

ArrayMerge( candidate.profession_id, 'ForeignElem.ExternalDispName', ', ' ) 

 

**Тип атрибута**

String


### `ForeignElem`


**Тип:** Свойство


Возвращает соответствующий элемент целевого массива (описанного в атрибуте FOREIGN-ARRAY). Если элемент не найден - возвращает ошибку.

    

**Тип атрибута**

Объект


### `ForeignObjectUrl`


**Тип:** Свойство


Атрибут возвращает url объекта, на который ссылается атрибут ForeignElem из текущего элемента. Целевой массив должен быть каталогом.

Конструкция используется, когда необходимо быстро открыть документ, на который ссылается  атрибут ForeignElem.

 

В новой объектной модели конструкция

person.org_id.ForeignObjectUrl

эквивалентна

person.org_id.ForeignElem.ObjectUrl

но, в отличие от последней, не осуществляет поиск целевого элемента, а только использует имя каталога из FOREIGN-ELEM.

 

В старой объектной модели конструкция

person.org_id.ForeignObjectUrl

эквивалентна

UrlFromDocID( person.org_id )

 

**Тип атрибута**

String


### `ForeignPrimaryObjectUrl`


**Тип:** Свойство


Атрибут аналогичен ForeignObjectUrl, но, в случае, если от целевого объекта могут существовать наследуемые объекты, пытается найти конечный элемент в цепочке наследования, и возвращает его объектный url.

 

**Тип атрибута**

String

 

**Пример**

event.person_id.ForeignPrimaryObjectUrl может вернуть 'x-db-obj://data/person/0x4D59AF5E5947A89.xm'

либо 'x-db-obj://data/candidate/0x4D59AF5E5947A89.xm', если целевой объект является кандидатом.


### `Form`


**Тип:** Свойство


Возвращает ссылку на форму, по которой был открыт документ. Если документ открыт без формы, возвращается ошибка.

 

**Тип атрибута**

Объект XmlForm


### `FormElem`


**Тип:** Свойство


Возвращает ссылку на элемент формы текущего элемента. Причем такая ссылка возвращается независимо от того, был документ открыт по форме или нет. Если элемент был создан по форме, выдается ссылка на элемент формы, по которой он был открыт. Если нет - на динамический элемент формы, созданный специально для данного элемента.


### `FormPath`


**Тип:** Свойство


Возвращает полный путь до текущего элемента относительно корня формы.

    

 

**Тип атрибута**

String


### `HasValue`


**Тип:** Свойство


Возвращает true, если элемент содержит значение, и false, если не содержит. Этот универсальный атрибут позволяет создавать единый код для всех типов элементов. Так, например, если элемент имеет тип string, то при отсутствии значения элемент будет равен пустой строке.  Если же элемент имеет тип, например, integer, то при отсуствии значения элемент будет равен null. Атрибут HasValue работает с элементами любого типа, в т.ч. "variant".

    

**Тип атрибута**

Bool


### `ImageUrl`


**Тип:** Свойство


Возвращает url картинки элемента. Атрибут доступен для записей в каталоге или для корневых элементов документа, и вычисляет значение атрибутов  элемента IMAGE-URL или IMAGE-URL-EXPR, описанных в XMD-форме.

 

**Тип атрибута**

String


### `Instances`


**Тип:** Свойство


Возвращает массив из одного текущего элемента.

Аналогичный атрибут объекта XmlMultiElem возвращает массив из всех относящихся к нему фактических элементов. Позволяет создавать один и тот же код, пригодный как для обработки одиночных элементов, так и множественных элементов типа MULTIPLE.

 

**Тип атрибута**

Объект Array


### `IsDynamic`


**Тип:** Свойство


Возвращает true, если элемент является динамическим, и false, если он построен по форме.


### `IsFirstSibling`

Проверяет, является ли данный элемент первым среди дочерних элементов своего родительского элемента.

 

**Тип атрибута**

**Тип:** Свойство


Проверяет, является ли данный элемент последним среди дочерних элементов своего родительского элемента.

    

**Тип атрибута**

Bool

**Тип атрибута**

Bool

**Тип:** Свойство


Содержит значение true, если элемент является временным (т.е. элемент формы, по которой был открыт текущий элемент, содержит атрибут TEMP="1").

 

**Тип атрибута**

### `IsTopElem`

### `NextSibling`

Возвращает элемент, следующий по порядку за текущим в списке дочерних элементов родительского элемента. если текущий элемент является последним, возвращается ошибка.

Объект XmlElem


### `ObjectUrl`


**Тип:** Свойство


Работает только для записей в каталоге, или корневых элементов объектных документов. Возвращает url объекта.

Использование этого атрибута позволяет применять быстрые конструкции, например OpenDoc( record.ObjectUrl ). Атрибут берет элемент id от данной записи, имя объекта, и формирует правильный url автоматически.

Атрибут также работает в старой объектной модели.

 

**Тип атрибута**

String

**Тип атрибута**

Объект XmlDoc


### `OptForeignElem`


**Тип:** Свойство


Возвращает соответствующий элемент целевого массива (описанного в атрибуте FOREIGN-ARRAY). Если элемент не найден - возвращает  undefined.

 

**Тип атрибута**

Объект

**Тип:** Свойство

Объект Screen

**Тип:** Свойство

Объект XmlElem


### `PrevSibling`


**Тип:** Свойство


Возвращает предыдущий относительно текущего элемент в списке дочерних элементов родительского элемента. Если элемент является первым, возвращает ошибку.

    

**Тип атрибута**

Объект XmlElem


### `PrimaryDispName`


**Тип:** Свойство


Возвращает первичное отображаемое имя объекта. Метод доступен для записей в каталоге, либо для корнвых элементов объектных документов.

 

**Тип аргумента**

String


### `PrimaryKey`


**Тип:** Свойство


Возвращает элемент - первичный ключ текущего элемента. Первичный ключ должен быть описан в форме как PRIMARY-KEY. Для записи в каталоге первичным ключом автоматически считается элемент <id>.


### `PrimaryObjectUrl`


**Тип:** Свойство


Атрибут работает аналогично ObjectUrl, но учитывает возможность наследования. Атрибут проверяет наличие наследуемого элемента, и, если таковой есть, возвращает последний объект в цепочке наследования.

Например, в E-staff candidate наследуется от person. Если в этом случае применить атрибут PrimaryObjectUrl к объекту candidate, то будет возвращен url объекта candidate. Если применить атрибут PrimaryObjectUrl к объекту person, и person является кандидатом, то будет возвращен url объекта candidate.

 

**Тип атрибута**

String


### `ReadOnly`


**Тип:** Свойство


Флаг, обозначающей, что элемент доступен только на чтение. Если данные элемент установить в true, то последующие попытки изменить содержимое элемента или его дочерних элементов вызовут ошибку.

При помещении ссылки на какой-либо элемент в объект web-сервера Session, у такого элемента флаг ReadOnly автоматически устанавливается в true, для предотвращения возможных одновременных изменений элемента из разных потоков, что является не thread-safe операцией.

    

**Тип атрибута**:

Bool


### `Screen`


**Тип:** Свойство

Объект Screen

**Тип:** Свойство


Атрибут применим только к элементам типа "string". Возвращает длину строки в байтах.

### `Title`


**Тип:** Свойство

**Тип атрибута**

String


### `Type`


**Тип:** Свойство


Возвращает тип данных элемента: 'string', 'integer' и т.д.

 

**Тип атрибута**

String


### `UseCData`


**Тип:** Свойство


Означает, что текущий элемент будет сохраняться не как обычный XML-элемент, а как CData. Этот атрибут может быть использован, например, при экспорте атрибута во внешние системы.

Атрибут доступен только на запись.

 

**Тип атрибута**

Bool


### `Value`


**Тип:** Свойство


Значение элемента. Явно вызывать его не обязательно, т.к. это свойство по умолчанию, т.е. при чтении значения элемента  выражения 'elem.Value' и просто 'elem' тождественны. Однако, при записи значения иногда требуется вызывать этот атрибут в явном виде. Если вызывается функция или метод, которая возвращает элемент (например, метод AddChild), то для установки значения нужно в явном виде указывать атрибут Value.

 

**Пример**

entries.AddChild().Value = 10;


### `Xml`


**Тип:** Свойство


Возвращает содержимое текущего элемента, включая дочерние элементы, если они есть, в виде строки в формате XML.

**Смотри также** XmlValue.

    

**Тип атрибута**

String


### `XmlValue`


**Тип:** Свойство


Возвращает значение текущего элемента, оформленное по правилам XML,  в том виде, в котором оно включается между тэгами.

Строка (string) будет замаскирована знаками <&, число (integer) останется без изменений, логическое значение (boolean) будет представлено как 0 или 1, дата будет записана в виде даты в формате XML.

 

**Тип атрибута**

String


### `XQueryLiteral`


**Тип:** Свойство


Формирует константу в языке XQuery со значением текущего элемента, в зависимости от типа значения. Действует аналогично функции XQueryLiteral().

    

**Тип атрибута**

String


## Элементы без подробного описания в исходном фрагменте

В исходном списке методов присутствуют следующие методы, но отдельные описания для них в предоставленном тексте отсутствуют:


- `Child()`
- `EvalCodeAttr()`
- `ObtainAttr()`
- `UpdateSecondaryData()`


Для Codex/LLM: при генерации рабочего кода не опираться на догадки по этим методам. Если задача требует их использования, нужно запросить дополнительный источник документации или найти реальные примеры в проектном коде.
