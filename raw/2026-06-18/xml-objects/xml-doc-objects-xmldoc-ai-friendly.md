# SP-XML: объект `XmlDoc`

## Назначение файла

Этот файл содержит структурированное описание объекта `XmlDoc` платформы SP-XML/WebTutor/WebSoft HCM. Материал подготовлен в AI-friendly формате для последующего включения в локальную документацию и использования Codex/LLM при разработке.

В исходном блоке также обозначены связанные объекты XML-документа:

- `XmlDoc`
- `XmlElem`
- `XmlMultiElem`

Подробно в данном файле описан только объект `XmlDoc`.

## Важные правила для AI-ассистента

1. Не считать серверный язык WebTutor обычным JavaScript. Синтаксис похож на JavaScript, но это среда SP-XML/WebTutor со своими объектами, методами и ограничениями.
2. Не выдумывать отсутствующие методы, свойства, сигнатуры и поведение. Если в этом файле нет описания конкретного свойства или метода, нужно запросить дополнительную документацию.
3. Для объектных документов нужно различать старую и новую объектную модель:
   - `XmlDoc.BindToDb()` указан как устаревший метод.
   - В новой объектной модели предпочтительнее `DefaultDb.OpenNewObjectDoc()` или `XmlDoc.BindToDbObjectType()`.
4. Некоторые методы и свойства являются экспериментальными или редко используемыми. Это явно отмечено в соответствующих разделах.
5. При массовой обработке документов агентами или конвертерами важно учитывать свойства, отключающие лишние действия при сохранении: `WriteDocInfo`, `WriteFt`, `RunActionOnBeforeSave`, `WriteAllNodes`, `WriteMaskLineBreaks`.
6. При работе с потоками через `EvalThread()` учитывать, что содержимое документа не является thread-safe.

## Краткая карта объекта `XmlDoc`

`XmlDoc` представляет XML-документ. Через него выполняются операции загрузки, сохранения, привязки к базе, обращения к корневому элементу, доступа к форме, экрану, url, флагам сохранения и служебным данным документа.

## Свойства `XmlDoc`

| Свойство | Назначение | Тип / результат | Особенности |
|---|---|---|---|
| `AutoSave` | Автоматическое сохранение измененного документа при выходе из системы или остановке сервера. | `Bool` | Обычно используется у документов с пользовательскими настройками. |
| `DocID` | Возвращает id объектного документа. | `Integer` или `String` | Только для объектных документов. Если документ не объектный, возникает ошибка. Id вычисляется на основании url. |
| `Form` | Указатель на форму, по которой открыт документ. | `XmlForm` | Если документ не имеет формы, возникает ошибка. |
| `FormUrl` | Возвращает url формы, по которой открыт документ. | `String` | Если документ не имеет формы, возвращается пустая строка. |
| `HasWriteAccess` | Показывает, открыт ли документ с правом записи. | `Bool` | `true` — документ открыт на запись; `false` — только на чтение. |
| `IsChanged` | Флаг изменения документа. | `Bool` | Обычно устанавливается автоматически при изменении данных в UI. В исходном описании указано, что в данный момент доступен только на чтение. |
| `IsSeparated` | Показывает, открыт ли документ как независимый, не привязанный к базе. | `Bool` | По умолчанию `false`. Может быть `true`, если документ открыт с опцией `Separated`. При сохранении такого документа не вызывается `OnSave` и т.д. |
| `LastSavedData` | Предыдущая сохраненная версия документа. | `XmlDoc` | Доступен в `OnSave()`, если для формы установлен `USE-LAST-SAVED-DATA="1"`. Используется для сравнения старых и текущих значений. |
| `NeverSaved` | Флаг, что документ создан и еще ни разу не сохранялся. | `Bool` | Полезен для логики первого сохранения. |
| `ObjectID` | Возвращает или устанавливает id объектного документа. | `Integer` или `String` | Новая версия `DocID` для новой объектной модели. Доступен на чтение и запись. Если документ не объектный, возникает ошибка. |
| `OptScreen` | Возвращает экран, если документ открыт в экране. | `Screen` или `undefined` | Безопасная альтернатива `Screen`, так как при отсутствии экрана возвращает `undefined`. |
| `RunActionOnBeforeSave` | Управляет вызовом метода формы `OnBeforeSave` при сохранении. | `Bool` | По умолчанию `true`. Можно поставить `false` для оптимизации массовой обработки документов. |
| `RunActionOnSave` | Свойство перечислено в исходном списке. | Нет данных | В исходном блоке нет отдельного описания. Не выдумывать поведение. |
| `Screen` | Возвращает ссылку на экран, если документ открыт в экране. | `Screen` | Если документ не открыт в экране, возникает ошибка. См. также `OptScreen`. |
| `TempData` | Объект для хранения произвольных временных данных. | Стандартный объект в режиме совместимости | Данные существуют только пока открыт документ. Можно использовать для передачи данных между `OnBeforeSave()` и `OnSave()`. |
| `TopElem` | Возвращает корневой элемент документа. | `XmlElem` | Более универсальный способ доступа к корневому элементу, чем обращение по имени корневого элемента. |
| `Url` | Возвращает или устанавливает url документа. | `String` | Если документ не имеет url, возвращается пустая строка. Атрибут доступен на запись. |
| `WriteAllNodes` | Управляет сохранением всех полей документа. | `Bool` | По умолчанию `false`: пустые поля не экспортируются. Для полной выгрузки/экспорта следует установить `true`. |
| `WriteAsync` | Свойство перечислено в исходном списке. | Нет данных | В исходном блоке нет отдельного описания. Не выдумывать поведение. |
| `WriteDocInfo` | Управляет записью структуры `doc_info` при сохранении. | `Bool` | По умолчанию `true`. При массовой программной обработке иногда целесообразно установить `false`. |
| `WriteFt` | Управляет полнотекстовым индексированием при сохранении. | `Bool` | По умолчанию `true`. Можно установить `false` для оптимизации массовой обработки, например в конвертерах. |
| `WriteMaskLineBreaks` | Управляет маскированием переносов строк при сохранении XML. | `Bool` | По умолчанию `true`: переносы строк маскируются последовательностью `&#10&#13`. При экспорте можно установить `false`. |

## Методы `XmlDoc`

| Метод | Назначение | Возвращает | Особенности |
|---|---|---|---|
| `BindToDb()` | Преобразует документ в объектный с присвоением нового id и url. | `undefined` | Устаревший метод. |
| `BindToDbObjectType()` | Преобразует документ в объектный с присвоением нового id и url для заданного типа объекта. | Нет | Используется в новой объектной модели. |
| `EvalThread()` | Запускает отдельный поток относительно данного документа. | Нет | Документ становится базовым указателем `This` внутри потока. Содержимое документа не thread-safe. |

| `KillActiveThread()` | Дает сигнал потоку, запущенному через `EvalThread()`, прекратить исполнение. | Нет | Поток не завершается мгновенно; завершение происходит на ближайшей контрольной точке с ошибкой `Cancel`. |
| `LoadFromLds()` | Загружает документ сервера приложений. | `object (SpXml)` | Экспериментальная функция для синхронизации документов. |
| `MakeReport()` | Метод перечислен в исходном списке. | Нет данных | В исходном блоке нет отдельного описания. Не выдумывать поведение. |
| `PrepareLastSavedData()` | Принудительно подготавливает предыдущую сохраненную версию документа. | Нет | Экспериментальный метод. См. также `LastSavedData`. |
| `Reload()` | Заново загружает содержимое документа из его url. | Нет данных | Используется для перечитывания документа. |
| `RemoveFromCache()` | Удаляет документ из кэша документов. | Нет данных | См. также `SetCached()`. |
| `Save()` | Сохраняет документ. | Нет | Без аргумента сохраняет под существующим url. С аргументом устанавливает url документа и сохраняет его. |
| `SaveToLds()` | Принудительно сохраняет документ на сервере приложений. | Нет данных | Экспериментальный метод для синхронизации. |
| `SaveToStream()` | Сохраняет содержимое документа в поток. | Нет | Может принимать опции экспорта XML-документов. |
| `SaveToUrl()` | Сохраняет содержимое документа под заданным url. | Нет | Текущий url документа при этом не изменяется. |
| `SetCached()` | Добавляет документ в кэш документов. | Нет | Обратная операция к `RemoveFromCache()`. |
| `SetChanged()` | Устанавливает флаг модификации документа. | Нет | По умолчанию устанавливает `true`. |
| `SetShared()` | Добавляет документ в список документов, чьи имена доступны в глобальном окружении. | Нет | Делает документ доступным через глобальное окружение. |
| `SubElem()` | Возвращает элемент документа по полному пути относительно документа. | `XmlElem` | Путь включает корневой элемент. |
| `TopOrChildElem()` | Возвращает корневой или дочерний элемент по имени. | `XmlElem` | Редко используемая функция. |
| `UpdateValues()` | Принудительно пересчитывает все вычисляемые элементы документа. | Нет | Пересчитывает поля с атрибутами `EXPR` или `EXPR-INIT`. |

## `XmlDoc.BindToDb()`

Преобразует документ, как правило вновь созданный, в объектный документ с присвоением нового id и соответствующего url.

### Статус

Устаревший метод.

В новой объектной модели используется `DefaultDb.OpenNewObjectDoc()`, который сразу создает новый объектный документ и присваивает ему id и url. Также в новой объектной модели используется `XmlDoc.BindToDbObjectType()`.

### Синтаксис

```javascript
XmlDoc.BindToDb([arg1])
```

### Аргументы

| Аргумент | Тип | Обязательность | Описание |
|---|---|---|---|
| `arg1` | `String` | Необязательный | Имя базы. |

### Результат

```javascript
undefined
```

### AI-заметка

Для нового кода не выбирать этот метод автоматически. Сначала проверить, можно ли использовать новую объектную модель: `DefaultDb.OpenNewObjectDoc()` или `BindToDbObjectType()`.

## `XmlDoc.Save()`

Сохраняет документ.

Если аргумент не указан, документ сохраняется под существующим url. Если аргумент указан, метод устанавливает url документа и сохраняет документ под этим url.

### Синтаксис

```javascript
XmlDoc.Save()
XmlDoc.Save(docUrl)
```

### Аргументы

| Аргумент | Тип | Обязательность | Описание |
|---|---|---|---|
| `docUrl` | `String` | Необязательный | Url, под которым должен быть сохранен документ. |

### Результат

Нет.

### Связанные свойства сохранения

- `WriteDocInfo`
- `WriteFt`
- `WriteAllNodes`
- `WriteMaskLineBreaks`
- `RunActionOnBeforeSave`
- `IsSeparated`

## `XmlDoc.SaveToUrl()`

Сохраняет содержимое документа под заданным url.

Текущий url документа при этом не изменяется.

### Синтаксис

```javascript
XmlDoc.SaveToUrl(destUrl)
XmlDoc.SaveToUrl(destUrl, options)
```

### Аргументы

| Аргумент | Тип | Обязательность | Описание |
|---|---|---|---|
| `destUrl` | `String` | Да | Новый url. |
| `options` | `String` | Необязательный | Опции экспорта документа. См. опции экспорта XML-документов. |

### Результат

Нет.

### Отличие от `Save(docUrl)`

- `Save(docUrl)` устанавливает url документа и сохраняет документ.
- `SaveToUrl(destUrl)` сохраняет содержимое под заданным url, но текущий url документа не изменяет.

## `XmlDoc.EvalThread()`

Запускает выполнение отдельного потока относительно данного документа.

В отличие от обычного объекта `Thread`, этот метод позволяет удобнее работать с потоками. Созданный поток получает документ в качестве базового указателя `This`. Можно создать документ с набором полей, содержащих методы, и запустить поток на основании этого документа. Поток будет видеть по умолчанию все поля этого документа.

По завершении потока этот документ автоматически освободится.

### Синтаксис

```javascript
XmlDoc.EvalThread(code)
```

### Аргументы

| Аргумент | Тип | Обязательность | Описание |
|---|---|---|---|
| `code` | `String` | Да | Строка, содержащая код. |

### Результат

Нет.

### Пример

```javascript
doc = OpenNewDoc('rcr_publich_vacancy.xmd');
doc.EvalThread('run()');
```

### Важное предупреждение о потокобезопасности

Содержимое документа не является thread-safe. Код, вызывающий `EvalThread()`, не должен обращаться к содержимому документа после запуска потока, если поток изменяет содержимое документа.

---

## `XmlDoc.KillActiveThread()`

Дает сигнал потоку, запущенному через `EvalThread()` относительно данного документа, прекратить исполнение.

Вызов метода не означает, что поток завершится мгновенно. Когда поток дойдет до ближайшей контрольной точки, поддерживающей ликвидацию потока, исполнение завершится с ошибкой `Cancel`, то есть операция будет отменена пользователем.

### Синтаксис

```javascript
XmlDoc.KillActiveThread()
```

### Аргументы

Нет данных в исходном блоке.

### Результат

Нет.

### См. также

- `XmlDoc.EvalThread()`

# Подробное описание свойств

## `XmlDoc.HasWriteAccess`

Возвращает `true`, если документ открыт с правом на запись, и `false`, если документ открыт только на чтение.

### Тип

`Bool`.

---

## `XmlDoc.TempData`

Свойство `TempData` содержит объект, в котором можно хранить произвольные данные.

Данные хранятся только на время, пока открыт документ. Свойство может использоваться для передачи данных между `OnBeforeSave()` и `OnSave()`.

`TempData` представляет собой стандартный объект в режиме совместимости.

### Тип

Стандартный объект в режиме совместимости.

---

## `XmlDoc.DocID`

Возвращает id документа только для объектных документов.

Id вычисляется на основании url документа. Если документ не является объектным, атрибут возвращает ошибку.

### Результат

`Integer` или `String`.

### См. также

- `XmlDoc.ObjectID`

---

## `XmlDoc.FormUrl`

Возвращает url формы, по которой был открыт документ.

Если документ не имеет формы, возвращается пустая строка.

### Тип

`String`.

---

## `XmlDoc.Screen`

Если документ открыт в экране, возвращает ссылку на экран. Иначе выдает ошибку.

### Тип

`Screen`.

### См. также

- `XmlDoc.OptScreen`

---

## `XmlDoc.TopElem`

Возвращает корневой элемент документа.

К корневому элементу также можно обратиться, используя его имя, но `TopElem` является более универсальным способом.

### Результат

`XmlElem`.

### Пример использования

```javascript
root = doc.TopElem;
```

---

## `XmlDoc.Url`

Возвращает url документа.

Документ может не иметь url. В этом случае возвращается пустая строка.

Атрибут доступен на запись.

### Результат

`String`.

---

## `XmlDoc.WriteDocInfo`

По умолчанию имеет значение `true`.

Если значение `true`, при сохранении документа записывается структура `doc_info`, содержащая:

- дату создания из поля `creation_date` документа;
- дату изменения из поля `last_mod_date` документа;
- логин пользователя, создавшего документ;
- логин пользователя, изменившего документ.

В отдельных случаях целесообразно присваивать `false`, если происходит массовая обработка документов программным способом, например при помощи агента.

### Тип

`Bool`.

### AI-заметка

При массовом изменении большого количества документов можно рассмотреть установку `WriteDocInfo = false`, но только если это не нарушает требования аудита и бизнес-логики.

---

## `XmlDoc.ObjectID`

Возвращает id объектного документа.

Это новая версия атрибута `DocID`, разработанная для новой объектной модели.

В отличие от `DocID`, свойство:

- доступно на чтение и запись;
- поддерживает целочисленные и строковые ID.

Для документа, не являющегося объектным, возникает ошибка.

### Результат

`Integer` или `String`.

---

## `XmlDoc.Form`

Указатель на форму, по которой был открыт документ.

Если документ не имеет формы, возникает ошибка.

### Тип

`XmlForm`.

---

## `XmlDoc.IsChanged`

Флаг, обозначающий, что документ был изменен.

Как правило, этот флаг автоматически устанавливается при изменении пользователем данных в пользовательском интерфейсе. Его также можно читать и изменять программно, но в исходном описании отдельно указано, что в данный момент атрибут доступен только на чтение.

### Тип

`Bool`.

### AI-заметка

Для программной установки флага изменения использовать `XmlDoc.SetChanged()`, а не прямую запись в `IsChanged`, если нет дополнительной документации о допустимости записи.

---

## `XmlDoc.NeverSaved`

Флаг, обозначающий, что документ был только что создан и еще ни разу не сохранялся.

### Тип

`Bool`.

---

## `XmlDoc.WriteAllNodes`

Если свойство имеет значение `false`, при сохранении документов используется сокращенная форма записи: поля документа, не содержащие значений, не экспортируются.

Это поведение используется по умолчанию и необходимо для сокращения размера документа.

При выгрузке или экспорте документов обычно требуется сохранять документы в полном формате. Для этого свойству следует присвоить значение `true`. В этом случае будут сохраняться все поля документа.

### Тип / результат

`Bool`.

---

## `XmlDoc.WriteMaskLineBreaks`

По умолчанию имеет значение `true`.

Если значение `true`, переносы строк, сохраняющиеся в XML-документах, маскируются последовательностью:

```text
&#10&#13
```

Выставлять значение `false` имеет смысл при выгрузке или экспорте документов. В этом случае переносы строк будут выгружаться как есть, без маскировки.

### Тип

`Bool`.

---

## `XmlDoc.AutoSave`

Если атрибут имеет значение `true`, при выходе из системы или при остановке сервера измененные документы будут сохраняться автоматически.

Обычно установлен у документов, содержащих настройки пользователя.

### Тип

`Bool`.

---

## `XmlDoc.LastSavedData`

Если для формы документа установлен параметр `USE-LAST-SAVED-DATA="1"`, этот атрибут доступен в методе `OnSave()` документа.

Он устанавливается в соответствии с предыдущей сохраненной копией корневого элемента документа. Атрибут используется, когда нужно сравнить предыдущее значение с текущим и определить, что значение было изменено.

Для документа, который еще ни разу не был сохранен, атрибут вернет пустой корневой элемент, созданный по той же форме, что и текущий документ.

### Тип

`XmlDoc`.

### См. также

- `XmlDoc.PrepareLastSavedData()`

---

## `XmlDoc.WriteFt`

По умолчанию имеет значение `true`.

Если значение `true`, документ будет занесен в полнотекстовый индекс при сохранении.

Можно установить значение `false`; при этом полнотекстовое индексирование для данного документа будет отключено. Это может использоваться для оптимизации массовой обработки документов, например в конвертерах.

### Тип

`Bool`.

---

## `XmlDoc.RunActionOnBeforeSave`

По умолчанию имеет значение `true`.

Если значение `true`, при сохранении документа исполняется метод формы документа `OnBeforeSave`.

Можно установить `false`, например для оптимизации массовой обработки документов.

### Тип

`Bool`.

### Важное замечание

В исходном блоке описание `RunActionOnBeforeSave` было продублировано дважды. В этом файле оно оставлено один раз.

---

## `XmlDoc.IsSeparated`

По умолчанию имеет значение `false`.

Может иметь значение `true`, если документ был открыт с опцией `Separated`. В таком случае документ открывается как независимый документ, не привязанный к базе данных.

При сохранении такого документа не вызывается `OnSave` и другие связанные действия сохранения.

### Тип

`Bool`.

---

## `XmlDoc.OptScreen`

Если документ открыт в экране, возвращает ссылку на экран. Иначе возвращает `undefined`.

### Тип

`Screen` или `undefined`.

### См. также

- `XmlDoc.Screen`

---

# Практические сценарии

## Получить корневой элемент документа

```javascript
root = doc.TopElem;
```

## Получить вложенный элемент по полному пути

```javascript
lastStateID = doc.SubElem('person.view.last_state_id');
```

## Сохранить документ под текущим url

```javascript
doc.Save();
```

## Сохранить документ с установкой нового url

```javascript
doc.Save('x-local://some/path/document.xml');
```

## Сохранить копию содержимого под другим url без изменения текущего url документа

```javascript
doc.SaveToUrl('x-local://export/document.xml');
```

## Создать объектный документ новой объектной модели

```javascript
doc = OpenNewDoc('x-app://rcr/rcr_candidate.xmd');
doc.BindToDbObjectType('data', 'candidate');
```

## Принудительно пересчитать вычисляемые поля

```javascript
doc.UpdateValues();
```

## Пометить документ как измененный

```javascript
doc.SetChanged(true);
```

## Использовать временные данные между обработчиками сохранения

```javascript
// OnBeforeSave
doc.TempData.someFlag = true;

// OnSave
if (doc.TempData.someFlag) {
    // логика после сохранения
}
```

## Оптимизация массовой обработки документов

При массовой обработке документов можно рассмотреть отключение отдельных действий сохранения:

```javascript
doc.WriteDocInfo = false;
doc.WriteFt = false;
doc.RunActionOnBeforeSave = false;
doc.Save();
```

Важно: это может повлиять на аудит, полнотекстовый поиск и бизнес-логику формы. Перед использованием нужно понимать последствия.

## Полный экспорт документа

```javascript
doc.WriteAllNodes = true;
doc.WriteMaskLineBreaks = false;
doc.SaveToUrl('x-local://export/full_document.xml');
```
