# SP-XML: Преобразование форматов и кодировок

## Назначение раздела

В этом разделе собраны встроенные функции SP-XML, связанные с преобразованием строк, байтовых данных, кодировок, MIME/URL/XML/SQL/XQuery-литералов, Base64, HEX, RTF, multipart/form-data и встроенным строковым шифрованием.

Материал предназначен для локальной AI-friendly документации WebTutor / WebSoft HCM. Его можно использовать как справочник при разработке серверных шаблонов, API-страниц, агентов и другой логики на платформе SP-XML.

> Важно: SP-XML серверный язык WebTutor синтаксически похож на JavaScript, но не является обычным JavaScript. При генерации кода нужно использовать именно функции и правила платформы WebTutor / SP-XML.

## Функции проверки и восстановления кодировки

### `AdjustUtf8()`

Проверяет строку в UTF-8 на валидность. Если в строке обнаруживается некорректная для UTF-8 последовательность байт, она заменяется на символ `?`.

Возвращается исходная либо измененная строка.

#### Синтаксис

```spxml
AdjustUtf8( str )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `string` | да | Исходная строка. |

#### Результат

`string` — исходная строка или строка с заменой некорректных UTF-8 последовательностей.

#### Когда использовать

Используется, когда нужно безопасно обработать строку, которая должна быть UTF-8, но может содержать поврежденные байтовые последовательности.

---

### `AutoDecodeCharset()`

Проверяет, является ли строка валидной строкой в кодировке UTF-8. Если да, возвращает саму строку. Если нет, декодирует строку так, как если бы она была в однобайтовой кодировке, установленной по умолчанию, обычно `windows-1251`.

Если приложение работает не в кодировке UTF-8, функция возвращает исходную строку без изменений.

Функция поддерживается только на сервере или десктопном клиенте.

#### Синтаксис

```spxml
AutoDecodeCharset( str )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `string` | да | Исходная строка. |

#### Результат

`string` — исходная строка либо строка, декодированная из кодировки по умолчанию.

#### Ограничения

- Поддерживается только на сервере или в десктопном клиенте.
- Если приложение не работает в UTF-8, строка возвращается без изменений.

---

### `RestoreBrokenCharsetEncoding()`

Проверяет строку на некорректную кодировку, вызванную одним из типовых случаев неправильной двойной конвертации.

Возвращает новую строку в правильной кодировке, если ее удалось восстановить. В противном случае возвращается `undefined`.

#### Синтаксис

```spxml
RestoreBrokenCharsetEncoding( str )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `string` | да | Исходная строка. |

#### Результат

`string` или `undefined`.

#### Когда использовать

Используется для попытки восстановления текста после ошибочной перекодировки, например при проблемах вида «кракозябры» после двойного преобразования кодировок.

## Преобразование кодировок

### `DecodeCharset()`

Переводит строку из заданной кодировки в кодировку, используемую в программе по умолчанию.

#### Синтаксис

```spxml
DecodeCharset( str, charset )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `string` | да | Строка, которую нужно перевести. |
| `charset` | `string` | да | Название исходной кодировки. |

#### Результат

`string`.

#### Поддерживаемые кодировки

На текущий момент поддерживаются:

- `windows-1251`
- `windows-1252`
- `utf-8`
- `utf-16`
- `koi8-r`
- `cp-866`

---

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `string` | да | Строка. |
| `charset` | `string` | да | Имя целевой кодировки. |

#### Результат

`string`.

#### Поддерживаемые кодировки

На текущий момент поддерживаются:

- `windows-1251`
- `windows-1252`
- `utf-8`
- `utf-16`

## MIME и заголовки

### `EncodeMimeHeaderValue()`

Кодирует строку, которая будет использоваться внутри MIME-заголовка, если содержимое требует кодирования, например содержит не-ASCII символы.

Результат имеет вид:

```text
=?utf-8?B?<base64data>
```

Если содержимое строки не требует кодирования, возвращается исходная строка.

#### Синтаксис

```spxml
EncodeMimeHeaderValue( str )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `string` | да | Строка для MIME-заголовка. |

#### Результат

`string`.

#### Когда использовать

Используется при формировании почтовых MIME-заголовков, например тем писем или имен вложений, если они могут содержать кириллицу или другие не-ASCII символы.

---

### `ParseHeaderPairs()`

Преобразует строку со значением поля MIME-заголовка в стандартный объект, содержащий список вспомогательных параметров.

#### Синтаксис

```spxml
ParseHeaderPairs( str )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `string` | да | Строка MIME-заголовка с параметрами. |

#### Результат

`object`.

#### Пример

```spxml
ParseHeaderPairs( 'text/html; charset=utf-8' )
```

Вернет:

```json
{
  "charset": "utf-8"
}
```

## XML

### `XmlEscape()`

Маскирует определенные символы в строке, чтобы результат мог быть использован в XML между тегами либо внутри атрибута.

#### Синтаксис

```spxml
XmlEscape( str )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `string` | да | Исходная строка. |

#### Результат

`string`.

#### Пример

```spxml
XmlEscape( 'aaa < bbb & ccc > ddd' )
```

Вернет:

```text
aaa &lt; bbb &amp; ccc &gt; ddd
```

---

### `XmlAttrEncode()`

Маскирует аргумент для вставки в XML в качестве значения атрибута.

В результате действия функции маскируются:

| Символ | Результат |
|---|---|
| перевод строки | `&#10;` |
| табуляция | `&#09;` |
| `<` | `&lt;` |
| `&` | `&amp;` |
| `"` | `&quot;` |

#### Синтаксис

```spxml
XmlAttrEncode( str )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `string` | да | Аргумент. |

#### Результат

`string`.

---

## SQL и XQuery литералы

### `SqlLiteral()`

Кодирует значение как константу, то есть литерал языка SQL.

Функция используется при генерации SQL-выражений из программы.

#### Синтаксис

```spxml
SqlLiteral( arg )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `arg` | `any` | да | Аргумент произвольного типа. |

#### Результат

`string`.

#### Правила преобразования

| Тип значения | Результат |
|---|---|
| `null` | Строка `NULL`. |
| `string` | Строка заключается в кавычки, существующие кавычки внутри строки маскируются по правилам SQL. |
| `integer` | Переводится в соответствующее строковое значение. |
| `date` | Переводится в строку с датой в SQL-формате. |

#### Когда использовать

Используется при ручной генерации SQL-строк, чтобы корректно оформить литералы и избежать синтаксических ошибок из-за кавычек, дат и `null`.

---

### `XQueryLiteral()`

Кодирует аргумент как константу XQuery.

Функция используется при генерации XQuery-выражений из программы.

#### Синтаксис

```spxml
XQueryLiteral( arg )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `arg` | `any` | да | Аргумент. |

#### Результат

`string`.

#### Правила преобразования

| Тип значения | Результат |
|---|---|
| `null` | `null()` |
| `string` | Значение заключается в одинарные кавычки, существующие кавычки внутри строки маскируются по правилам XQuery. |
| `integer` | Переводится в соответствующее строковое значение. |
| `real` | Переводится в соответствующее строковое значение. |
| `bool` | `true()` или `false()`. |
| `date` | Строка вида `date( '2011-01-30T10:30:00' )`. |

#### Когда использовать

Используется при динамическом формировании XQuery-запросов, особенно если в запрос подставляются строковые значения, даты, булевы значения или `null`.

## Практические замечания для разработки

### 1. XML-вывод

Для вывода пользовательских строк в XML нужно использовать функции XML-маскирования:

- `XmlEscape()` — для текста между тегами или общего XML-экранирования;
- `XmlAttrEncode()` — для значений XML-атрибутов;
- `XmlStr()` — для формирования простого XML-тега со значением.

### 2. Генерация XQuery и SQL

При динамической сборке запросов нельзя вручную подставлять строковые значения без кодирования. Для этого предусмотрены:

- `SqlLiteral()` — для SQL;
- `XQueryLiteral()` — для XQuery.

Это особенно важно при подстановке строк, дат, булевых значений и `null`.

### 3. URL-параметры

Для формирования URL-параметров используются:

- `UrlEncode()` — кодирование одного значения;
- `UrlDecode()` — декодирование значения;
- `UrlEncodeQuery()` — сборка query string из объекта;
- `UrlEncodeQueryExt()` — сборка query string из объекта с возможностью указания кодировки;
- `UrlEncode16()` — кодирование через UTF-16 escape-последовательности.

### 4. Работа с бинарными данными

В SP-XML строка может использоваться как контейнер байтовых данных. Для таких случаев используются:

- `Base64Encode()` / `Base64Decode()`;
- `HexData()` / `DataFromHex()`.

### 5. Проблемы кодировки

Для диагностики и исправления проблем кодировки полезны:

- `AdjustUtf8()` — проверка валидности UTF-8;
- `AutoDecodeCharset()` — автоматическое декодирование в UTF-8-приложении;
- `RestoreBrokenCharsetEncoding()` — попытка восстановления после неправильной двойной конвертации;
- `DecodeCharset()` / `EncodeCharset()` — явное преобразование кодировок.

## Минимальный индекс для поиска LLM

- UTF-8: `AdjustUtf8`, `AutoDecodeCharset`, `Utf16ToUtf8`
- broken charset / кракозябры: `RestoreBrokenCharsetEncoding`, `DecodeCharset`, `EncodeCharset`
- MIME: `EncodeMimeHeaderValue`, `ParseHeaderPairs`
- Base64: `Base64Encode`, `Base64Decode`
- HEX: `HexData`, `DataFromHex`
- XML: `XmlEscape`, `XmlAttrEncode`, `XmlStr`
- URL: `UrlEncode`, `UrlDecode`, `UrlEncode16`, `UrlEncodeQuery`, `UrlEncodeQueryExt`
- HTTP form-data: `MultipartFormEncode`
- SQL: `SqlLiteral`
- XQuery: `XQueryLiteral`
- RTF: `RtfEncode`, `RtfToPlainText`
- encryption: `StrStdEncrypt`, `StrStdDecrypt`

## Черновые вопросы для уточнения по документации

Эти пункты желательно проверить на реальной среде или в оригинальной документации:

1. Точный синтаксис `AutoDecodeCharset()` в источнике указан без отдельного блока синтаксиса. Предположительно используется форма `AutoDecodeCharset( str )`.
2. Для `ParseHeaderPairs()` в исходном фрагменте не указан явный список аргументов; по примеру предполагается один строковый аргумент.
3. Для `DataFromHex()` в исходном фрагменте не указан явный блок аргументов; по примеру предполагается один строковый аргумент.
4. Для `MultipartFormEncode()` желательно уточнить, как задается boundary и возвращает ли функция только тело запроса или также требует отдельной установки HTTP-заголовка `Content-Type`.
5. Для `StrStdEncrypt()` / `StrStdDecrypt()` желательно уточнить область применения: можно ли использовать для секретов, или это только внутреннее обратимое кодирование платформы.
