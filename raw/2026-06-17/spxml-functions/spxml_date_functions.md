# SP-XML: Работа с датами

## Назначение раздела

Этот файл описывает встроенные функции SP-XML для создания, проверки, изменения, сравнения, преобразования и форматирования дат.

Раздел полезен при разработке серверной логики WebTutor/WebSoft HCM, где часто требуется:

- создать дату из строки или отдельных компонентов;
- проверить корректность даты;
- получить год, месяц, день, часы, минуты или секунды;
- сдвинуть дату на заданное количество секунд;
- сравнить две даты;
- преобразовать дату между часовыми поясами;
- вывести дату в строку для интерфейса, XML, MIME или локального формата.

> Важно: серверный язык SP-XML похож по синтаксису на JavaScript, но не является обычным JavaScript. Для работы с датами нужно опираться на функции платформы и учитывать особенности WebTutor/WebSoft HCM.

## Важные особенности

### 1. `Date()` и `OptDate()` — не одно и то же

`Date()` является конструктором объекта даты, но при некорректных календарных значениях может вести себя по-разному в десктопной и web-версиях.

Для проверки корректности даты рекомендуется использовать `OptDate()` или `IsValidDate()`.

Пример:

```spxml
OptDate( "2019-04-20" )
OptDate( 2019, 04, 20 )
IsValidDate( Date( '2020-02-29' ) )
```

### 2. Разница дат возвращается в секундах

`DateDiff()` возвращает разницу между двумя датами в секундах. Если первая дата меньше второй, результат будет отрицательным.

### 3. Часовой пояс может храниться внутри даты

Некоторые функции работают с часовым поясом, который хранится внутри значения даты:

- `DateNewTimeZone()`;
- `DateToLocalDate()`;
- `DateToTimeZoneDate()`;
- `TimeZone()`;
- `GetLocalTimeZone()`.

При сохранении даты в базу данных или передаче по сети информация о часовом поясе внутри даты может потеряться.

### 4. `DateNewTimeZone()` не изменяет время суток

`DateNewTimeZone()` меняет значение часового пояса внутри даты, оставляя часы, минуты и секунды без изменений. Эта функция обычно нужна для локальных вычислений.

### 5. `DateToTimeZoneDate()` сохраняет глобальное значение даты

`DateToTimeZoneDate()` преобразует дату к другому часовому поясу, оставляя глобальное значение даты без изменений.

# Создание и разбор даты

## `Date()`

Конструктор стандартного объекта даты.

### Синтаксис

```spxml
Date()
Date( date )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `date` | `Date` / `String` | Нет | Начальное значение даты или строка, содержащая дату. Если аргумент не указан, объект инициализируется текущей датой. |

### Результат

`Date`

### Примечание

Для проверки корректности календарной даты лучше использовать `OptDate()` или `IsValidDate()`.

---

## `OptDate()`

Конструирует значение типа `date`. Возвращает `undefined`, если указаны недопустимые аргументы.

### Синтаксис

```spxml
OptDate( str )
OptDate( year, month, day )
OptDate( year, month, day, hour, minute )
OptDate( year, month, day, hour, minute, second )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `String` | Да, для строковой формы | Строка с датой. |
| `year` | `Integer` | Да, для числовой формы | Год. |
| `month` | `Integer` | Да, для числовой формы | Месяц. |
| `day` | `Integer` | Да, для числовой формы | День. |
| `hour` | `Integer` | Нет | Час. |
| `minute` | `Integer` | Нет | Минута. |
| `second` | `Integer` | Нет | Секунда. |

### Результат

`Date` или `undefined`

### Примеры

```spxml
OptDate( "2019-04-20" )
OptDate( "2019-04-20T12:30" )
OptDate( "2019-04-20T12:30:00" )
OptDate( 2019, 04, 20 )
OptDate( 2019, 04, 20, 12, 30 )
OptDate( 2019, 04, 20, 12, 30, 0 )
```

### Рекомендация

`OptDate()` рекомендуется использовать, когда нужно проверить корректность даты, например 29 февраля.

Обычная функция `Date()` может вести себя по-разному на десктопной и web-версиях.

---

## `ParseDate()`

Пытается распознать строку с датой в одном из неформальных форматов.

Например:

```text
1 ноября 2011 года
```

Если распознать дату не удалось, возвращается ошибка.

### Синтаксис

```spxml
ParseDate( str )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `String` | Да | Строка с датой. |

### Результат

`Date`

### Рекомендация

Если строка содержит дату в стандартном формате, например `2011-11-01` или `01.11.2011`, рекомендуется использовать более простую и эффективную функцию `OptDate()`.

---

## `ParseMimeDate()`

### Синтаксис

## `IsValidDate()`

Проверяет валидность даты по календарю. Возвращает `true` или `false`.

Функция может потребоваться, поскольку не все функции, работающие с датами, имеют встроенную проверку валидности.

### Синтаксис

```spxml
IsValidDate( date )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `date` | `Date` | Да | Дата. |

### Результат

`Bool`

### Пример

```spxml
IsValidDate( Date( '2020-02-29' ) )
```

# Сравнение и изменение дат

## `DateDiff()`

Возвращает разницу между двумя датами в секундах.

Если первая дата меньше второй, разница будет отрицательным числом.

### Синтаксис

```spxml
DateDiff( date1, date2 )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `date1` | `Date` | Да | Дата 1. |
| `date2` | `Date` | Да | Дата 2. |

### Результат

`Integer`

---

## `DateOffset()`

Сдвигает дату на указанное число секунд.

Если значение второго аргумента отрицательное, дата сдвигается назад.

### Синтаксис

```spxml
DateOffset( date, seconds )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `date` | `Date` | Да | Заданная дата. |
| `seconds` | `Integer` | Да | Сдвиг в секундах. |

### Результат

`Date`

---

## `DateNewTime()`

Изменяет значение времени в заданной дате. Возвращает измененную дату.

Если указан только первый аргумент, функция возвращает дату без времени.

### Синтаксис

```spxml
DateNewTime( date )
DateNewTime( date, hour )
DateNewTime( date, hour, minute )
DateNewTime( date, hour, minute, second )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `date` | `Date` | Да | Дата. |
| `hour` | `Integer` | Нет | Час. |
| `minute` | `Integer` | Нет | Минута. |
| `second` | `Integer` | Нет | Секунда. |

### Результат

`Date`

# Форматирование даты в строку

## `StrDate()`

Преобразует дату в строку в формате, используемом по умолчанию в операционной системе.

Если в качестве аргумента передается `null` или пустая строка, функция возвращает пустую строку.

В качестве второго аргумента может быть передан стандартный объект `options` или два отдельных булевых аргумента `showTime` и `showSeconds` в устаревшей нотации.

### Синтаксис

```spxml
StrDate( date )
StrDate( date, options )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `date` | `Date` | Да | Дата. |
| `options` | `Object` | Нет | Объект с опциями форматирования. |

### Результат

`String`

### Опции

| Опция | Тип | Описание |
|---|---|---|
| `ShowTime` | `Bool` | Включать время. |
| `ShowSeconds` | `Bool` | Включать секунды во времени. |
| `ShowLocalTime` | `Bool` | Преобразовывать в местное время. |

## `StrTime()`

Преобразует значение времени внутри даты в строку.

Если время не содержится внутри даты, возвращается пустая строка.

### Синтаксис

```spxml
StrTime( date )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `date` | `Date` | Да | Дата. |

### Результат

`String`

### Пример

```spxml
StrTime( Date( '26.12.2011 10:45' ) )
```

вернет:

```text
10:45
```

---

## `StrXmlDate()`

Преобразует дату в строку в формате, используемом в XML.

Если в качестве аргумента передается `null` или пустая строка, функция возвращает пустую строку.

В качестве второго аргумента может быть передан стандартный объект `options` или булевый аргумент `showTime` в устаревшей нотации.

### Синтаксис

```spxml
StrXmlDate( date )
StrXmlDate( date, options )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `date` | `Date` | Да | Дата. |
| `options` | `Object` | Нет | Объект с опциями форматирования. |

### Результат

`String`

### Опции

| Опция | Тип | Описание |
|---|---|---|
| `ShowTime` | `Bool` | Включать время. |
| `ShowTimeZone` | `Bool` | Добавлять информацию о часовом поясе. |

### Примеры

```spxml
StrXmlDate( Date( '26.12.2011' ) )
```

вернет:

```text
2011-12-26
```

```spxml
StrXmlDate( Date( '26.12.2011 10:45' ) )
```

вернет:

```text
2011-12-26T10:45
```

# Практические рекомендации

## Проверка пользовательской даты

Для дат, которые приходят из пользовательского ввода, лучше использовать `OptDate()`:

```spxml
date = OptDate( str );
if ( date == undefined )
{
    // дата некорректна
}
```

## Сравнение дат

Для сравнения двух дат можно использовать `DateDiff()`:

```spxml
diff = DateDiff( date1, date2 );
```

- `diff > 0` — `date1` больше `date2`;
- `diff == 0` — даты равны;
- `diff < 0` — `date1` меньше `date2`.

## Сдвиг даты

Для прибавления или вычитания времени используется `DateOffset()`:

```spxml
// плюс один день
newDate = DateOffset( date, 86400 );

// минус один день
prevDate = DateOffset( date, -86400 );
```

## Обнуление времени

Если нужно получить дату без времени, можно использовать `DateNewTime()` только с первым аргументом:

```spxml
dateOnly = DateNewTime( date );
```

## Формат для XML

Для сохранения или передачи даты в XML-ориентированном формате используйте `StrXmlDate()`:

```spxml
str = StrXmlDate( date, { ShowTime: true, ShowTimeZone: true } );
```

## Формат для MIME-заголовков

Для MIME-дат используйте пару функций:

```spxml
str = StrMimeDate( date );
date = ParseMimeDate( str );
```

# Частые ошибки и спорные места источника

## `Date()` может быть недостаточно безопасен для проверки даты

В исходном описании отдельно указано, что `OptDate()` рекомендуется использовать для проверки корректности даты, потому что обычная функция `Date()` может вести себя по-разному в десктопной и web-версиях.

## `StrMimeDate()` в источнике указан с результатом `Date`

В описании функции `StrMimeDate()` сказано, что она преобразует дату в MIME-формат, а пример возвращает строку вида:

```text
Mon, 26 Dec 2011 10:45:00 +0400
```

По смыслу результат должен восприниматься как `String`, хотя в исходном тексте указан результат `Date`.

## Временная зона внутри даты может потеряться

Для функций `DateNewTimeZone()`, `DateToLocalDate()` и `DateToTimeZoneDate()` указано, что при сохранении даты в базе данных или передаче по сети информация о хранимом часовом поясе внутри даты может потеряться.

Поэтому такие функции лучше использовать для локальных вычислений, а не как единственный механизм сохранения timezone-логики.
