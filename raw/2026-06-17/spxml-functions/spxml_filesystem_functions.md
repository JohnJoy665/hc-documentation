# SP-XML: Работа с файловой системой

## 1. Назначение раздела

Функции работы с файловой системой в SP-XML используются для чтения, записи, проверки, создания, удаления и перемещения файлов и директорий.

Раздел важен для серверной разработки WebTutor/WebSoft HCM, так как через эти функции часто реализуются:

- загрузка и сохранение временных файлов;
- чтение шаблонов, XML, JSON, CSV и других файлов;
- формирование выгрузок и отчетов;
- работа с файлами во временных директориях;
- проверка существования файлов и директорий;
- перенос, копирование и удаление файлов;
- получение путей к системным и пользовательским директориям.

## 4. Важные понятия

### 4.1. Путь и URL

В документации часть функций принимает `path`, часть — `url`, а часть — `path` или `url`.

Для AI-friendly документации важно различать:

- `path` — путь в файловой системе, например `C:\temp\file.txt` или `/tmp/file.txt`;
- `url` — URL, который может отображаться в файловую систему;
- `pathOrUrl` / `filePathOrUrl` — функция допускает оба варианта.

### 4.2. Файл и директория

Некоторые функции проверяют только файл, некоторые — файл или директорию, а некоторые — именно директорию.

Например:

- `FileExists()` — проверяет существование файла по URL или пути;
- `FilePathExists()` — проверяет существование файла или директории по пути;
- `PathIsDirectory()` — проверяет, существует ли директория по указанному пути;
- `IsDirectory()` — проверяет, является ли указанный путь или URL директорией.

### 4.3. Текстовый и бинарный режим

Для чтения и записи файлов есть два основных режима:

- текстовый режим: `LoadFileText()`, `PutFileText()`;
- бинарный режим: `LoadFileData()`, `LoadFileDataRange()`, `PutFileData()`, `PutFileDataRange()`.

`LoadFileText()` учитывает BOM и кодировки. `LoadFileData()` возвращает бинарные данные в виде строки.

Возвращает путь к директории, из которой запущено приложение.

#### Синтаксис

```spxml

```

#### Аргументы

Нет.

#### Результат

`String`

Используется, когда нужно получить установочную или рабочую директорию приложения.


#### Аргументы

Нет.

#### Результат

`String`

---

## 6. Работа с путями

### FilePath()

Склеивает абсолютный и относительный файловые пути.

Если в качестве относительного пути передан абсолютный путь, вернется он же, а первый аргумент будет проигнорирован.

Функция унифицирует разделители в относительном пути (`/` и `\`) под файловую систему текущей операционной системы.

#### Синтаксис

```spxml
FilePath( basePath, addPath )
```

#### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `basePath` | `String` | Базовый абсолютный путь |
| `addPath` | `String` | Добавляемый относительный путь |

#### Результат

`String`

#### Пример использования

```spxml
path = FilePath( AppDirectoryPath(), 'logs/app.log' );
```

---

### FileName()

Извлекает имя файла из пути, переданного в качестве аргумента.

#### Синтаксис

```spxml
FileName( url )
```

#### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `url` | `String` | Путь к файлу |

#### Результат

`String`

#### Примеры

```spxml
FileName( 'work\data\doc.pdf' )
// вернет 'doc.pdf'

FileName( 'work\data\' )
// вернет 'data'
```

---

## 10. Чтение файлов

### LoadFileText()

Загружает содержимое файла с заданным путем с учетом наличия BOM.

Правила обработки:

- если файл начинается на UTF-16 BOM, происходит конвертация из UTF-16 в текущую кодировку UTF-8;
- если файл начинается на UTF-8 BOM, возвращается содержимое файла после BOM;
- если BOM отсутствует, происходит конвертация из однобайтовой кодировки по умолчанию, например Windows-1251, в текущую кодировку UTF-8.

#### Синтаксис

```spxml
LoadFileText( filePath )
```

#### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `filePath` | `String` | Путь к файлу |

#### Результат

`String`

#### Когда использовать

Использовать для чтения текстовых файлов: `.txt`, `.html`, `.xml`, `.json`, `.csv`, шаблонов и конфигураций.

---

### LoadFileData()

Загружает содержимое файла по заданному пути. Результат возвращается в виде строки, содержащей бинарные данные.

#### Синтаксис

```spxml
LoadFileData( path )
```

#### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `path` | `String` | Путь к файлу |

#### Результат

`String`

#### Когда использовать

Использовать для бинарных файлов: архивов, изображений, PDF, произвольных вложений и файлов, где нельзя применять преобразование кодировки.

---

### LoadFileDataRange()

Загружает содержимое фрагмента файла. Возвращает результат в виде бинарной строки.

Если конечная позиция превышает фактический размер файла, возвращается фрагмент до фактической верхней границы.

#### Синтаксис

```spxml
LoadFileDataRange( filePath, startPos, endPos )
```

#### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `filePath` | `String` | Путь к файлу |
| `startPos` | `Integer` | Начальная позиция |
| `endPos` | `Integer` | Конечная позиция |

#### Результат

`String`

#### Когда использовать

Использовать для чтения части большого бинарного файла без загрузки файла целиком.

## 11. Запись файлов

### PutFileText()

Сохраняет содержимое строки в файл с заданным путем с использованием UTF-8 BOM.

#### Синтаксис

```spxml
PutFileText( path, str )
```

#### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `path` | `String` | Путь к файлу |
| `str` | `String` | Текстовые данные |

#### Результат

`undefined`

#### Когда использовать

Использовать для сохранения текстовых файлов, когда нужен UTF-8 BOM.

> Примечание: в исходном фрагменте был указан только заголовок `PutFileText` и описание без блока синтаксиса. Синтаксис выше восстановлен по смыслу и по аналогии с `PutFileData(path, str)`.

---

### PutFileData()

Сохраняет содержимое строки в файл. Содержимое строки интерпретируется как бинарные данные.

#### Синтаксис

```spxml

```

#### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `path` | `String` | Путь к файлу |
| `str` | `String` | Данные |

#### Результат

`undefined`

#### Когда использовать

---

| `filePath` | `String` | Путь к файлу |
| `startPos` | `Integer` | Начальная позиция |
| `endPos` | `Integer` | Конечная позиция |
| `data` | `String` | Строка, содержащая бинарные данные |

#### Результат

`undefined`

## 16. Практические рекомендации

### 16.1. Для текстовых файлов использовать `LoadFileText()` и `PutFileText()`

Если файл является текстовым, лучше использовать текстовые функции:

```spxml
text = LoadFileText( filePath );
PutFileText( outPath, text );
```

`LoadFileText()` учитывает BOM и кодировки, поэтому безопаснее для HTML, XML, TXT, JSON и CSV.

### 16.2. Для бинарных файлов использовать `LoadFileData()` и `PutFileData()`

Для PDF, архивов, изображений и вложений не нужно применять текстовую перекодировку:

```spxml
data = LoadFileData( filePath );
PutFileData( newPath, data );
```

### 16.3. Для директорий лучше использовать `ObtainDirectory()`

Если директория может уже существовать, удобнее использовать:

```spxml
ObtainDirectory( dirPath, true );
```

Это безопаснее, чем напрямую вызывать `CreateDirectory()` без проверки.

### 16.4. Для путей использовать `FilePath()`

Не стоит склеивать пути обычной конкатенацией строк, особенно если код может выполняться на разных ОС.

Лучше:

```spxml
filePath = FilePath( basePath, 'reports/report.xml' );
```

### 16.5. Перед удалением всегда проверять путь

Перед `DeleteFile()` и особенно перед `DeleteDirectory()` желательно:

- проверить существование файла или директории;
- проверить, что путь находится внутри ожидаемой рабочей директории;
- не передавать в удаление путь, полученный напрямую от пользователя без нормализации и проверки.

## 19. Спорные и требующие проверки места источника

### 19.1. `PutFileText()`

В исходном материале функция указана как `PutFileText` без скобок и без полного блока синтаксиса.

По смыслу функция сохраняет строку в файл с использованием UTF-8 BOM. В этом файле синтаксис указан как:

```spxml
PutFileText( path, str )
```

Этот синтаксис нужно проверить по официальной карточке функции или экспериментально.

### 19.2. `FileExists()` и директории

`FileExists()` описана как проверка существования файла. Для проверки файла или директории лучше использовать `FilePathExists()`, а для директории — `PathIsDirectory()` или `IsDirectory()`.

### 19.3. `IsDirectory()` vs `PathIsDirectory()`

Обе функции связаны с проверкой директории, но формулировки отличаются:

- `IsDirectory()` проверяет, является ли указанный путь или URL директорией;
- `PathIsDirectory()` проверяет, существует ли директория по указанному пути.

Практически лучше выбирать функцию по типу входного значения: URL/path или только path.

### 19.4. `ReadDirectory()` vs `ReadDirectoryByPath()`

Разница в формате результата:

- `ReadDirectory()` возвращает URL вложенных файлов или директорий;
- `ReadDirectoryByPath()` возвращает полные пути.

## 20. Краткое резюме для AI-агента

При разработке на SP-XML:

1. Для сборки путей используй `FilePath()`, а не ручную конкатенацию.
2. Для текстовых файлов используй `LoadFileText()` и `PutFileText()`.
3. Для бинарных файлов используй `LoadFileData()` и `PutFileData()`.
4. Для создания директории, если она может отсутствовать, используй `ObtainDirectory()`.
5. Для временных файлов используй `ObtainTempFile()` или `ObtainSessionTempFile()`.
6. Для временной директории используй `ObtainTempDirectoryPath()`.
7. Перед удалением файлов и директорий обязательно проверяй путь.
8. `DeleteDirectory()` удаляет директорию вместе со всем содержимым.
9. `ReadDirectory()` возвращает URL, `ReadDirectoryByPath()` возвращает полные пути.
10. Не путай `FileExists()`, `FilePathExists()`, `IsDirectory()` и `PathIsDirectory()`.
