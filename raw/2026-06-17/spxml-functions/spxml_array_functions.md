# SP-XML: Работа с массивами

## 1. Общее описание

Массив в SP-XML — это объект, содержащий список значений. Элементами массива могут быть как простые значения, так и другие объекты.

Минимально любой массив можно перебирать последовательно:

```spxml
for ( elem in array )
{
    // обработка элемента
}
```

Часть массивов поддерживает **прямой доступ по индексу**. Для таких массивов заранее известно количество элементов, и к элементу можно обращаться как `array[i]`:

```spxml
elemsNum = ArrayCount( array );

for ( i = 0; i < elemsNum; i++ )
{
    elem = array[i];
}
```

Гарантированно прямой доступ поддерживают:

- стандартный массив `Array`;
- объект `XmlElem`.

Для остальных массивов поддержка прямого индексирования заранее неизвестна. Если нужен доступ по индексу, например для выбора случайного элемента или многократной обработки, массив лучше заранее привести к массиву прямого доступа через `ArrayDirect()` или `ArraySelectAll()`.

## 2. Важные особенности массивов в SP-XML

### 2.1. Последовательные и индексируемые массивы

В SP-XML массив может быть не обычным JavaScript-массивом, а абстрактным перечислителем или результатом запроса. Поэтому некоторые операции могут быть быстрыми, а некоторые — дорогими.

Например, `ArrayCount()` для обычного массива срабатывает мгновенно, но для результата XQuery может потребовать обращения к серверу или полного перебора элементов.

### 2.2. Когда использовать `ArrayDirect()` и `ArraySelectAll()`

Используйте `ArrayDirect()` или `ArraySelectAll()`, если:

- нужно обращаться к элементам по индексу;
- массив будет обрабатываться несколько раз;
- исходный массив является результатом XQuery;
- в цикле нужно удалять элементы массива;
- требуется выбрать случайный элемент;
- важно избежать повторного ленивого чтения данных.

`ArrayDirect()` предпочтительнее, когда нужно просто гарантировать прямой доступ: если массив уже индексируемый, функция вернет исходный массив.

`ArraySelectAll()` всегда возвращает стандартный массив `Array`, содержащий все элементы исходного массива.

## 5. Проверка массива и получение размера

### IsArray()

Проверяет, является ли аргумент массивом.

#### Синтаксис

```spxml
IsArray( arg )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `arg` | `any` | да | Проверяемое значение. |

#### Возвращает

`Bool`.

---

### ArrayCount()

Возвращает число элементов массива.

Для массивов прямого доступа функция работает быстро. Для сложных массивов, например результатов XQuery, вызов может повлечь обращение к серверу или полный перебор элементов.

#### Синтаксис

```spxml
ArrayCount( array )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Массив. |

#### Возвращает

`Integer`.

#### Пример

```spxml
elemsNum = ArrayCount( array );
```

---

### ArrayOptSize()

Возвращает размер массива, если он известен заранее. Если размер заранее неизвестен, возвращает `undefined`.

#### Синтаксис

```spxml
ArrayOptSize( array )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Массив. |

#### Возвращает

`Integer` или `undefined`.

#### Когда использовать

`ArrayOptSize()` удобна, когда нужно узнать размер массива без принудительного полного перебора или загрузки данных.

## 6. Прямой доступ, копирование и диапазоны

### ArrayDirect()

Преобразует массив к массиву с прямым индексированием.

Если исходный массив уже поддерживает прямое индексирование, возвращается сам исходный массив. Иначе функция работает аналогично `ArraySelectAll()` и возвращает стандартный массив `Array`, содержащий копию исходного массива.

#### Синтаксис

```spxml
ArrayDirect( array )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Исходный массив. |

#### Возвращает

`Array`.

#### Пример

```spxml
directArray = ArrayDirect( queryResult );
elem = directArray[0];
```

---

### ArraySelectAll()

Возвращает все элементы исходного массива в виде стандартного массива `Array`.

#### Синтаксис

```spxml
ArraySelectAll( array )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Исходный массив. |

#### Возвращает

`Array`.

#### Основные сценарии

1. Сложная или многократная обработка медленных массивов, например результатов XQuery.
2. Выборочное удаление элементов в цикле прохода по массиву.
3. Необходимость получить обычный массив со всеми элементами.

---

### ArrayRange()

Возвращает фрагмент массива с заданной позиции. Результат возвращается в виде стандартного массива.

Скалярные значения копируются. Для объектов копируются ссылки на те же объекты.

#### Синтаксис

```spxml
ArrayRange( array, startPos, elemsNum )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Исходный массив. |
| `startPos` | `integer` | да | Начальная позиция. |
| `elemsNum` | `integer` | да | Количество элементов. |

#### Возвращает

`Array`.

#### Особенности производительности

Эффективность зависит от типа исходного массива.

Для массивов в памяти функция быстро копирует фрагмент. Для абстрактных перечислителей с отложенной загрузкой данных функция выполняет быструю перемотку к нужной позиции, затем считывает нужное количество элементов.

`Any`.

## 8. Поиск и выборка по ключу

### ArrayOptFindByKey()

Ищет первый элемент массива с заданным значением ключевого поля.

Если элемент не найден, возвращает `undefined`.

#### Синтаксис

```spxml
ArrayOptFindByKey( array, keyValue )
ArrayOptFindByKey( array, keyValue, keyName )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Массив. |
| `keyValue` | `string` | да | Значение ключа. |
| `keyName` | `string` | нет | Имя ключевого элемента. Если не указано, используется первичный ключ. |

#### Возвращает

`Any` или `undefined`.

---

### ArrayOptFindBySortedKey()

Ищет первый элемент массива с заданным значением ключевого поля.

Если элемент не найден, возвращает `undefined`.

Массив должен быть предварительно отсортирован по ключевому полю по возрастанию. Это повышает скорость поиска по сравнению с `ArrayOptFindByKey()`.

#### Синтаксис

```spxml
ArrayOptFindBySortedKey( array, keyValue )
ArrayOptFindBySortedKey( array, keyValue, keyName )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Массив. |
| `keyValue` | `string` | да | Значение ключа. |
| `keyName` | `string` | нет | Имя ключевого элемента. Если не указано, используется первичный ключ. |

#### Возвращает

`Any` или `undefined`.

#### Когда использовать

Функция полезна для частого поиска в фиксированном справочнике большого размера, если этот справочник заранее отсортирован.

---

### ArraySelectByKey()

Выбирает элементы массива с заданным значением ключевого поля.

Функция аналогична `ArraySelect()`, но работает быстрее для поиска по ключу.

#### Синтаксис

```spxml
ArraySelectByKey( array, keyValue )
ArraySelectByKey( array, keyValue, keyName )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Массив. |
| `keyValue` | `string` | да | Значение ключа. |
| `keyName` | `string` | нет | Имя ключевого элемента. Если не указано, используется первичный ключ. |

#### Возвращает

`Array`.

---

### ArraySelectBySortedKey()

Выбирает элементы массива с заданным значением ключевого поля.

Массив должен быть предварительно отсортирован по возрастанию значения ключа. За счет этого функция существенно эффективнее `ArraySelectByKey()`.

Порядок сортировки возвращаемых элементов не определен.

#### Синтаксис

```spxml
ArraySelectBySortedKey( array, keyValue, keyName )
ArraySelectBySortedKey( array, keyValue, keyName, ignoreCase )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Массив, поддерживающий прямой доступ по индексу. |
| `keyValue` | `any` | да | Значение ключа. |
| `keyName` | `string` | да | Имя ключевого элемента. |
| `ignoreCase` | `bool` | нет | Игнорировать регистр для строковых ключей. |

#### Возвращает

`Array`.

---

### ArraySelectBySortedKeyPrefix()

Выбирает элементы массива, у которых значение ключевого поля начинается с заданного префикса.

Массив должен быть предварительно отсортирован по возрастанию значения ключа. Это делает функцию существенно эффективнее простого перебора.

Порядок сортировки возвращаемых элементов не определен.

#### Синтаксис

```spxml
ArraySelectBySortedKeyPrefix( array, prefixStr, keyName )
ArraySelectBySortedKeyPrefix( array, prefixStr, keyName, ignoreCase )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Массив, поддерживающий прямой доступ по индексу. |
| `prefixStr` | `string` | да | Префикс. |
| `keyName` | `string` | да | Имя ключевого элемента. |
| `ignoreCase` | `bool` | нет | Игнорировать регистр для строковых ключей. |

#### Возвращает

`Array`.

## 9. Фильтрация массива

### ArraySelect()

Выбирает элементы массива, удовлетворяющие заданному критерию.

#### Синтаксис

```spxml
ArraySelect( array, qualExpr )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Массив. |
| `qualExpr` | `string` | да | Выражение, определяющее соответствие элемента критерию. Вычисляется относительно элемента массива. |

#### Возвращает

`Array`.

#### Пример

```spxml
activeElems = ArraySelect( array, 'is_active == true' );
```

#### Примечание

В исходном тексте аргумент указан как `qulExpr`; по смыслу используется `qualExpr`.

---

### ArraySelectHierSubset()

Производит выборку иерархического подмножества из массива, содержащего элементы `id` и `parent_id`.

Выборка выполняется по заданному `parent_id`, не включая головной элемент.

#### Синтаксис

```spxml
ArraySelectHierSubset( array, keyVal, parentKeyName )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Массив. |
| `keyVal` | `any` | да | Значение родительского элемента. |
| `parentKeyName` | `string` | да | Имя поля родительского элемента. Для каталогов обычно `'parent_id'`. |

#### Возвращает

`Array`.

#### Типовой сценарий

Функция полезна для работы с каталогами и справочниками, где элементы связаны через `parent_id`.

## 10. Извлечение значений из массива

### ArrayExtract()

Выбирает определенное значение из каждого элемента массива.

Возвращает новый массив той же длины, содержащий выбранные значения.

#### Синтаксис

```spxml
ArrayExtract( array, fieldExpr )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Исходный массив. |
| `fieldExpr` | `string` | да | Выражение, вычисляемое относительно каждого элемента исходного массива. |

#### Возвращает

`Array`.

#### Пример

```spxml
names = ArrayExtract( array, 'name' );
```

---

### ArrayExtractKeys()

Выбирает определенное поле или атрибут из каждого элемента массива.

Функция аналогична более универсальной `ArrayExtract()`, но работает быстрее.

#### Синтаксис

```spxml
ArrayExtractKeys( array, field )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Исходный массив. |
| `field` | `string` | да | Имя поля. |

#### Возвращает

`Array`.

#### Пример

```spxml
ids = ArrayExtractKeys( array, 'id' );
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Массив. |
| `elemExpr` | `string` | да | Выражение, вычисляющее значение поля относительно элемента массива. |

#### Возвращает

`Any` или `undefined`.

#### Примечание

В исходном тексте описание `ArrayOptMin()` повторяется два раза. В этом файле дубль удален.

---

### ArrayMax()

Возвращает элемент массива, содержащий максимальное значение определенного поля.

Если массив пустой, функция завершается с исключением.

#### Синтаксис

```spxml
ArrayMax( array, elemExpr )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Массив. |
| `elemExpr` | `string` | да | Выражение, вычисляющее значение поля относительно элемента массива. |

#### Возвращает

`Any`.

#### Примечание

Функция присутствует в общем списке методов, но в предоставленном фрагменте отдельное подробное описание `ArrayMax()` отсутствует. Описание восстановлено по аналогии с `ArrayMin()` и `ArrayOptMax()`.

## 13. Объединение, пересечение и раскрытие массивов

### ArrayUnion()

Последовательно объединяет несколько массивов в один.

#### Синтаксис

|---|---:|---:|---|

| `...` | `object` | нет | Дополнительные массивы. |

#### Возвращает

### ArrayIntersect()

Возвращает массив, содержащий элементы первого массива, у которых значение ключевого поля совпадает хотя бы с одним элементом второго массива.

#### Синтаксис

```spxml
ArrayIntersect( array1, array2 )
ArrayIntersect( array1, array2, fieldExpr1 )
ArrayIntersect( array1, array2, fieldExpr1, fieldExpr2 )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array1` | `object` | да | Первый массив. |
| `array2` | `object` | да | Второй массив. |
| `fieldExpr1` | `string` | нет | Выражение, вычисляемое относительно элемента первого массива и возвращающее ключ сравнения. Если не указано, используется сам элемент `This`. |
| `fieldExpr2` | `string` | нет | Выражение, вычисляемое относительно элемента второго массива и возвращающее ключ сравнения. Если не указано, используется сам элемент `This`. |

#### Возвращает

`Array`.

#### Особенности результата

Порядок элементов в возвращаемом массиве не определен.

Если в первом массиве есть элементы с одинаковым значением ключа, попадание всех таких элементов в результат не гарантируется. Гарантированно в результат попадут элементы с дублирующимся ключом в количестве, в каком элементы с этим же ключом присутствуют во втором массиве.

## 14. Сортировка

### ArraySort()

Сортирует массив по заданным полям. Возвращает новый массив отсортированных значений.

Функция требует нечетного числа аргументов, не менее трех. Для каждого нового уровня сортировки добавляется два аргумента: выражение поля и направление сортировки.

#### Синтаксис

```spxml
ArraySort( array, elemExpr1, order1 )
ArraySort( array, elemExpr1, order1, elemExpr2, order2 )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---:|---:|---|
| `array` | `object` | да | Массив. |
| `elemExprN` | `string` | да | Выражение, вычисляющее значение поля для сортировки. |
| `orderN` | `string` | нет | Направление сортировки: `'+'` или `'-'`. |

#### Возвращает

`Array`.

#### Пример

```spxml
sortedArray = ArraySort( array, 'name', '+', 'date', '-' );
```

## 15. Исключение или `undefined`

Во многих функциях есть парные варианты:

| Вариант с исключением | Безопасный вариант | Поведение при отсутствии результата |
|---|---|---|
| `ArrayFirstElem()` | `ArrayOptFirstElem()` | `ArrayFirstElem()` выбрасывает исключение, `ArrayOptFirstElem()` возвращает `undefined`. |
| `ArrayFind()` | `ArrayOptFind()` | `ArrayFind()` выбрасывает исключение, `ArrayOptFind()` возвращает `undefined`. |
| `ArrayMin()` | `ArrayOptMin()` | `ArrayMin()` выбрасывает исключение на пустом массиве, `ArrayOptMin()` возвращает `undefined`. |
| `ArrayMax()` | `ArrayOptMax()` | `ArrayMax()` выбрасывает исключение на пустом массиве, `ArrayOptMax()` возвращает `undefined`. |

Практическое правило:

- если отсутствие элемента является ошибкой бизнес-логики — можно использовать вариант без `Opt`;
- если отсутствие элемента допустимо — лучше использовать `Opt`-вариант и явно проверять `undefined`.

## 16. Обычный поиск vs поиск по отсортированному ключу

Функции с `SortedKey` требуют, чтобы массив был заранее отсортирован по ключу по возрастанию.

| Обычная функция | Функция для отсортированного массива | Когда использовать |
|---|---|---|
| `ArrayOptFindByKey()` | `ArrayOptFindBySortedKey()` | Когда нужен один элемент по ключу. |
| `ArraySelectByKey()` | `ArraySelectBySortedKey()` | Когда нужна выборка всех элементов по ключу. |
| Перебор / `ArraySelect()` | `ArraySelectBySortedKeyPrefix()` | Когда нужна выборка по префиксу ключа. |

Перед использованием `SortedKey`-функций нужно гарантировать:

1. массив поддерживает прямой доступ;
2. массив отсортирован по нужному ключу;
3. сортировка выполнена по возрастанию;
4. ключ действительно соответствует полю `keyName`.

## 17. Практические рекомендации

### Для результатов XQuery

Если массив получен из XQuery и будет использоваться несколько раз:

```spxml
array = ArraySelectAll( queryResult );
```

или:

```spxml
array = ArrayDirect( queryResult );
```

Это позволит избежать повторного перечисления и даст прямой доступ к элементам.

### Для поиска необязательного элемента

```spxml
elem = ArrayOptFind( array, 'id == 123' );

if ( elem != undefined )
{
    // элемент найден
}
```

### Для обязательного элемента

```spxml
elem = ArrayFind( array, 'id == 123' );
```

Если элемент не найден, выполнение завершится исключением.

### Для справочников большого размера

Если нужно часто искать элементы по ключу:

```spxml
sortedArray = ArraySort( array, 'id', '+' );
elem = ArrayOptFindBySortedKey( sortedArray, 123, 'id' );
```

### Для получения списка значений поля

```spxml
ids = ArrayExtractKeys( array, 'id' );
```

### Для получения уникальных значений

```spxml
uniqueIds = ArraySelectDistinctKeys( array, 'id' );
```

### Для суммы по полю

```spxml
total = ArraySum( array, 'amount' );
```

## 19. Спорные и уточняемые места источника

В исходном материале есть несколько мест, которые стоит проверить при последующей сверке с официальной документацией или практическим тестом:

1. `ArrayMax()` присутствует в общем списке функций, но подробный блок описания в предоставленном фрагменте отсутствует.
2. `ArrayOptMin()` продублирована два раза.
3. В описании `ArraySelect()` аргумент указан как `qulExpr`; вероятно, имеется в виду `qualExpr`.
4. Для `ArraySort()` сказано, что `orderN` необязателен, но одновременно указано, что функция требует нечетного числа аргументов и минимум три аргумента. Нужно проверить допустим ли вызов без направления сортировки.
5. Для `ArrayExpand()` название аргумента в синтаксисе — `elemCode`, а в описании встречается `elemCode` как выражение поля с подмассивом.

## 20. Короткое резюме

Раздел работы с массивами в SP-XML важен тем, что массивы могут быть не только обычными `Array`, но и абстрактными перечислителями, результатами XQuery или другими объектами с последовательным доступом.

Главные практические правила:

- для безопасного прямого индексирования используйте `ArrayDirect()`;
- для полной материализации массива используйте `ArraySelectAll()`;
- для допустимого отсутствия результата выбирайте `Opt`-функции;
- для частого поиска по справочникам используйте предварительную сортировку и функции `SortedKey`;
- для больших или ленивых массивов осторожно используйте `ArrayCount()`, так как он может быть дорогим.
