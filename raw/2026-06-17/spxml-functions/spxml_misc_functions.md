## ModalDlg()

Вызывает модальный диалог.

### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `arg1` | Object SpXml | Документ, в котором будут сохранены результаты модального диалога. |

### Пример

```js

```

## AppDataDirectoryPath()

Возвращает путь к базовой директории хранения данных приложения (серверной или клиентской части).

По умолчанию совпадает с установочной директорией, если иное не задано в `app_config.xml`.

## DebugMsg()

Модификация функции `alert()`, предназначенная для целей отладки.

В отличие от последней, `DebugMsg()` может быть оставлена в релизном коде.

Если в `SpXml.ini` указан параметр `SHOW-DEBUG-MSG`, функция работает аналогично `alert()`.

В остальных случаях она осуществляет запись значения в журнал по умолчанию.

Функция возвращает переданное значение.

### Синтаксис

```js
DebugMsg( value )
```

### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `value` | Any | Произвольное значение. |

## EncodeJson()

Преобразует базовый объект, массив, либо скалярное значение в строку в формате JSON.

Функция работает аналогично функции `JSON.stringify()` в JavaScript, однако второй аргумент представляет из себя набор опций.

### Синтаксис

```js
EncodeJson( value )
EncodeJson( value, options )
```

### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `value` | Any | Значение. |
| `options` | object optional | Базовый объект, содержащий набор опций. |

### Описание

Поддерживаются следующие опции:

| Опция | Тип | Описание |
|---|---|---|
| `PrettyFormat` | bool | Форматирование с отступами и переводами строк. |
| `ExportLargeIntegersAsStrings` | bool | Экспортировать целые числа, значение которых больше `0x7FFFFFFF`, в виде строковой константы в кавычках. |

### Пример

```js
EncodeJson( respObj, {PrettyFormat:true} )
```

## ParseJson()

Преобразует строку, содержащую JSON, в базовый объект, массив, либо скалярное значение.

Функция работает аналогично функции `JSON.parse()` в JavaScript, однако второй аргумент представляет из себя набор опций.

Также `ParseJson()` менее требовательна к формату, и по умолчанию обрабатывает строки не только в формате JSON, но и JavaScript literal, разрешая в том числе любые типы кавычек, а также значения `undefined`.

### Синтаксис

```js
ParseJson( str )
ParseJson( str, options )
```

### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `str` | string | Строка. |
| `options` | object optional | Базовый объект, содержащий набор опций. |

### Описание

Поддерживаются следующие опции:

| Опция | Тип | Описание |
|---|---|---|
| `StrictMode` | bool | Требовать строгого соблюдения стандарта JSON. |

## EnableLogExt()

Включает заданный журнал.

В отличие от функции `EnableLog()` позволяет задать дополнительные опции ведения журнала.

### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `name` | String | Название журнала. |
| `options` | String optional | Набор опций вида `param1=value1;param2=value2;...`. |

### Результат

`undefined`

### Пример

```js

```

### Возможные опции

| Опция | Тип | Описание |
|---|---|---|
| `life-time` | String | Период, на который заводится новый файл журнала: `day`, `month`, `permanent`. По умолчанию свой файл журнала заводится на каждую дату (`day`). |
| `base-dir` | String | Директория, в которой будут заводиться журнал. По умолчанию используется директория `Logs` в установочной директории. |
| `use-std-event-prefix` | Bool | Включать в начало каждой строки журнала стандартный префикс: дата, время, ID потока. По умолчанию `true`. |
| `header-str` | String | Строка заголовка, добавляемая в начало каждого нового файла журнала. По умолчанию строка заголовка не добавляется. |

### Результат

`Any`

## CheckCurThread()

Проверяет, не должен ли текущий поток завершиться, обычно после прерывания его пользователем.

Если да, вызывает исключение аналогично функции `Cancel()`.

### Результат

`undefined`

Поскольку поток, как правило, нельзя завершить принудительно, любая задача, выполняющая длинные вычисления, должна периодически вызывать эту функцию, чтобы проверить, не было ли выполнение потока отменено пользователем.

Если поток вызывает в цикле функцию `Sleep()` либо любую функцию, вызывающую сетевой запрос, то вызывать функцию `CheckCurThread()` нет необходимости, поскольку вышеназванные функции сами проверяют запрос на отмену выполнения потока.

## HttpRequest()

Выполняет HTTP-запрос.

Возвращает объект `HttpResponse`.

### Синтаксис

```js
HttpRequest( url )
HttpRequest( url, method )
HttpRequest( url, method, body )
HttpRequest( url, method, body, options )
```

### Аргументы

| Аргумент | Тип | Описание |
|---|---|---|
| `url` | string | Запрашиваемый URL. |
| `method` | string optional | Метод: `get`, `post` и др. По умолчанию `get`. |
| `body` | string optional | Тело запроса в виде строки либо объекта `Binary`. Если передано значение `undefined`, то для совместимости с устаревшей нотацией следующий параметр `options` должен быть обязательно указан. |
| `options` | string optional | Список опций и дополнительных полей заголовка HTTP-запроса в формате INI: имя, значение, перевод строки. Известные поля интерпретируются как опции. Остальные добавляются как заголовки запроса. |

### Описание

Среди списка дополнительных полей заголовка возможно использование следующих опций, которые обрабатываются отдельно и не попадают в передаваемый заголовок:

| Опция | Описание |
|---|---|
| `Content-Type` | Поле. |
| `Ignore-Errors` | Игнорировать наличие кода ошибки HTTP в ответе. Если указана эта опция, код ошибки можно получить через атрибут `RespCode` возвращаемого объекта. По умолчанию функция завершается с ошибкой в случае получения кода ошибки по HTTP. |
| `Auto-Redirect` | Автоматически следовать редиректам HTTP 303, HTTP 304. По умолчанию `true`. |
| `Ask-Auth` | Показывать окно авторизации при получении результата HTTP 401. По умолчанию `true` на клиентской части приложения. |
| `AuthLogin` | Логин для авторизации. |
| `AuthPassword` | Пароль для авторизации. |
| `IgnoreStoredCookies` | Игнорировать сохраненные ранее куки. |
| `CertFilePath` | Путь к файлу с клиентским сертификатом для авторизации по TLS-сертификату. |
| `KeyFilePath` | Путь к файлу с закрытым ключом для авторизации по TLS-сертификату. |
| `CaCertFilePath` | Путь к файлу с CA серверного сертификата для проверки серверного сертификата. |

### Пример

```js
HttpRequest( 'http://reg.datex-soft.com/' )
HttpRequest( 'http://reg.datex-soft.com/login.htm', 'post', UrlEncodeQuery( {login:'xxx',password:'xxx'} ) )
HttpRequest( 'http://reg.datex-soft.com/login.htm', 'post', '<xxx>111</xxx>', 'Content-type: text/xml\nIgnore-Errors: 1\nAskAuth: 0' )
```

## UserError()

Создает объект типа `Error`, содержащий словесное описание ошибки, пригодное для восприятия обычным пользователем.

Такая ошибка в пользовательском интерфейсе будет показана с изначально скрытой отладочной информацией, но которую можно будет раскрыть.

| `desc` | string | Текст ошибки. |
| `innerError` | optional | Исходная ошибка, вызвавшая данную ошибку. |

### Пример

```js
try
{
    CallXxxService();
}
catch ( e )
{
    throw UserError( 'Unable to call registration service', e );
}
```
