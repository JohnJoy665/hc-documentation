# SP-XML: Работа со строками

## Назначение файла

Файл описывает встроенные функции SP-XML для работы со строками: проверка строк, поиск подстрок, получение фрагментов, замена, разбиение, преобразование регистра, работа с символами и кодами символов.

Материал подготовлен как AI-friendly справочник для дальнейшего использования в локальной документации WebTutor/WebSoft HCM.

## Важные замечания

### Байты и символы

В SP-XML часть строковых функций работает с позициями и длинами **в байтах**, а часть — **в символах**.

Это особенно важно для строк в UTF-8:

- `StrLen()` возвращает длину строки **в байтах**.
- `StrCharCount()` возвращает количество **символов**.
- `StrLeftRange()` возвращает фрагмент по длине **в байтах**.
- `StrLeftCharRange()` возвращает фрагмент по длине **в символах**.
- `StrOptSubStrPos()`, `StrOptSubStrPosB()`, `StrOptSubStrRightPos()` возвращают позиции **в байтах**.
- `StrCharRangePos()` работает с позициями **в символах**.

Для строк, содержащих не ASCII-символы, нельзя бездумно смешивать байтовые позиции и символьные позиции. Если нужна заранее известная длина в символах, безопаснее использовать функции с `Char` в названии.

### Дубли и спорные места источника

В исходном материале функция `StrEqual()` была продублирована. В этом файле она оставлена один раз.

Также в исходном материале есть несколько мест, которые стоит дополнительно проверить по официальной документации или в тестовом стенде:

- `StrCharRangePos()` описана как функция, возвращающая фрагмент строки, но в синтаксисе указаны только `startPos` и `endPos`, без аргумента строки.
- `StrSplit2()` описана как функция разбиения по заданному разделителю, но в приведённом синтаксисе аргумент разделителя не указан.
- `StrReplaceOne()` описана как замена первого вхождения, но в исходном описании перечислены только два аргумента.
- `StrUpperCase()` по названию должна переводить строку в верхний регистр, однако в исходном тексте указано: "Преобразует строку в нижний регистр". Это похоже на ошибку источника.

# Проверки строк

## StrEqual()

Проверяет две строки на равенство. Может выполнять сравнение без учёта регистра.

### Синтаксис

```spxml
StrEqual( str1, str2 )
StrEqual( str1, str2, ignoreCase )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str1` | `string` | да | Первая строка. |
| `str2` | `string` | да | Вторая строка. |
| `ignoreCase` | `bool` | нет | Если `true`, регистр символов игнорируется. |

### Результат

`Bool`.

### Пример

```spxml
StrEqual( 'Test', 'test', true )
```

# Длина строки и получение фрагментов

## StrLen()

Возвращает длину строки в байтах.

Длина в байтах может быть больше количества символов, если приложение использует UTF-8. Для определения количества символов нужно использовать `StrCharCount()`.

### Синтаксис

```spxml
StrLen( str )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `String` | да | Строка. |

### Результат

`Integer`.

## StrCharCount()

Возвращает количество символов в строке.

### Синтаксис

```spxml
StrCharCount( str )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `String` | да | Строка. |

### Результат

`Integer`.

## StrLeftRange()

Возвращает часть строки `str` длиной `length` в байтах, начиная с первого символа строки.

Если второй аргумент больше длины переданной строки, возвращается исходная строка.

### Синтаксис

```spxml
StrLeftRange( str, length )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `string` | да | Строка. |
| `length` | `integer` | да | Длина фрагмента в байтах. |

### Результат

`String`.

### Особенность UTF-8

Так как строки хранятся в UTF-8, для строк с не ASCII-символами использование `StrLeftRange()` безопасно только в случаях, когда `length` получен в результате поиска в строке. Для получения фрагмента заранее известной длины нужно использовать `StrLeftCharRange()`.

## StrLeftCharRange()

Возвращает часть строки `str` длиной `length` в символах, начиная с первого символа строки.

Если второй аргумент больше длины переданной строки, возвращается исходная строка.

### Синтаксис

```spxml

```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |

| `str` | `string` | да | Строка. |
| `length` | `integer` | да | Длина фрагмента в символах. |

### Результат

`String`.

# Поиск подстрок

## StrOptSubStrPos()

Возвращает позицию подстроки внутри строки. Позиция возвращается в байтах. Если подстрока не найдена, возвращает `undefined`.

### Синтаксис

```spxml
StrOptSubStrPos( str, subStr )
StrOptSubStrPos( str, subStr, ignoreCase )
StrOptSubStrPos( str, subStr, ignoreCase, startPos )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `string` | да | Строка. |
| `subStr` | `string` | да | Подстрока. |
| `ignoreCase` | `bool` | нет | Без учёта регистра. |
| `startPos` | `integer` | нет | Позиция в байтах, с которой должен начинаться поиск. |

### Результат

`Integer` или `undefined`.

## StrOptSubStrPosB()

Возвращает позицию подстроки внутри строки в бинарном режиме поиска. Строки могут содержать любые данные, включая символ `\0`. Если подстрока не найдена, возвращает `undefined`.

### Синтаксис

```spxml
StrOptSubStrPosB( str, subStr )
StrOptSubStrPosB( str, subStr, ignoreCase )
StrOptSubStrPosB( str, subStr, ignoreCase, startPos )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `string` | да | Строка. |
| `subStr` | `string` | да | Подстрока. |
| `ignoreCase` | `bool` | нет | Без учёта регистра. |
| `startPos` | `integer` | нет | Позиция в байтах, с которой должен начинаться поиск. |

### Результат

`Integer` или `undefined`.

`String`.

# Разбиение строк и работа со строками как с массивами

## StrSplit2()

Разбивает строку на две подстроки, используя заданный разделитель. Возвращает массив из двух подстрок. Если разделитель отсутствует, возвращает массив из исходной строки и пустой строки.

### Синтаксис из источника

```spxml
StrSplit2( str )
StrSplit2( str, useQuotes )
```

### Аргументы из источника

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `String` | да | Строка. |
| `useQuotes` | `bool` | нет | Если `true`, и значение слева или справа от разделителя обёрнуто в кавычки, кавычки удаляются из результата. |

### Результат

`Array` из двух строк.

### Примечание

В описании упоминается заданный разделитель, но в исходном синтаксисе отдельный аргумент разделителя не указан. Это место нужно проверить.

## StrSplitToLines()

Разбивает строку на подстроки, используя разделители `LF` либо `CRLF`. Возвращает массив подстрок.

### Синтаксис

```spxml

StrSplitToLines( str, options )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `String` | да | Строка. |
| `options` | `object` | нет | Объект с набором опций вызова. |

### Поддерживаемые опции

| Опция | Тип | Описание |
|---|---|---|
| `LineFolding` | `bool` | Опция обработки строк. В исходном материале дополнительных деталей нет. |

### Результат

`Array` строк.

# Сканирование строки по шаблону

## StrScan()

Извлекает из строки фрагменты в соответствии с заданным шаблоном. Если строка не соответствует шаблону, возвращается ошибка.

### Синтаксис

```spxml
StrScan( str, arg )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `string` | да | Строка. |
| `arg` | `string` | да | Шаблон. |

### Элементы шаблона

| Элемент | Описание |
|---|---|
| `%s` | Вхождение подстроки, которое возвращается в результате. |
| `%*s` | Вхождение подстроки, которое используется для сопоставления, но не возвращается. |

### Результат

`Array` найденных фрагментов. Если строка не соответствует шаблону, возникает ошибка.

### Пример

```spxml
obj = StrScan( str, '%*s/vacancy/%s.htm' );
vacancyID = obj[0];
```

## StrOptScan()

Извлекает из строки фрагменты в соответствии с заданным шаблоном. Если строка не соответствует шаблону, возвращает `undefined`.

### Синтаксис

```spxml
StrOptScan( str, arg )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `string` | да | Строка. |
| `arg` | `string` | да | Шаблон. |

### Элементы шаблона

| Элемент | Описание |
|---|---|
| `%s` | Вхождение подстроки, которое возвращается в результате. |
| `%*s` | Вхождение подстроки, которое используется для сопоставления, но не возвращается. |

### Результат

`Array` найденных фрагментов или `undefined`.

### Пример

```spxml
obj = StrOptScan( str, '%*s/vacancy/%s.htm' );
if ( obj != undefined )
{
    vacancyID = obj[0];
}
```

### Отличие `StrScan()` от `StrOptScan()`

- `StrScan()` подходит, когда несоответствие строки шаблону считается ошибкой.
- `StrOptScan()` подходит, когда несоответствие строки шаблону является допустимым вариантом и должно обрабатываться через `undefined`.

# Практические рекомендации для разработки

## 1. Для пользовательских строк предпочитать символьные функции

Если строка может содержать кириллицу или другие UTF-8 символы, при отрезании фрагментов заранее известной длины безопаснее использовать:

```spxml
StrLeftCharRange( str, length )
StrCharCount( str )
```

А не:

```spxml
StrLeftRange( str, length )
StrLen( str )
```

## 2. Байтовые позиции использовать только осознанно

Функции поиска возвращают позиции в байтах. Эти позиции можно использовать с другими байтовыми функциями, но нельзя автоматически считать их номерами символов.

Пример связки, которую нужно применять аккуратно:

```spxml
pos = StrOptSubStrPos( str, subStr );
if ( pos != undefined )
{
    leftPart = StrLeftRange( str, pos );
}
```

## 3. Для необязательного парсинга использовать `StrOptScan()`

Если строка может не соответствовать шаблону, лучше использовать `StrOptScan()` и проверять результат на `undefined`.

```spxml
obj = StrOptScan( url, '%*s/vacancy/%s.htm' );
if ( obj != undefined )
{
    vacancyID = obj[0];
}
```

## 4. Для обязательного парсинга использовать `StrScan()`

Если несоответствие шаблону должно считаться ошибкой, можно использовать `StrScan()`.

```spxml
obj = StrScan( url, '%*s/vacancy/%s.htm' );
vacancyID = obj[0];
```

## 5. Для подготовки строк к сравнению использовать нормализацию

Перед сравнением пользовательского ввода часто полезно убрать лишние пробелы и привести строку к одному регистру.

```spxml
normalized = StrLowerCase( UnifySpaces( Trim( str ) ) );
```

# Мини-чеклист перед использованием функций

- Если нужна длина в байтах — `StrLen()`.
- Если нужно количество символов — `StrCharCount()`.
- Если нужно отрезать первые N символов — `StrLeftCharRange()`.
- Если нужно отрезать по позиции, найденной поиском — можно использовать байтовые функции.
- Если строка может содержать кириллицу — не смешивать байты и символы.
- Если поиск может ничего не найти — использовать функции с `Opt` и проверять `undefined`.
- Если шаблон должен совпадать строго — использовать `StrScan()`.
- Если шаблон может не совпасть — использовать `StrOptScan()`.
