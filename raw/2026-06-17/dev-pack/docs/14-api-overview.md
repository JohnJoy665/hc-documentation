# API Websoft HCM – обзор

Платформа Websoft HCM предоставляет набор API для интеграции с внешними системами.  API позволяет управлять объектами (курсы, пользователи, события), запускать процессы и получать данные.  Разработчики могут использовать API для построения собственных веб‑порталов, мобильных приложений или интеграции с системами HR, аналитики и документооборота.

## Версии API

На момент подготовки документации доступны две основных версии API:

* **API версия 1** – классический SOAP/XML интерфейс.  Поддерживает создание, чтение, обновление и удаление объектов посредством стандартных операций (Create, Get, Update, Delete).  Используется в старых интеграциях и совместим с WebTutor начиная с версии 2.7.  Подробнее см. файл `docs/15-api-v1.md`.
* **API версия 3** – современный OpenAPI/JSON интерфейс.  Соответствует спецификации OpenAPI 3.0 и поддерживает методы `GET`, `POST`, `PATCH`, `DELETE`【465071165735325†L124-L131】.  Данные передаются в формате `application/json`, авторизация возможна анонимно или через Basic/Token.  Подробности – в файле `docs/16-api-v3.md`.

Помимо перечисленных версий, существуют вспомогательные REST‑сервисы (например, сервисы мониторинга) и закрытые API, используемые самим порталом, однако они за рамками данного пакета.

## Низкоуровневый и высокоуровневый доступ

API версии 3 поддерживает два варианта использования:

* **Низкоуровневый (Low‑level)** – позволяет работать напрямую с объектами платформы (курсы, сотрудники, события).  В запросах указываются классы объектов и набор прав, определённых в профиле безопасности.  Такой способ не требует дополнительного программирования, но предоставляет широкий доступ и требует детальной настройки профиля.
* **Высокоуровневый (High‑level)** – ориентирован на вызов конкретных функций (методов), описанных в системе.  В этом случае разработчик описывает функции в библиотеке программного кода или использует встроенные функции, а затем публикует их через API.  Для каждой функции настраивается профиль доступа и подготавливается файл JSON с описанием методов【542364000200017†screenshot】.  Высокоуровневый вариант упрощает интеграцию и лучше контролирует доступ.

## Организация доступа

Доступ к API обеспечивается многоуровневой моделью безопасности【542364000200017†screenshot】:

1. **Регистрация приложения** – в модуле «Безопасность → API → Внешние приложения» создаётся приложение и ему присваивается уникальный идентификатор `APP‑ID`.  Этот идентификатор передаётся в заголовке `X‑App‑Id` каждого запроса.
2. **Учётные данные** – для приложения создаются учётные записи (логин/пароль) в модуле «Безопасность → API → Удостоверения».  Учётные данные позволяют аутентифицировать запросы по Basic‑Auth или получать токен.
3. **Профили безопасности** – в модуле «Безопасность → API → Профили безопасности» создаётся профиль с перечислением доступных классов объектов и функций.  Профиль может включать списки объектов (низкоуровневый доступ) и набор функций (высокоуровневый доступ).  Профиль сохраняется в файле `wtv\oapi\profiles` в формате XML【789053750046110†screenshot】.
4. **Публикация функций** – для высокоуровневого доступа разработчик формирует JSON‑файл описания функций (например, с помощью JSDoc) и публикует его через раздел «Формирование и публикация файла JSON».

## Типовое использование

* **Интеграция с HR‑системой** – получение списка сотрудников, назначение курсов, управление обучением.
* **Внешние порталы** – использование Websoft HCM в качестве back‑end сервиса: создание курсов, обработка заявок, формирование отчётов.
* **Мобильные приложения** – отображение каталога курсов и прохождение тестов через API.
