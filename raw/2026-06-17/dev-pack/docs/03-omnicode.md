# Редактор программного кода WebSoft OmniCode for HCM

OmniCode — это расширение для Visual Studio Code, предназначенное для разработки серверного кода на платформе WebSoft HCM. С его помощью можно создавать и редактировать серверные агенты, библиотеки программного кода, удалённые действия и другие объекты. OmniCode поставляется начиная с релиза 2023.2 и поддерживает C# и Server‑Side JavaScript.

## Установка OmniCode

1. **Требования.** Перед установкой убедитесь, что на сервере развернута актуальная версия WebSoft HCM. Рекомендуется работать на тестовом стенде, так как редактор может вносить изменения в код в реальном времени【931861937373518†screenshot】.
2. **Установка расширения.** OmniCode устанавливается как расширение Visual Studio Code. Для установки скачайте файл расширения с портала клиентов и установите через VS Code (меню *Extensions* → *Install from VSIX*). Права администратора необходимы.
3. **Подключение к серверу.** После установки создайте новое подключение, указав адрес сервера WebSoft HCM и учетные данные администратора. Рекомендуется работать в одиночном режиме, чтобы избежать одновременного изменения одних и тех же объектов несколькими разработчиками.

## Интерфейс OmniCode【399222508907901†screenshot】

Редактор состоит из трёх основных областей:

1. **Панель серверов** — список доступных серверов WebSoft HCM. Здесь можно добавить новое подключение, выбрать активный сервер и просматривать объекты.
2. **Область кода** — вкладки для редактирования файлов. Подсветка синтаксиса зависит от языка (C# или JavaScript). OmniCode позволяет открывать несколько объектов одновременно, искать по коду и создавать новые файлы.
3. **Консоль** — вывод сообщений от сервера, результатов выполнения скриптов и информации об ошибках. Используется при отладке и работе с интерактивными командами.

## Работа с объектами【981968999901797†screenshot】

OmniCode позволяет создавать и редактировать различные типы серверных объектов:

| Тип объекта | Описание |
|---|---|
| **Агенты** | Серверные процессы, выполняющие периодические задачи. |
| **Библиотеки программного кода** | Файлы `.js` или `.pl` с группой функций и процедур. |
| **Удалённые действия** | Команды, которые могут вызываться с клиентской стороны или из бизнес‑процесса. |
| **Выборки и показатели** | Запросы к данным и расчётные показатели, используемые в аналитике. |
| **Веб‑шаблоны** | Фрагменты HTML/XAML для отображения интерфейса портала. |

Для каждого типа объектов OmniCode предоставляет базовые шаблоны. Созданный объект сохраняется в соответствующем каталоге на сервере и сразу доступен для использования.

## Логи и отладка【365014394326919†screenshot】

Все события, связанные с работой редактора, записываются в папку `.../WebSoftServer/Logs`. В случае ошибок или нестандартного поведения рекомендуется:

- включить режим отладки, установив в конфигурационном файле параметр `DOTNETCORE-DEBUG: 1`;
- проверять сообщения консоли OmniCode — здесь выводятся результаты выполнения скриптов и предупреждения;
- просматривать файл `omnicode.log` в каталоге логов для получения подробных сведений о выполнении команд.

## Работа с C#

OmniCode поддерживает написание кода на языке C#. Разработчик может создавать объекты (удалённые действия, выборки, агенты) как классы, подключать пространства имён и сторонние сборки. В редакторе доступна подсветка синтаксиса, автодополнение и отладка с установкой точек останова. Пример создания удалённого действия:

```csharp
using System;
using WebSoft.Common;

public class MyAction : IWebTutorAction
{
    public object Execute(object context)
    {
        // бизнес‑логика
        return "Completed";
    }
}
```

## Работа с Server‑Side JavaScript

Помимо C#, редактор поддерживает серверный JavaScript — основной язык для бизнес‑логики в WebSoft HCM. Можно создавать скрипты для агентов, удалённых действий и библиотек. Пример функции JavaScript:

```javascript
function sendEmail(userId, subject, body) {
    var user = tools.get_user(userId);
    tools.send_notification(user, subject, body);
}
```

Код JavaScript выполняется на сервере, поэтому имеет доступ ко всем встроенным объектам и функциям, описанным в справке. Отладку скриптов можно вести через консоль OmniCode.

## Рекомендации

- Разрабатывайте и тестируйте код на выделенном стенде, а затем переносите на продуктивную систему.
- Используйте систему контроля версий (например, Git) для хранения исходников объектов.
- Разделяйте код на функции и модули, чтобы облегчить сопровождение. Библиотеки программного кода помогают группировать общий функционал.

OmniCode является основным инструментом разработки для WebSoft HCM и позволяет объединить работу с серверным кодом, настройками и отладкой в одной среде.
