# Интерфейс системы Websoft HCM

Пользовательский интерфейс Websoft HCM основан на XAML‑разметке, которая описывает элементы экранных форм, диалогов, кнопок и других компонентов.  Разработчики могут создавать и редактировать интерфейсные формы с помощью встроенного редактора, а также подключать собственные элементы, написанные на JavaScript или C#.

## Основные сведения

* **XAML‑разметка** — текстовый язык разметки, подобный XML, используемый для описания экранных форм.  Каждый элемент имеет имя (например, `Action`, `Button`, `Grid`) и набор атрибутов, определяющих его параметры и обработчики событий.  Список элементов приведён в разделе «XAML‑справочник» (см. файл `docs/13-xaml-reference.md`).
* **Screen Forms (.xms)** — экранные формы описывают пользовательские интерфейсы, отображаемые в клиенте.  Документация по файлам `.xms` находится в внешнем руководстве SP‑XML; в модуле Websoft они используются для настройки интерфейса и бизнес‑процессов【567339561225027†screenshot】.
* **Data Forms (.xmd)** — формы описания данных определяют структуру объектов и поля для ввода информации.  Как и экранные формы, подробное описание находится в справке SP‑XML【420292193409555†screenshot】.
* **Интерфейс OmniCode** — для разработки серверного кода и шаблонов используется плагин OmniCode for HCM, предоставляющий интеграцию с VS Code (см. файл `docs/03-omnicode.md`).  OmniCode позволяет создавать объекты (агенты, библиотеки, удалённые действия), просматривать каталоги сервера и вести отладку кода.

## Разработка интерфейса

Разработчик может создавать и изменять XAML‑формы через встроенный редактор форм или с помощью OmniCode.  Для каждого элемента доступны свойства (атрибуты) и события.  Например, элемент `Action` определяет удалённое действие, которое выполняется при событии клика на кнопке или при загрузке формы.  В документации к `Action` указано, что атрибут `Name` задаёт имя действия, `Source` — ссылку на объект, а `Success` и `Failure` позволяют обработать результат выполнения【575597786908733†screenshot】.

При создании пользовательских форм следуйте принципам:

1. **Разделяйте логику и представление**.  Бизнес‑логика должна содержаться в серверном коде (агентах, библиотеках), а XAML‑формы – только описывать интерфейс.
2. **Используйте стандартные элементы**.  Платформа предоставляет широкий набор элементов: `Button`, `Label`, `Grid`, `ListBox`, `Menu`, `StackPanel`, и др.  Список приведён в XAML‑справочнике.
3. **Соблюдайте единообразие**.  Следуйте корпоративным стандартам по цветам, отступам и размерам элементов, чтобы интерфейс был удобен для пользователя.

## Навигация и меню

Портал Websoft HCM использует древовидное меню для навигации по разделам (видно в левой части портала).  При разработке приложений вы можете программно управлять доступностью пунктов меню, используя API и функции библиотек.

Дополнительные сведения об элементах интерфейса и примерах XAML‑разметки приведены в файле `docs/13-xaml-reference.md`.
