# Рекомендации по функциональному дизайну

Раздел «Рекомендации по функциональному дизайну выборок, удалённых действий и показателей» содержит рекомендации по разработке пользовательских выборок (report), удалённых действий (remote actions) и KPI/показателей в Websoft HCM.  Полный текст находится в базе знаний и не был полностью обработан в рамках данного пакета.  Ниже приведены основные принципы, которые следует учитывать при проектировании этих элементов.

## Выборки (Reports)

* **Оптимальность запросов** – старайтесь минимизировать количество XQuery и SQL‑операций, выбирая только необходимые поля и объекты.  Избегайте вложенных циклов и неоптимальных условий.
* **Пользовательские параметры** – используйте параметры, чтобы позволить пользователю фильтровать данные по дате, категории, сотруднику и т. п.
* **Кэширование результатов** – для тяжёлых выборок можно настроить кэширование результатов (см. файл `docs/19-caching.md`).

## Удалённые действия (Remote actions)

* **Бизнес‑логика на сервере** – все расчёты и проверка прав должны выполняться в серверном коде (агенты, библиотеки).  Клиентская часть должна только инициировать действие.
* **Обработка исключений** – предусматривать обработку ошибок и отправку пользователю понятных сообщений.
* **Минимальный набор параметров** – передавайте только необходимые данные.  Используйте сериализацию JSON для передачи сложных структур.

## Показатели и KPI

* **Единая метрика** – для каждого показателя определяйте чёткую метрику и единицу измерения.
* **Динамическое обновление** – показатели должны обновляться автоматически при изменении данных (используйте агенты и триггеры).
* **Визуализация** – используйте диаграммы, прогресс-бары и индикаторы для наглядного отображения результатов.

> Важно: приведённые принципы являются общими и не заменяют собой полную инструкцию.  Для получения подробного руководства обратитесь к оригинальной статье «Рекомендации по функциональному дизайну…» в базе знаний.
