# XAML‑справочник элементов Websoft HCM

Этот документ представляет краткий справочник по элементам XAML, используемым в интерфейсе Websoft HCM.  Каждый элемент задаёт определённый тип визуального компонента, а атрибуты определяют его поведение и внешний вид.  Полная документация доступна на портале Websoft HCM; здесь приведены лишь основные сведения.

## Структура справочника

Раздел «Интерфейс системы → Описание XAML разметки» в базе знаний содержит список элементов, который включает:

* **Action** – определяет удалённое действие, которое может быть вызвано по событию.  Параметры: `Name` (имя вызываемой функции), `Source` (объект, содержащий код), `Success`/`Failure` (обработчики результата)【575597786908733†screenshot】.
* **Button** – кнопка, генерирующая событие при нажатии.  Атрибуты: `Caption` (подпись), `Action` (вызываемое действие), `Style` (стиль оформления).
* **Label** – текстовая метка, предназначенная для отображения статичных или динамических строк.
* **Grid** – контейнер для размещения элементов в виде таблицы.  Поддерживает настройку строк и столбцов.
* **ListBox** – список значений, позволяющий выбирать один или несколько элементов.
* **Menu** – выпадающее меню, содержащее пункты (элементы `MenuItem`) и действия.
* **Panel** – универсальный контейнер для группировки элементов; может быть горизонтальным, вертикальным или свободно плавающим.
* **StackPanel** – упрощённый контейнер, который размещает элементы в один столбец (вертикально) или строку (горизонтально).
* **TextBox** – поле ввода текста для пользователя.
* **Image** – отображение изображения из ресурсов или внешнего URL.
* **BackgroundImage** – настройка фонового изображения для формы или элемента【449033167658910†screenshot】.
* **CheckBox**, **RadioButton**, **ComboBox** – стандартные элементы выбора.

Каждый элемент может содержать дочерние элементы и поддерживает обработку событий (например, `OnClick`, `OnChange`).  При проектировании сложных форм рекомендуется использовать контейнеры (`Grid`, `Panel`, `StackPanel`) для управления компоновкой.

> Важно: элементы XAML внутри Websoft HCM являются частью SP‑XML и могут иметь отличия от стандартного XAML из мира .NET/WPF.  Используйте справочник Websoft HCM для уточнения поддерживаемых атрибутов и событий.

## Пример разметки с элементом Action

```xml
<SPXMLScreen>
    <Action Name="SendReport" Source="libAnalytics" Success="OnSuccess" Failure="OnError" />
    <Button Caption="Отправить отчёт" Action="SendReport" />
    <Label Caption="Нажмите кнопку для отправки отчёта" />
</SPXMLScreen>
```

В примере описана простая экранная форма, содержащая действие `SendReport`, кнопку и подпись.  При нажатии на кнопку выполняется метод `SendReport` из библиотеки `libAnalytics`; результат обрабатывается в событиях `OnSuccess` и `OnError`.

Подробный список элементов можно найти в справке «Интерфейс системы» на портале Websoft HCM.  В этот пакет входит файл `xaml/xaml-elements.md`, содержащий таблицу доступных элементов со ссылками на исходники.
