# Обзор среды разработки Websoft HCM

Раздел «Websoft HCM: среда разработки» представляет собой базу знаний для разработчиков, работающих с платформой WebTutor/WebSoft HCM. Здесь собраны сведения о встроенных языках и библиотеках, инструментах разработки, интерфейсе системы, API и рекомендациях по проектированию. В рамках данного пакета были изучены основные темы и часть вложенных статей. Некоторые разделы остаются на дальнейшую проработку.

## Основные возможности

- **Инструменты разработки.** Платформа включает собственный редактор программного кода OmniCode, поддерживающий C# и серверный JavaScript. OmniCode устанавливается как расширение для Visual Studio Code и позволяет создавать и редактировать серверные объекты (агенты, библиотеки, действия)【931861937373518†screenshot】.  
- **Языки и скрипты.** Поддерживаются два основных языка: C# и Server‑Side JavaScript, а для запросов к базе данных используется язык XQuery. Для серверного JavaScript доступен обширный набор встроенных функций, объектов и глобальных переменных【529610742775508†screenshot】.  
- **Библиотеки.** Встроенные библиотеки (Tools, Tools_library, Tools_web, Ms_tools) предоставляют набор служебных функций, а отдельные библиотеки программного кода (.js, .pl) позволяют группировать собственные процедуры и вызывать их через `tools.call_code_library_method`【131408160490410†screenshot】.  
- **API.** Для интеграции с внешними системами предусмотрен API. Поддерживаются две версии: версия 1 (SOAP/XML) и версия 3 (OpenAPI/JSON). Версия 3 соответствует спецификации OpenAPI 3.0 и использует форматы JSON и методы GET, POST, PATCH, DELETE【927502131024056†screenshot】.
- **Интерфейс и XAML.** Интерфейс WebSoft HCM описывается посредством XAML‑разметки. Справочник элементов XAML включает элементы Action, Button, Grid, Label, ListBox и т. д. Каждый элемент имеет собственные атрибуты и события【575597786908733†screenshot】.
- **Рекомендации и примеры.** В разделе содержатся советы по разработке выборок, удалённых действий, показателей, а также рекомендации по кэшированию и дизайну.

## Ограничения данного пакета

Этот пакет является частично собранной документацией. Из‑за большого объёма базы знаний и ограничения времени не удалось подробно обработать все разделы. В частности, вложенные страницы многих функций библиотеки Tools, подробный референс API, редактор веб‑страниц и дизайн‑рекомендации требуют дальнейшего изучения. В файле `progress-report.md` приводится отчёт о текущем состоянии и планах на будущее.
