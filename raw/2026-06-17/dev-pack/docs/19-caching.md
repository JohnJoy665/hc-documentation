# Кэширование в системе Websoft HCM

Механизм кэширования играет важную роль в производительности портала Websoft HCM.  Разработчик может управлять различными видами кэша — от кэширования выборок и страниц до кэширования метаданных и шаблонов.  В данном документе приведены общие рекомендации; подробности смотрите в разделе «Кэширование в системе» базы знаний.

## Виды кэша

| Тип кэша | Описание |
| --- | --- |
| **Кэш данных** | Хранит результаты выборок, запросов API и расчётов для повторного использования.  Позволяет сократить время отклика на повторные запросы. |
| **Кэш шаблонов** | Сохраняет откомпилированные XAML/HTML‑шаблоны.  Ускоряет генерацию страниц и форм. |
| **Кэш сессий** | Хранит данные сеансов пользователей.  Управляется настройками в `xhttp.ini` и влияет на масштабируемость. |
| **Кэш прав доступа** | Сохраняет результаты проверки прав пользователя к объектам и функциям.  Обновляется при изменении ролей. |

## Рекомендации

* **Устанавливайте разумные сроки жизни кэша** – длительное хранение устаревших данных может приводить к некорректному отображению информации.  Настройте TTL (Time‑to‑Live) для каждого вида кэша в соответствии с частотой обновления данных.
* **Очистка кэша** – предоставьте пользователям и администраторам возможность очищать кэш при необходимости (например, после обновления данных или внедрения новой функциональности).  Внутри серверного кода можно вызывать функции очистки кэша.
* **Избегайте кэширования чувствительной информации** – не храните в кэше персональные данные или данные, доступ к которым должен быть ограничен.
* **Используйте кэширование выборок** – для сложных отчётов и выборок настройте кэширование, чтобы уменьшить нагрузку на БД.  При этом учитывайте параметры, по которым пользователь фильтрует данные, чтобы не выводить неправильные результаты.

> Примечание: оригинальный раздел «Кэширование в системе» содержит параметры конфигурации, названия секций и примеры.  В рамках данного пакета эти сведения частично отсутствуют.  Рекомендуется ознакомиться с полной статьёй в базе знаний для реализации оптимального кэширования.
