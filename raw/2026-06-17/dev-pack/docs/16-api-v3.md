# API версия 3 (OpenAPI/JSON)

API v3 — современный REST‑интерфейс Websoft HCM, реализованный согласно спецификации OpenAPI 3.0.  Он обеспечивает взаимодействие по протоколам HTTP/HTTPS и использует данные в формате JSON.  Этот интерфейс предназначен для интеграции внешних систем с платформой: управления учебными объектами, пользователями, событиями и вызова программных функций.

## Базовая информация

* **Формат данных** – `application/json`【465071165735325†L124-L131】.
* **Методы** – поддерживаются HTTP‑методы `GET`, `POST`, `PATCH`, `DELETE`【465071165735325†L124-L131】.
* **Аутентификация** – возможен анонимный доступ, а также Basic‑аутентификация.  Для токен‑аутентификации используется параметр `x-app-id` и учетные данные приложения.
* **Спецификация** – описание доступных ресурсов и методов можно получить по endpoint `/openapi/hcm/v1/swagger.json` или `/openapi/hcm/v1/swagger.yaml`.

## Варианты использования

### Низкоуровневый доступ

Низкоуровневый (Low‑level) вариант позволяет обращаться непосредственно к объектам системы.  Запросы включают имя класса (например, `employee`, `course`) и действие (`get`, `list`, `create`).  Пример получения списка сотрудников:

```http
GET /openapi/hcm/v1/employee?limit=50&offset=0
Authorization: Basic <credentials>
X-App-Id: my_app

Response:
{
  "data": [
    { "id": 12345, "firstName": "Иван", "lastName": "Иванов" },
    …
  ],
  "total": 1000
}
```

Права доступа к классам определяются в профиле безопасности (см. раздел «Организация доступа»).

### Высокоуровневый доступ

Высокоуровневый (High‑level) вариант предназначен для вызова конкретных программных функций.  Такие функции могут быть:

* **Стандартные функции** встроенных библиотек, отмеченные как доступные для API.
* **Методы библиотек программного кода**, которые разработчик публикует через JSON‑описание.

Чтобы опубликовать функцию, необходимо:

1. Создать файл JSON с описанием функции (метод, параметры, структура ответа).  Для генерации такого файла можно использовать `JSDoc` и команду:

   ```bash
   jsdoc -X path/to/library.js > library.json
   ```

2. Загрузить JSON через раздел «Формирование и публикация файла JSON для подключения библиотеки программного кода через API».
3. Добавить новую функцию в профиль безопасности в блок `<functions>` файла профиля【789053750046110†screenshot】.
4. Вызывать функцию через endpoint `/openapi/hcm/v1/functions/<functionName>`.

Пример запроса для вызова функции `sendNotification` из библиотеки `libMain`:

```http
POST /openapi/hcm/v1/functions/sendNotification
Authorization: Basic <credentials>
X-App-Id: my_app
Content-Type: application/json

{
  "employeeId": 12345,
  "message": "Добрый день! Вы записаны на курс."
}

Response:
{
  "status": "success"
}
```

## Организация доступа

Доступ к API регулируется тремя уровнями безопасности (см. подробности в файле `docs/14-api-overview.md`):

1. **Регистрация приложения** с получением уникального `APP‑ID`.
2. **Создание учётных данных** (логин/пароль) для приложения.
3. **Настройка профиля безопасности**, в котором перечисляются доступные классы объектов и функции.  Профиль хранится в каталоге `wtv\oapi\profiles` и имеет XML‑структуру с блоками `<objects>` и `<functions>`【789053750046110†screenshot】.

## Получение описания (definition) и проверка функций

Для проверки работоспособности API v3 используйте интерфейс Swagger (OpenAPI).  Доступ к нему обычно предоставляется по ссылке `/openapi/hcm/v1/index.html`.  Здесь можно:

* Просмотреть описание всех доступных методов и моделей данных.
* Выполнить тестовый вызов методом Try it out.
* Проверить правильность заголовков (`X-App-Id`, `Authorization`) и структуры запроса.

## Пример программирования

Ниже приведён пример на JavaScript (Node.js) для вызова метода API v3 с использованием библиотеки `axios`:

```javascript
const axios = require('axios');

async function getEmployees() {
  const response = await axios.get('https://portal.example.com/openapi/hcm/v1/employee', {
    params: { limit: 50, offset: 0 },
    headers: {
      'X-App-Id': 'my_app',
      'Authorization': 'Basic ' + Buffer.from('user:password').toString('base64')
    }
  });
  console.log(response.data);
}

getEmployees().catch(console.error);
```

Этот код отправляет GET‑запрос к API для получения списка сотрудников.  В реальном проекте необходимо обработать ошибки и реализовать повторные попытки при сбоях.

API v3 даёт гибкие возможности для интеграции с Websoft HCM, однако требует корректной настройки профилей безопасности и приложений.  При проектировании интеграции соблюдайте принципы минимальных привилегий и публикуйте только те функции и классы, которые действительно нужны внешней системе.
