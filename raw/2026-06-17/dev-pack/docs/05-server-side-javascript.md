# Server‑Side JavaScript

Серверный JavaScript (SSJS) — основной скриптовый язык для реализации бизнес‑логики в WebSoft HCM. Скрипты выполняются на сервере в контексте платформы SP‑XML и имеют доступ к многочисленным встроенным объектам и функциям. Использование SSJS позволяет:

- обрабатывать данные пользователей и курсов;
- запускать фоновые процессы и агенты;
- взаимодействовать с базой данных и внешними сервисами;
- формировать XML/XHTML‑страницы.

## Формы описания данных (.xmd)【420292193409555†screenshot】

Формы `.xmd` определяют структуру данных для объектов и хранятся в форматах SP‑XML. Описание форм и их атрибутов находится в отдельной справочной системе. На портале приводятся только ссылки на внешнюю документацию. Важно учитывать, что такие формы используются для создания пользовательских таблиц и описания данных в бизнес‑процессах.

## Экранные формы (.xms)【567339561225027†screenshot】

Файлы `.xms` содержат описание экранных форм, используемых для отображения и ввода данных. Документация на портале также отсылает к внешней справке SP‑XML. Экранные формы основаны на XAML‑разметке, поэтому для работы с ними полезно изучить раздел «Интерфейс системы».

## Встроенные функции

Серверный JavaScript предоставляет большой набор встроенных функций, сгруппированных по назначению. Каждая функция вызывается через глобальный объект `tools` или напрямую, если объявлена в глобальной области. Среди категорий:

| Категория | Примеры функций | Назначение |
|---|---|---|
| **Преобразование типов** | `Int()`, `Real()`, `OptInt()`, `OptReal()`, `DataType()` | Преобразуют аргументы к числовым типам, определяют тип данных. |
| **Работа со строками** | `Concat()`, `StrLen()`, `LowerCase()`, `UpperCase()` | Операции со строками: объединение, длина, изменение регистра. |
| **Работа с датами** | `Date()`, `AddDays()`, `FormatDate()` | Создание и форматирование дат, арифметика с датами. |
| **Массивы и коллекции** | `Array()`, `Push()`, `Sort()`, `IndexOf()` | Создание и обработка массивов и списков. |
| **Файловая система** | `FileRead()`, `FileWrite()`, `FileDelete()` | Чтение и запись файлов на сервере. |
| **Работа с URL** | `EncodeUrl()`, `DecodeUrl()`, `GetQueryParam()` | Кодирование и разбора URL. |
| **Почта и уведомления** | `SendMail()`, `SendNotification()` | Отправка письма или системного уведомления пользователям. |
| **XML/JSON** | `ParseXml()`, `ToJson()`, `FromJson()` | Разбор XML и JSON, преобразование между форматами. |
| **Безопасность** | `XssNeutralize()`, `CheckSumSid()` | Экранрование опасных символов, проверка контрольных сумм. |

Каждая функция имеет собственный синтаксис, список аргументов и возвращаемое значение. Полный реестр функций расположен в библиотеке **Tools** (см. раздел «Встроенные библиотеки»). Например, функция `Int(value)` преобразует переданный аргумент к целому неотрицательному числу; если преобразование невозможно, генерируется ошибка. Для безопасного преобразования рекомендуется использовать `OptInt(value)`, которая возвращает `0` при неуспешном преобразовании.

> Важно:
> В оригинальной документации количество встроенных функций очень велико. В данном пакете приведены только основные категории. Для подробного справочника используйте раздел «Библиотека Tools» и вложенные страницы функций.

## Встроенные объекты

Язык SSJS предоставляет встроенные объекты, представляющие ключевые сущности портала:

- `tools` — основной объект с функциями для работы с пользователями, курсами, датами, файловой системой и т. д.;
- `user` — объект текущего пользователя (в контексте запроса), содержит свойства `Id`, `FullName`, `Login` и методы для работы с профилем;
- `process` — объект бизнес‑процесса, позволяет управлять этапами и параметрами процесса;
- `session` — предоставляет доступ к данным текущей сессии;
- `application` — доступ к параметрам приложения и глобальным настройкам.

Каждый объект имеет методы и свойства, которые описаны в справочнике SP‑XML. Например, `tools.get_user(userId)` возвращает объект пользователя по идентификатору, а `user.Manager` возвращает руководителя пользователя.

## Глобальные переменные

В SSJS существуют предопределённые глобальные переменные, позволяющие получить контекст выполнения:

- `$current_user` — идентификатор текущего пользователя;
- `$current_date` — текущая дата и время на сервере;
- `$current_process` — идентификатор текущего бизнес‑процесса;
- `$current_language` — язык интерфейса пользователя.

Эти переменные доступны во всех скриптах и используются для адаптации логики под конкретного пользователя и время.

## Базовые объекты языка

Платформа поддерживает набор базовых объектов (String, Number, Date, Array, Object и т. д.), аналогичных стандартам JavaScript. Однако в контексте WebSoft HCM могут быть доступны дополнительные методы, например:

- `String.prototype.trim()` — удаляет пробелы в начале и конце строки;
- `Date.prototype.addDays(n)` — возвращает новую дату, сдвинутую на *n* дней;
- `Array.prototype.remove(value)` — удаляет элемент из массива.

Полный список базовых объектов и их расширений описан в справочнике SP‑XML.

## Работа со встроенными объектами в OmniCode

При разработке в OmniCode для JavaScript действует подсветка и автодополнение. Рекомендуется:

- импортировать необходимые библиотеки через `lib.` (см. раздел «Библиотеки программного кода»);
- использовать функцию `tools.call_code_library_method` для вызова методов из библиотек;
- обрабатывать исключения при помощи конструкции `try/catch`.

## Заключение

Server‑Side JavaScript — мощный инструмент для расширения функциональности WebSoft HCM. Грамотное использование встроенных функций, объектов и переменных позволяет реализовывать сложные бизнес‑процессы, интеграцию и гибкий интерфейс.
