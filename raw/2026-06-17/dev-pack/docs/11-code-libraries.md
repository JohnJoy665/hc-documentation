# Библиотеки программного кода

В Websoft HCM начиная с релиза 3Q 2020 появились **библиотеки программного кода**.  Это отдельные объекты системы, содержащие набор функций, написанных на Server‑Side JavaScript или C#.  Библиотеки хранятся в каталоге `\wtv\libs` (JS) или `\components` (C#) и позволяют повторно использовать программные модули в различных процессах.

Основные особенности библиотек:

* **Разделение логики** – библиотека группирует функции по назначению. Можно создавать библиотеки для аналитики, обучения, управления событиями и т.п.
* **Вызов через Tools** – для обращения к функциям библиотеки используется метод `tools.call_code_library_method(libraryCode, methodName, parameters)`【131408160490410†screenshot】.  Метод возвращает результат функции, определённый в библиотеке.
* **Получение параметров** – некоторые библиотеки используют параметры, заданные в настройках бизнес‑процессов или приложений.  Для их получения применяется функция `tools.get_process_param(library_code, parameter_name)`【521268940785931†screenshot】.
* **Хранение параметров** – глобальные параметры библиотек хранятся в отдельном объекте и позволяют переиспользовать единые настройки для портала и шаблонов.

## Пример вызова функции из библиотеки

```javascript
// Вызов функции calculateKPI из библиотеки libAnalytics для сотрудника
var result = tools.call_code_library_method('libAnalytics', 'calculateKPI', {
    employeeId: 12345,
    period: '2026-Q2'
});

// Получение параметра из библиотеки
var threshold = tools.get_process_param('libAnalytics', 'kpi_threshold');
```

## Структура библиотеки и документация

Каждая библиотека содержит один или несколько файлов с исходным кодом (`.js`, `.pl` или `.cs`), в которых определяются функции.  Дополнительно в библиотеке могут быть описаны параметры, которые настраиваются в объектах **Настройки** (бизнес‑процессы) и **Приложения**.  Документация по функциям библиотек может быть встроена в комментарии к коду и доступна через OmniCode или IDE.

Для создания собственной библиотеки выполните следующие шаги:
1. Создайте новый объект типа «Библиотека программного кода» в модуле **Администрирование → Разработка**.
2. Укажите код библиотеки (латинские символы), описание и язык (JS или C#).
3. Добавьте файл(ы) с исходным кодом и определите экспортируемые функции.
4. При необходимости задайте параметры библиотеки в разделе «Параметры».
5. Сохраните библиотеку и протестируйте её через `tools.call_code_library_method`.
