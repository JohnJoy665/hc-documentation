# Глобальные настройки системы

Объект `global_settings` определяет глобальные параметры, влияющие на работу всей платформы. Эти настройки позволяют задавать пути к файлам, коды подразделений, параметры процессов и другие системные значения. Изменение глобальных настроек обычно выполняет администратор портала.

## Основные параметры

Ниже приведены примеры параметров из разделов «Глобальные настройки системы» и «Настройки settings»【400092422210296†screenshot】. Полный список гораздо шире и должен быть дополнен при дальнейшем изучении:

| Параметр | Назначение |
|---|---|
| `branch_code` | Код организации (филиала), используемый в идентификаторах для сотрудников и курсов. |
| `courses_path` | Путь к каталогу, в котором хранятся материалы курсов и ресурсов. |
| `global_process_type_id` | ID типа бизнес‑процесса, применяемого по умолчанию для новых процессов. |
| `admin_not_auth` | Разрешает (true)/запрещает (false) открывать портал без авторизации с административными правами. |
| `allow_self_register` | Разрешает/запрещает самостоятельную регистрацию пользователей. |
| `auth_check_eval` | Включает проверку экзаменов при авторизации. |
| `login_title` | Заголовок, отображаемый на странице входа. |
| `default_locale` | Код языка по умолчанию (например, `ru_RU`), используемый при отображении интерфейса. |
| `date_format` | Формат отображения дат. |
| `time_zone` | Часовой пояс портала. |

Большинство параметров имеют тип `string` или `boolean`. Для редактирования используются административный интерфейс или константы в конфигурационных файлах.

## Настройки settings

Объект `settings` содержит большое количество параметров, влияющих на интерфейс и работу портала. Примеры таких параметров:

- `papi_url` — адрес API для внешних интеграций;
- `email_from` — адрес отправителя по умолчанию для почтовых уведомлений;
- `allow_anonymous_courses` — флаг, позволяющий анонимный доступ к курсам;
- `enable_password_expiration` — включает срок действия пароля;
- `default_password_length` — длина пароля по умолчанию;
- `use_https_only` — принудительное использование HTTPS;
- `max_upload_size` — максимальный размер загружаемого файла.

> Важно:
> Любое изменение глобальных параметров рекомендуется предварительно тестировать на стенде, так как некоторые настройки могут потребовать перезапуска сервера. Для получения полного списка и описания всех параметров обратитесь к оригинальной документации WebSoft HCM (подраздел «Настройки settings» в базе знаний).
