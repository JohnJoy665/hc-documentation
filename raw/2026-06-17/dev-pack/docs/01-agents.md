# Агент в системе Websoft HCM

Агенты в Websoft HCM представляют собой серверные процессы, выполняющие автоматические операции без участия пользователя. Они запускаются по расписанию или по событию и позволяют организовать фоновую обработку данных, отправку уведомлений, сбор статистики и другие действия. Различают несколько типов агентов:

| Тип агента | Назначение |
|---|---|
| **Выполняемый код** | Запускает пользовательский скрипт на заданном языке (Server‑Side JS или C#), например для обработки данных или интеграции. |
| **Задача** | Используется для планирования операций, которые должны выполняться периодически (ежедневно, еженедельно и т.д.). |
| **Импорт Excel** | Автоматически импортирует данные из Excel‑файлов. |
| **Скачивание обновлений** | Периодически проверяет наличие обновлений и загружает их на сервер. |

## Управление агентами

Агенты создаются и настраиваются через административный интерфейс. Каждому агенту задаётся тип, набор параметров и расписание. Например, для агента, выполняющего скрипт, указывается код библиотеки и параметр «Время выполнения». При разработке серверных агентов рекомендуется:

* разделять бизнес‑логику на отдельные функции и вызывать их внутри агента;
* тщательно тестировать код на тестовом стенде перед публикацией;
* контролировать ресурсы: фоновые процессы не должны влиять на производительность портала;
* использовать функции логирования (см. раздел о логах OmniCode) для диагностики.

> Важно:
> В исходной документации упоминается, что некоторые типы агентов (например, агенты, которые скачивают обновления) могут быть встроенными и не требуют создания вручную. Для написания своих агентов используется Server‑Side JavaScript или C#, как описано в соответствующих разделах.

Дополнительные сведения и примеры кода для создания агентов можно найти в разделе о библиотеке Tools и OmniCode.
