# Системные требования WebSoft HCM

Перед установкой WebSoft HCM необходимо подготовить серверное окружение. Разработчики сообщают о минимальных требованиях к аппаратуре и программному обеспечению, а также о технологиях, используемых в продукте.

## Используемые технологии

WebSoft HCM построен на модульной архитектуре и сочетает несколько технологий:

* **Node.js и TypeScript** — ядро и микросервисы реализованы на JavaScript/TypeScript. Это требует установки среды Node.js на сервере.
* **.NET Core/ASP.NET** — отдельные компоненты (например, WebSoftIDM и сервер записи) используют .NET 6 и ASP.NET. Для Windows‑установки требуется наличие .NET 6 Runtime.
* **СУБД** — поддерживаются Microsoft SQL Server и PostgreSQL. Для работы с внешней БД необходимы дополнительные компоненты .NET и конфигурация базы (см. файл 06‑external‑database.md).

## Аппаратные требования

В документации указано, что требования зависят от количества одновременно работающих пользователей. Чем больше нагрузка, тем мощнее должен быть сервер. В обобщённом виде рекомендации выглядят следующим образом【693318755733946†screenshot】:

| Количество активных пользователей | CPU | RAM | Диск |
|---|---|---|---|
| До 500 | несколько ядер (4 и более) | 8 ГБ и более | SSD, ≥50 ГБ свободного пространства |
| 500–2000 | 4–8 ядер | 16 ГБ и более | SSD/RAID, ≥100 ГБ |
| Более 2000 | 8 ядер и более | 32 ГБ и более | Быстрый SSD/RAID 10, резерв под БД |

> **Важно:** При размещении базы данных на том же сервере рекомендуется увеличивать объём оперативной памяти и выделять отдельный высокопроизводительный диск для СУБД.

## Программные требования

* **Операционная система:** Поддерживаются современные дистрибутивы Linux (например, Alt Linux p9/p10, Ubuntu 20.04/22.04) и Windows Server 2016/2019/2022. Для установки в контейнерах используется Docker и Docker Compose.
* **Дополнительные пакеты:** Для записи видео (сервер VClass) и телеметрии требуются пакеты FFmpeg и вспомогательные утилиты【848555681754747†screenshot】.
* **Доступ к сети:** Сервер должен иметь доступ к интернету для загрузки образов Docker и установки обновлений. При конфигурации внешних сервисов (SMTP, S3 и т. д.) необходимо настроить соответствующие порты и брандмауэр.

Эти требования являются отправной точкой. Конкретные параметры могут отличаться в зависимости от сценария использования, поэтому перед закупкой оборудования или выделением ресурсов стоит проанализировать предполагаемую нагрузку.
