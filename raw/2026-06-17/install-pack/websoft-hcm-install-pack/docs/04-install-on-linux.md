# Установка WebSoft HCM на Linux

В этом разделе приводятся общие рекомендации по развёртыванию WebSoft HCM на операционных системах семейства Linux. Конкретные шаги могут различаться для разных дистрибутивов (Alt Linux, Ubuntu, Debian и т. п.), однако основная последовательность действий одинакова.

## Подготовка системы

1. **Обновите систему и установите зависимости.** Необходимо установить Node.js, .NET 6 Runtime, необходимые библиотеки (libssl, libicu и др.) и сервер базы данных (MSSQL Server for Linux или PostgreSQL). Для дистрибутивов Alt Linux и Ubuntu команды будут отличаться, например:

   ```bash
   # Alt Linux
   sudo apt-get update
   sudo apt-get install dotnet-runtime-6.0 nodejs libssl1.1 ffmpeg

   # Ubuntu
   sudo apt update
   sudo apt install dotnet-runtime-6.0 nodejs ffmpeg
   ```

2. **Создайте системного пользователя.** Для безопасности рекомендуется запускать службы под отдельной учётной записью (например, `websoft`):

   ```bash
   sudo useradd -m -s /bin/bash websoft
   sudo passwd websoft
   ```

## Установка WebSoft HCM на Alt Linux

Документация предлагает пакетный установщик для Alt Linux, который разворачивает файлы и создаёт необходимые службы. В общих чертах процесс выглядит так【808917639115557†screenshot】:

1. Скачайте архив или rpm‑пакет из официального репозитория.
2. Установите пакет:

   ```bash
   sudo rpm -Uvh websoft-hcm-server-*.rpm
   ```

3. Установщик распакует файлы в каталог `/opt/websoft/hcm` и создаст службу systemd (`websoft-hcm.service`).
4. Отредактируйте конфигурационный файл `/etc/websoft/hcm/config.json`, указав параметры подключения к базе данных, порту HTTP, каталогам логов и др.
5. Запустите и проверьте службу:

   ```bash
   sudo systemctl enable --now websoft-hcm.service
   sudo systemctl status websoft-hcm.service
   ```

## Установка WebSoft HCM на Ubuntu

В качестве примера для Ubuntu 20.04/22.04 можно использовать установщик в формате `.deb`. Общая процедура:

1. Скачайте .deb‑пакет с сервера WebSoft.
2. Установите его, удовлетворив зависимости:

   ```bash
   sudo dpkg -i websoft-hcm-server_*.deb || sudo apt-get -f install
   ```

3. После установки файлы будут размещены в `/usr/share/websoft/hcm`. В директории `/etc/websoft/hcm` создаётся конфигурационный файл.
4. Отредактируйте конфигурацию, указав подключение к базе данных и параметры SMTP, если планируется отправка уведомлений.
5. Запустите службу:

   ```bash
   sudo systemctl restart websoft-hcm
   sudo systemctl enable websoft-hcm
   ```

Дополнительные действия, такие как настройка HTTPS и запуск сервера как службы, описаны в соответствующих разделах (см. 07‑https.md и 08‑run‑as‑service.md).

## Создание тестового стенда

Для создания тестового стенда рекомендуется клонировать рабочую среду на отдельный сервер или виртуальную машину, скопировать базу данных и выполнить обновление приложений на нём. Это позволяет тестировать обновления без риска для боевой системы【392378813991336†screenshot】. Существуют инструкции и по развёртыванию тестового стенда с помощью Docker, что упрощает восстановление среды.

## Примечания

* Статьи по установке под Linux сопровождаются файлами конфигурации и примерами сервисов `systemd`. Внимательно следуйте инструкциям, не изменяя ключевые параметры.
* Если база данных расположена на отдельном сервере, убедитесь, что порт СУБД открыт и сервер WebSoft HCM имеет доступ к базе.
* В случае Alt Linux рекомендуют использовать репозиторий WebSoft, чтобы получать обновления через `apt-get update && apt-get upgrade`.
