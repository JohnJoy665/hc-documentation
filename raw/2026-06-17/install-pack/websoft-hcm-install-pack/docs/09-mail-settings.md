# Настройка отправки почтовых уведомлений

WebSoft HCM поддерживает уведомления о событиях (регистрации, назначении курсов, отчётах и т. п.) по электронной почте. Для корректной отправки писем необходимо указать параметры SMTP‑сервера и настроить использование TLS. Руководство описывает процедуру через административный интерфейс.

## Основные настройки SMTP

1. Перейдите в **Административный интерфейс → Сервис → Общие настройки → Администрирование → Отправка сообщений**. Здесь настраиваются глобальные параметры почты.
2. В поле **Адрес отправителя** укажите e‑mail, который будет указан в строке From (например `noreply@example.com`).
3. Заполните **SMTP‑сервер** и, при необходимости, порт (обычно 25 или 587).
4. Отметьте **Аутентификация на SMTP‑сервере** и введите логин/пароль, если сервер требует авторизации. После сохранения параметры вступят в силу【589265140645102†screenshot】.

## Включение TLS/SSL

Если SMTP‑сервер работает через защищённое соединение, необходимо дополнительно настроить использование TLS.

1. Откройте карту системного уведомления **email1** (Менеджер → Конструктор → Системные уведомления) и найдите параметр **Flags**. Добавьте значение `Smtps`, чтобы включить работу через SSL【589265140645102†screenshot】.
2. В свойствах уведомления появятся дополнительные поля:
   * `bUseTLS` — включение шифрования TLS. Для SSL/TLS установите `true`; для STARTTLS установите `true` для `bUseTLS` и `false` для `bUseTLSPort`【583507882027265†screenshot】.
   * `bUseTLSPort` — признак использования порта для TLS. Установите `true`, если сервер использует отдельный порт для SSL (обычно 465), или `false`, если используется STARTTLS на стандартном порту.
   * `bUseStartTLS` — (опционально) включает расширение STARTTLS.

3. После изменения параметров сохраните уведомление и **перезапустите сервер WebSoft HCM**, чтобы обновлённые параметры применились【583507882027265†screenshot】.

## Рекомендации

* Используйте почтовый сервер, поддерживающий современный TLS 1.2; при необходимости укажите альтернативный порт (465/587).
* Проверьте работоспособность, отправив тестовое письмо из административного интерфейса. В случае ошибок проверьте лог `mail.log` в каталоге `logs` и корректность учётных данных.
* Для рассылок большого объёма сообщений рекомендуется настроить ограничения на количество писем в минуту во избежание блокировок со стороны провайдера.

Настройка почтовых уведомлений позволяет оперативно информировать пользователей о событиях в системе. Соблюдайте требования безопасности: используйте сложные пароли, шифрование и проверенные SMTP‑серверы.
