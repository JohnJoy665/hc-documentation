# Установка WebSoft HCM на Windows

Официальная документация предлагает установщик для Windows, который разворачивает портал WebSoft HCM и связанные компоненты через графический мастер.  Ниже приведён обзор основных шагов, которые нужно выполнить, чтобы корректно установить систему и подготовить сервер к работе.

## Запуск инсталлятора

1. **Скачайте установочный пакет** с портала WebSoft и запустите его от имени администратора. В начале мастер предложит выбрать язык.
2. **Примите лицензионное соглашение** и перейдите к выбору каталога установки. Документация рекомендует устанавливать WebSoft HCM в корень диска (например `C:\WebSoftServer`) и **не использовать путь с пробелами** или папку `Program Files`【775412749964422†screenshot】. Это упростит настройки прав доступа.
3. **Выберите компоненты**. В типовом сценарии отмечают компоненты «Администратор» и «Сервер»; остальные элементы можно оставить по умолчанию【556589187024341†screenshot】.
4. На странице **обзора** проверьте выбранные параметры и нажмите **Install/Установить**. Мастер распакует файлы и зарегистрирует службы.

## Действия после установки

После завершения установки необходимо выполнить несколько дополнительных настроек:

* **Права на каталоги.** Для корректной работы сервису требуются права на запись в ряд каталогов: `logs`, `applications`, `wt\web\webtutor`, директорию полнотекстовых индексов Lucene и другие. В документе перечисляется список путей, и рекомендовано предоставить учетной записи сервиса полные права на весь каталог `WebSoftServer`【64675945729154†screenshot】.
* **Записи в файле hosts.** Если внешний DNS не разрешает доменное имя портала, добавьте соответствующие строки в `C:\Windows\System32\drivers\etc\hosts` (например, `127.0.0.1 portal.example.com`)【529233705570330†screenshot】. Это позволит открыть портал по доменному имени до появления записи в DNS.
* **Запуск как служба.** По умолчанию веб‑сервер WebSoft HCM работает как приложение. Чтобы запустить его в виде службы Windows и автоматически запускать при старте, воспользуйтесь инструкциями в разделе 08 «Запуск как служба».
* **Лицензирование.** После установки необходимо активировать лицензию. В базе знаний есть отдельная статья о получении и установке лицензии; следуйте ей, чтобы добавить лицензионные файлы.

> **Важно:** Путь установки без пробелов и корректно настроенные права доступа критически важны для работы сервиса.  Несоблюдение этих требований может привести к ошибкам при записи файлов и запуске приложений【775412749964422†screenshot】.

Установка на Windows завершится появлением ярлыков «WebSoft HCM Server» и «WebSoft Administrator» в меню «Пуск». При открытии администратора проверьте, что портал запускается по указанному доменному имени. При необходимости внесите изменения в сетевые настройки или обратитесь к администратору домена.
