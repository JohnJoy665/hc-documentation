## Назначение

`Object` — конструктор объекта языка SP-XML, близкого по синтаксису к JavaScript.

Он создает новый объект, в который можно добавлять динамические свойства и получать их значения по имени.

По умолчанию для объекта доступны:

- свойство/метод проверки наличия атрибута `HasProperty`;
- методы `AddProperty()`;
- `SetProperty()`;
- `GetProperty()`;
- `GetOptProperty()`.

> Важно: серверный язык SP-XML похож на JavaScript по синтаксису, но не является обычным JavaScript. При генерации кода для WebTutor / WebSoft HCM нужно ориентироваться на возможности SP-XML, а не на современные JS API.

## Создание объекта

```javascript
a = new Object;
```

Объект после создания может использоваться как контейнер для произвольных именованных свойств.

## Свойства и методы

### Свойство / проверка

| Имя | Назначение |
|---|---|
| `HasProperty` | Проверяет, существует ли свойство с заданным именем. |

### Методы

| Метод | Назначение |
|---|---|
| `AddProperty()` | Добавляет новое свойство и присваивает ему значение. |
| `GetProperty()` | Возвращает значение свойства. Если свойства нет — вызывает ошибку. |
| `GetOptProperty()` | Возвращает значение свойства. Если свойства нет — возвращает `undefined` или значение по умолчанию. |
| `SetProperty()` | Устанавливает значение свойства. Если свойства нет — добавляет его. |

## `Object.HasProperty`

Проверяет, существует ли атрибут / свойство с заданным именем.

Возвращает:

- `true`, если свойство существует;
- `false`, если свойство отсутствует.

См. также: [`GetOptProperty()`](#objectgetoptproperty), [`SetProperty()`](#objectsetproperty).

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `propName` | `string` | да | Наименование атрибута объекта. |

### Результат

| Тип | Описание |
|---|---|
| `Bool` | Признак наличия свойства. |

### Пример

```javascript
if ( obj.HasProperty( 'name' ) )
{
    value = obj.GetProperty( 'name' );
}
```

## `Object.GetProperty()`

Возвращает значение атрибута объекта.

Если атрибут отсутствует, метод вызывает ошибку.

См. также: [`GetOptProperty()`](#objectgetoptproperty), [`SetProperty()`](#objectsetproperty).

### Синтаксис

```javascript
GetProperty( propName )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `propName` | `string` | да | Наименование атрибута объекта. |

### Результат

| Тип | Описание |
|---|---|
| `Any` | Значение свойства объекта. |

### Пример

```javascript
AppConfig.GetProperty( 'alt-app-name' );
```

### Поведение при отсутствии свойства

Если свойства с указанным именем нет, возникает ошибка.

Для безопасного чтения необязательных свойств следует использовать `GetOptProperty()`.

## `Object.GetOptProperty()`

Возвращает значение атрибута объекта.

Если атрибут отсутствует, возвращает:

- `undefined`, если значение по умолчанию не передано;
- `defaultVal`, если значение по умолчанию передано вторым аргументом.

### Синтаксис

```javascript
GetOptProperty( propName )
GetOptProperty( propName, defaultVal )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `propName` | `string` | да | Наименование атрибута объекта. |
| `defaultVal` | `Any` | нет | Значение по умолчанию, возвращаемое при отсутствии атрибута. По умолчанию равно `undefined`. |

### Результат

| Тип | Описание |
|---|---|
| `Any` | Значение свойства, `undefined` или переданное значение по умолчанию. |

### Пример

```javascript
appName = AppConfig.GetOptProperty( 'alt-app-name', 'WebTutor' );
```

## `Object.SetProperty()`

Устанавливает значение атрибута объекта.

Если атрибут отсутствует, метод добавляет его.

См. также: [`GetOptProperty()`](#objectgetoptproperty), [`GetProperty()`](#objectgetproperty).

### Синтаксис

```javascript
SetProperty( propName, propVal )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `propName` | `string` | да | Наименование атрибута объекта. |
| `propVal` | `Any` | да | Значение атрибута объекта. |

### Результат

| Тип | Описание |
|---|---|
| `Any` | Значение результата по документации указано как `Any`. |

### Пример

```javascript
obj.SetProperty( 'status', 'active' );
```

## `Object.AddProperty()`

Добавляет в объект новый атрибут и присваивает ему значение.

### Синтаксис

```javascript
AddProperty( propName, propVal )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `arg1` / `propName` | `string` | да | Наименование атрибута. |
| `arg2` / `propVal` | `variant` | да | Значение атрибута. |

### Результат

| Тип | Описание |
|---|---|
| `undefined` | Метод не возвращает полезного значения. |

### Пример

```javascript
obj.AddProperty( 'login', 'user1' );
```

## Сравнение методов работы со свойствами

| Метод / свойство | Если свойство существует | Если свойства нет | Основной сценарий |
|---|---|---|---|
| `HasProperty( propName )` | Возвращает `true` | Возвращает `false` | Проверка перед чтением или логикой ветвления. |
| `GetProperty( propName )` | Возвращает значение | Вызывает ошибку | Чтение обязательного свойства. |
| `GetOptProperty( propName )` | Возвращает значение | Возвращает `undefined` | Безопасное чтение необязательного свойства. |
| `GetOptProperty( propName, defaultVal )` | Возвращает значение | Возвращает `defaultVal` | Безопасное чтение с дефолтом. |
| `SetProperty( propName, propVal )` | Обновляет значение | Добавляет свойство | Универсальная установка значения. |
| `AddProperty( propName, propVal )` | Добавляет свойство | Добавляет свойство | Явное добавление нового свойства. |

## Практические замечания для WebTutor / SP-XML

1. Для чтения обязательных параметров можно использовать `GetProperty()`, но нужно помнить, что отсутствие свойства приведет к ошибке.

2. Для чтения пользовательских, конфигурационных и необязательных параметров безопаснее использовать `GetOptProperty()`.

3. Если нужно установить значение независимо от того, было ли свойство раньше создано, используйте `SetProperty()`.

4. `Object` удобно использовать как простой словарь / контейнер параметров при передаче настроек в функции.

5. Не следует автоматически заменять эти методы современными JavaScript-подходами без проверки совместимости с SP-XML.

## Краткая сводка для Codex

`Object` в SP-XML — базовый динамический объект для хранения именованных свойств.

Главные правила:

- `new Object` создает новый объект;
- `HasProperty( name )` проверяет наличие свойства;
- `GetProperty( name )` читает обязательное свойство и падает с ошибкой, если свойства нет;
- `GetOptProperty( name )` безопасно читает необязательное свойство и возвращает `undefined`, если свойства нет;
- `GetOptProperty( name, defaultVal )` возвращает `defaultVal`, если свойства нет;
- `SetProperty( name, value )` устанавливает или добавляет свойство;
- `AddProperty( name, value )` добавляет свойство и возвращает `undefined`.

При генерации кода для WebTutor / WebSoft HCM предпочтительно использовать `GetOptProperty()` для необязательных параметров и `SetProperty()` для универсальной записи значений.
