## Назначение

`String` — базовый объект языка SP-XML для хранения и обработки строковых данных.

По набору методов объект частично похож на строковый объект JavaScript, но при генерации кода для WebTutor / WebSoft HCM нужно учитывать особенности SP-XML.

Главное отличие: `String.length` в данной документации описан как длина строки **в байтах**, а не как количество символов. Для нового кода часто рекомендуется использовать специализированные встроенные функции SP-XML, например `StrLen()`, `StrLowerCase()`, `StrRangePos()` и другие.

> Важно: серверный язык SP-XML похож на JavaScript по синтаксису, но не является обычным JavaScript. Нельзя автоматически переносить поведение современных JavaScript-строк на SP-XML без проверки по документации.

## `String.length`

Возвращает длину строки в байтах.

### Результат

| Тип | Описание |
|---|---|
| `Integer` | Длина строки в байтах. |

### Рекомендация

В документации рекомендуется использовать аналогичную функцию `StrLen()`.

### Важное отличие от JavaScript

В обычном JavaScript `string.length` обычно воспринимается как количество UTF-16 code units, а не байтов. В SP-XML по данному описанию `String.length` возвращает именно длину в байтах.

Это важно для строк в UTF-8 и других многобайтовых кодировках: количество байтов может не совпадать с количеством видимых символов.

## `String.Allocate()`

Устанавливает длину строкового объекта.

Если новая длина больше текущей, содержимое добавленной части строки неопределено.

Метод обычно используется для передачи возвращаемых строковых аргументов в методы `DllWrapper`.

### Синтаксис

```javascript
Allocate( newLen )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `newLen` | `Integer` | да | Новая длина строки в байтах. |

### Результат

| Тип | Описание |
|---|---|
| `undefined` | Метод изменяет строковый объект и не возвращает полезное значение. |

### Практическое замечание

`Allocate()` не следует использовать как обычный способ создания или дополнения строк. Метод предназначен для специальных сценариев, в частности для взаимодействия с `DllWrapper`.

## `String.charAt()`

Возвращает символ с указанной позиции в виде односимвольной строки.

Если `index` превышает количество символов в строке, возвращает пустую строку.

### Синтаксис

```javascript
charAt( index )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `index` | `Integer` | да | Номер позиции в строке. Отсчет ведется с нуля. |

### Результат

| Тип | Описание |
|---|---|
| `String` | Символ в виде односимвольной строки или пустая строка, если индекс вне диапазона. |

## `String.charCodeAt()`

## `String.indexOf()`

Ищет первое местоположение подстроки в пределах объекта `String`.

Если подстрока не найдена, возвращает `-1`.

### Синтаксис

```javascript
indexOf( subString )
indexOf( subString, startPos )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `subString` | `String` | да | Подстрока для поиска. |
| `startPos` | `Integer` | нет | Индекс начала поиска. |

### Результат

| Тип | Описание |
|---|---|
| `Integer` | Индекс первого вхождения подстроки или `-1`, если подстрока не найдена. |

### Пример

```javascript
pos = text.indexOf( 'login' );
if ( pos != -1 )
{
    // Подстрока найдена.
}
```

## `String.lastIndexOf()`

Ищет последнее местонахождение подстроки в пределах объекта `String`.

Если подстрока не найдена, возвращает `-1`.

### Синтаксис

```javascript

lastIndexOf( substring, startindex )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `substring` | `String` | да | Подстрока для поиска. |
| `startindex` | `Integer` | нет | Индекс, с которого начинается поиск. |

### Результат

| Тип | Описание |
|---|---|
| `Integer` | Индекс последнего вхождения подстроки или `-1`, если подстрока не найдена. |

### Аргументы

| Аргумент | Тип | Обязательный | Описание |

### Результат

### Рекомендация

## `String.ToWinLineBreaks()`

Метод указан в списке методов объекта `String`.

В предоставленном фрагменте документации подробное описание метода отсутствует.

По названию метода можно предположить, что он приводит переносы строк к Windows-формату, то есть к `\r\n`, но это нужно подтвердить по первичной документации перед использованием в критичном коде.

### Синтаксис

```javascript
ToWinLineBreaks()
```

### Результат

В предоставленном фрагменте документации результат не указан.

## Рекомендованные аналоги через встроенные функции

В исходной документации для части методов объекта `String` указаны предпочтительные аналоги через встроенные функции SP-XML.

| Метод / свойство `String` | Рекомендуемый аналог | Комментарий |
|---|---|---|
| `String.length` | `StrLen()` | Рекомендуется вместо прямого обращения к `length`. |
| `String.slice()` | `StrRangePos()`, `StrLeftRange()` | Рекомендуются для получения фрагментов строки. |
| `String.substr()` | `StrRightRangePos()` | Рекомендуется для получения подстроки. |
| `String.toLowerCase()` | `StrLowerCase()` | Рекомендуется для перевода строки в нижний регистр. |

## Практические замечания для WebTutor / SP-XML

1. `String.length` возвращает длину строки в байтах. Не использовать его бездумно как количество символов.
2. Для нового серверного кода WebTutor предпочтительно использовать документированные функции `Str*`, если документация прямо рекомендует их вместо методов объекта `String`.
3. Методы `indexOf()` и `lastIndexOf()` возвращают `-1`, если подстрока не найдена.
4. `charAt()` возвращает пустую строку, если индекс выходит за пределы строки.
5. `Allocate()` предназначен для специальных сценариев, например для работы с `DllWrapper`, а не для обычной обработки текста.
6. `split()` возвращает обычный массив `Array`.
7. Поведение методов со строками в многобайтовых кодировках нужно проверять по SP-XML-документации, а не переносить напрямую из JavaScript.

## Краткая сводка для Codex

`String` в SP-XML — базовый объект для работы со строками. Он похож на JavaScript `String`, но не должен считаться полностью совместимым с JavaScript.

Ключевые правила генерации кода:

- не считать SP-XML обычным JavaScript;
- помнить, что `String.length` возвращает длину в байтах;
- предпочитать `StrLen()` вместо `String.length`, если нужна документированная функция длины;
- предпочитать `StrLowerCase()` вместо `toLowerCase()`;
- предпочитать `StrRangePos()`, `StrLeftRange()`, `StrRightRangePos()` для операций выделения подстрок, если это соответствует задаче;
- использовать `indexOf()` / `lastIndexOf()` с проверкой результата на `-1`;
- использовать `split()` для разделения строки на `Array`;
- использовать `Allocate()` только в специальных сценариях, например при работе с `DllWrapper`;
- не использовать метод `ToWinLineBreaks()` без проверки полного описания, так как в текущем фрагменте документации указан только его заголовок.
