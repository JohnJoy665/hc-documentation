# ClassObject

## Краткое описание

`ClassObject` — базовый объект для всех объектов определенного класса.

Объекты такого типа создаются через конструкторы конкретных классов платформы или библиотек.

## Пример

```javascript
item = new ui2_client.card_view_item;
```

В этом примере создается объект класса `ui2_client.card_view_item`.

## Важные замечания для разработки

- `ClassObject` не используется как обычный прикладной контейнер данных.
- На практике разработчик чаще работает не с самим базовым типом `ClassObject`, а с объектами конкретных классов платформы.
- Конкретный набор свойств и методов зависит от класса создаваемого объекта.
- При генерации кода нельзя считать, что все `ClassObject` имеют одинаковые прикладные свойства.

## Мини-сводка для Codex

`ClassObject` — базовый тип объектов определенного класса. Используется через конкретные классы, например `new ui2_client.card_view_item`. Свойства и методы нужно уточнять по документации конкретного класса.

# ClassObjectsArray

## Краткое описание

`ClassObjectsArray` — массив, содержащий объекты определенного класса.

В отличие от стандартного массива `Array`, объект `ClassObjectsArray`:

- контролирует типы добавляемых элементов;
- предназначен для хранения объектов одного заданного класса;
- поддерживает метод `Add()`.

## Пример

```javascript
items = new ClassObjectsArray( ui2_client.card_view_item );
```

В этом примере создается массив, предназначенный для хранения объектов класса `ui2_client.card_view_item`.

## Отличие от Array

| Объект | Назначение | Контроль типов | Особенности |
|---|---|---:|---|
| `Array` | Обычный массив значений | Нет | Может содержать произвольные значения |
| `ClassObjectsArray` | Массив объектов определенного класса | Да | Проверяет соответствие элементов заданному классу, поддерживает `Add()` |

## Важные замечания для разработки

- `ClassObjectsArray` следует использовать там, где API платформы ожидает массив объектов конкретного класса.
- Нельзя автоматически заменять `ClassObjectsArray` на обычный `Array`, если объект передается во внутренние методы платформы.
- При создании `ClassObjectsArray` необходимо передать класс элементов.
- Метод `Add()` упомянут в документации, но его подробное описание в исходном фрагменте не приведено. При необходимости использовать `Add()` нужно уточнить сигнатуру по документации конкретного объекта или по рабочим примерам.

## Мини-сводка для Codex

`ClassObjectsArray` — типизированный массив объектов заданного класса. Создается через `new ClassObjectsArray( SomeClass )`. Не является полной заменой стандартного `Array`, так как контролирует типы и используется в API, где требуется коллекция объектов конкретного класса.

# DataCache

## Краткое описание

`DataCache` — объект для хранения пар `"имя ключа"` — `"значение"`.

Объект поддерживает:

- быстрый поиск по ключу;
- thread-safe-доступ;
- установку времени жизни отдельных записей.

## Глобальные объекты

В приложении заранее доступен объект:

```javascript
GlobalCache
```

`GlobalCache` является объектом `DataCache` и может использоваться для хранения данных в общем кэше приложения.

Также существует глобальный объект:

```javascript
GlobalStorage
```

`GlobalStorage` обладает теми же свойствами и методами, что и `GlobalCache`, но хранит значения перманентно между запусками приложения.

## Создание нового DataCache

Помимо использования `GlobalCache`, можно создавать другие объекты `DataCache`:

```javascript
cache = new DataCache;
```

## Методы DataCache

- `DeleteEntry()`
- `GetEntry()`
- `SetEntry()`

---

## DataCache.DeleteEntry()

Удаляет запись с заданным ключом.

Если запись отсутствует, ничего не делается.

### Синтаксис

```javascript
DeleteEntry( keyName )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `keyName` | `string` | Да | Имя ключа |

### Возвращаемое значение

В исходном описании возвращаемое значение не указано.

Для практического использования метод следует воспринимать как операцию удаления записи из кэша.

### Пример

```javascript
GlobalCache.DeleteEntry( "some_key" );
```

---

## DataCache.GetEntry()

Возвращает значение записи с заданным ключом.

Если запись не найдена, возвращает `undefined`.

### Синтаксис

```javascript
GetEntry( keyName )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `keyName` | `string` | Да | Имя ключа |

### Возвращаемое значение

| Тип | Описание |
|---|---|
| Любой тип значения или `undefined` | Значение записи, если ключ найден. Если ключ отсутствует, возвращается `undefined`. |

### Пример

```javascript
value = GlobalCache.GetEntry( "some_key" );

if ( value == undefined )
{
    // Запись отсутствует в кэше.
}
```

---

## DataCache.SetEntry()

Устанавливает значение записи с заданным ключом.

### Синтаксис

```javascript
SetEntry( keyName, value )
SetEntry( keyName, value, options )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `keyName` | `string` | Да | Имя ключа |
| `value` | Любой тип | Да | Значение записи |
| `options` | `object` | Нет | Базовый объект, содержащий опции |

### Поддерживаемые опции

| Опция | Тип | Описание |
|---|---|---|
| `TTL` | `integer` | Время жизни записи в секундах |

### Возвращаемое значение

В исходном описании возвращаемое значение не указано.

### Примеры

Запись значения без ограничения времени жизни:

```javascript
GlobalCache.SetEntry( "some_key", "some_value" );
```

Запись значения с TTL:

```javascript
GlobalCache.SetEntry( "some_key", "some_value", { TTL: 300 } );
```

В этом примере запись будет жить 300 секунд.

## GlobalCache и GlobalStorage

| Объект | Хранение | Переживает перезапуск приложения | Назначение |
|---|---|---:|---|
| `GlobalCache` | Временный кэш | Нет / не заявлено в описании | Быстрый общий кэш приложения |
| `GlobalStorage` | Перманентное хранилище | Да | Хранение значений между запусками приложения |

## Важные замечания для разработки

- `DataCache` подходит для данных, к которым нужен быстрый доступ по строковому ключу.
- `DataCache` является thread-safe, поэтому может использоваться в многопоточном окружении приложения.
- Для временных данных лучше использовать `GlobalCache` или отдельный `DataCache`.
- Для данных, которые должны сохраняться между запусками приложения, следует использовать `GlobalStorage`.
- Если используется `TTL`, код должен учитывать, что запись может исчезнуть из кэша после истечения времени жизни.
- `GetEntry()` возвращает `undefined`, если ключ не найден. Это нужно явно проверять.

## Мини-сводка для Codex

`DataCache` — thread-safe key-value-хранилище с быстрым поиском по ключу. В системе есть `GlobalCache`; также доступен `GlobalStorage`, который сохраняет значения между запусками приложения. Основные методы: `SetEntry(keyName, value[, options])`, `GetEntry(keyName)`, `DeleteEntry(keyName)`. Опция `TTL` задает время жизни записи в секундах.

# Error

## Краткое описание

`Error` — объект, содержащий ошибку или исключение, возникающее при выполнении кода.

Ошибка прерывает дальнейшее выполнение и содержит:

- описание ошибки;
- отладочную информацию;
- стек вызовов.

Ошибка может генерироваться:

- интерпретатором;
- встроенной функцией;
- методом объекта;
- свойством объекта;
- пользовательским кодом через оператор `throw`.

## Создание ошибки

Следующие конструкции эквивалентны:

```javascript
throw 'Some error';
```

```javascript
throw new Error( 'Some error' );
```

## Важные замечания для разработки

- Для явной генерации ошибки можно использовать строку или объект `Error`.
- Если нужно сделать намеренное прерывание выполнения с понятным текстом, можно использовать `throw new Error( "Описание ошибки" )`.
- В SP-XML коде WebTutor важно давать ошибкам понятные сообщения, потому что они попадают в отладочную информацию и помогают анализировать проблемы на сервере.
- При генерации кода не следует без необходимости скрывать ошибки пустыми `catch`-блоками, так как это затрудняет диагностику.

## Пример

```javascript
if ( userID == undefined )
{
    throw new Error( "User ID is undefined" );
}
```

## Мини-сводка для Codex

`Error` — объект ошибки/исключения в SP-XML. Ошибка может быть создана через `throw 'text'` или `throw new Error('text')`; эти формы считаются эквивалентными. Ошибка прерывает выполнение и содержит описание и стек вызовов.

# MethodReference

## Краткое описание

`MethodReference` — объект, ссылающийся на метод с определенным именем в заданной библиотеке.

Обычно используется для передачи обработчика сообщения или callback-метода в другие объекты.

## Пример создания MethodReference

```javascript
Application.OnClientMessage = MethodReference( lib_recruit, 'OnClientMessage' );
```

В этом примере в `Application.OnClientMessage` передается ссылка на метод `OnClientMessage` из библиотеки `lib_recruit`.

## Метод

- `Call()`

---

## MethodReference.Call()

Позволяет явно вызвать метод, на который ссылается объект `MethodReference`.

### Синтаксис

```javascript
Call( argsArray )
```

### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `argsArray` | `object` | Да | Массив значений аргументов |

### Возвращаемое значение

В исходном описании возвращаемое значение не указано.

Вероятно, при вызове возвращается результат метода, на который ссылается `MethodReference`, если такой результат есть. Это нужно проверять по конкретному использованию.

### Пример

```javascript
handler.Call( [msg] );
```

В этом примере явно вызывается метод-обработчик, на который указывает `handler`, и ему передается один аргумент `msg`.

## Важные замечания для разработки

- `MethodReference` полезен для регистрации обработчиков событий и сообщений.
- В объект передается не результат вызова метода, а ссылка на метод.
- Для явного вызова используется `Call()`.
- Аргументы передаются массивом.
- При генерации кода важно не перепутать:

```javascript
MethodReference( lib_recruit, 'OnClientMessage' )
```

и прямой вызов:

```javascript
lib_recruit.OnClientMessage( msg )
```

Первый вариант создает ссылку на метод, второй сразу выполняет метод.

## Мини-сводка для Codex

`MethodReference` — ссылка на метод в библиотеке. Используется как обработчик/callback. Создается, например, так: `MethodReference( lib_recruit, 'OnClientMessage' )`. Для явного вызова используется `handler.Call( [arg1, arg2] )`.

# Общая сводка для Codex

## Объекты из этого файла

| Объект | Краткое назначение | Главные особенности |
|---|---|---|
| `ClassObject` | Базовый объект конкретного класса | Используется через конкретные классы платформы |
| `ClassObjectsArray` | Типизированный массив объектов класса | Контролирует тип элементов, поддерживает `Add()` |
| `DataCache` | Thread-safe key-value-кэш | Быстрый поиск по ключу, TTL, `GlobalCache`, `GlobalStorage` |
| `Error` | Ошибка/исключение | Может быть создан через `throw` или `new Error()` |
| `MethodReference` | Ссылка на метод библиотеки | Используется для обработчиков, вызывается через `Call()` |

## Практические правила

1. Не заменять `ClassObjectsArray` на `Array`, если API ожидает типизированную коллекцию объектов.
2. Для временного общего кэша использовать `GlobalCache`.
3. Для хранения значений между запусками приложения использовать `GlobalStorage`.
4. При чтении из `DataCache` проверять `undefined`, так как ключ может отсутствовать или запись могла истечь по `TTL`.
5. Для явных ошибок использовать понятный текст в `throw new Error( "..." )`.
6. `MethodReference` передает ссылку на метод, а не вызывает метод сразу.
7. `MethodReference.Call()` принимает массив аргументов.
