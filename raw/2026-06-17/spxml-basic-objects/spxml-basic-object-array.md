## Назначение

`Array` — базовый объект языка SP-XML, предназначенный для хранения упорядоченного набора элементов.

По набору свойств и методов объект похож на массивы JavaScript, но при разработке под WebTutor/WebSoft HCM важно не считать его полноценным JavaScript-массивом. Использовать следует только документированные возможности SP-XML.

## Состав объекта

В разделе базовых объектов языка рядом с `Array` также упоминаются:

- `Binary`
- `ClassObject`
- `ClassObjectsArray`
- `DataCache`
- `Error`
- `MethodReference`
- `Object`
- `PlainSeq`
- `SafeArray`
- `Stream`
- `String`

Данный файл описывает только объект `Array`.

## Свойства

### `Array.length`

Возвращает количество элементов массива.

#### Синтаксис

```spxml
arrayObj.length
```

#### Результат

```text
integer
```

#### Описание

Свойство `length` содержит текущую длину массива, то есть количество элементов в массиве.

#### Пример

```spxml
var a = new Array();
a.push('fff', 'ggg', 'hhh');

// a.length == 3
```

## Методы

### `Array.AppendUniqueValue()`

Ищет элемент с заданным значением в массиве.

Если элемент уже существует, метод возвращает `false`.

Если элемент не существует, метод добавляет элемент с заданным значением в массив и возвращает `true`.

#### Синтаксис

```spxml
arrayObj.AppendUniqueValue(value)
```

#### Аргументы

| Аргумент | Тип | Обязательность | Описание |
|---|---|---:|---|
| `value` | any | да | Значение, которое необходимо найти или добавить. |

#### Результат

```text
bool
```

#### Логика работы

```text
если value уже есть в массиве:
    вернуть false
иначе:
    добавить value в массив
    вернуть true
```

#### Когда использовать

Метод удобен, когда массив должен хранить только уникальные значения, а разработчику важно знать, был ли элемент реально добавлен.

---

### `Array.ByValueExists()`

Проверяет, существует ли в массиве элемент с заданным значением.

#### Синтаксис

```spxml
arrayObj.ByValueExists(value)
```

#### Аргументы

| Аргумент | Тип | Обязательность | Описание |
|---|---|---:|---|
| `value` | any | да | Значение, наличие которого необходимо проверить. |

#### Результат

```text
bool
```

#### Логика работы

```text
если value есть в массиве:
    вернуть true
иначе:
    вернуть false
```

#### Когда использовать

Метод используется для проверки наличия значения без добавления нового элемента.

---

### `Array.FindIndexBySortedValue()`

Находит первое вхождение элемента с заданным значением в отсортированном массиве и возвращает его порядковый индекс.

Если элемент не найден, метод возвращает `-1`.

Метод предполагает, что массив отсортирован по возрастанию значений. За счет этого он работает существенно быстрее, чем `indexOf()`.

#### Синтаксис

```spxml
arrayObj.FindIndexBySortedValue(value)
arrayObj.FindIndexBySortedValue(value, startIndex)
```

#### Аргументы

| Аргумент | Тип | Обязательность | Описание |
|---|---|---:|---|
| `value` | any | да | Значение, которое необходимо найти. |
| `startIndex` | integer | нет | Индекс, с которого необходимо начинать поиск. По умолчанию поиск ведется с начала массива. |

#### Результат

```text
integer
```

#### Возвращаемые значения

| Ситуация | Результат |
|---|---:|
| Значение найдено | Индекс первого найденного элемента |
| Значение не найдено | `-1` |

#### Важное условие

Массив должен быть отсортирован по возрастанию значений.

Если массив не отсортирован, использовать этот метод нельзя: результат поиска может быть некорректным.

---

### `Array.indexOf()`

Находит первое вхождение элемента с заданным значением и возвращает его порядковый индекс.

Если элемент не найден, метод возвращает `-1`.

#### Синтаксис

```spxml
arrayObj.indexOf(value)
arrayObj.indexOf(value, startIndex)
```

#### Аргументы

| Аргумент | Тип | Обязательность | Описание |
|---|---|---:|---|
| `value` | any | да | Значение, которое необходимо найти. |
| `startIndex` | integer | нет | Индекс, с которого необходимо начинать поиск. По умолчанию поиск ведется с начала массива. |

#### Результат

```text
integer
```

#### Возвращаемые значения

| Ситуация | Результат |
|---|---:|
| Значение найдено | Индекс первого найденного элемента |
| Значение не найдено | `-1` |

#### Когда использовать

`indexOf()` подходит для обычного поиска по массиву, когда массив не обязательно отсортирован.

Для больших отсортированных массивов предпочтительнее использовать `FindIndexBySortedValue()`.

---

### `Array.ObtainBySortedValue()`

Ищет элемент с заданным значением в отсортированном массиве.

Если элемент существует, метод возвращает индекс найденного элемента.

Если элемент не существует, метод добавляет элемент с заданным значением в нужную позицию так, чтобы сохранилась сортировка, и возвращает индекс добавленного элемента.

#### Синтаксис

```spxml
arrayObj.ObtainBySortedValue(value)
```

#### Аргументы

| Аргумент | Тип | Обязательность | Описание |
|---|---|---:|---|
| `value` | any | да | Значение, которое необходимо найти или добавить. |

#### Результат

```text
integer
```

#### Логика работы

```text
если value уже есть в отсортированном массиве:
    вернуть индекс найденного элемента
иначе:
    вставить value в позицию, сохраняющую сортировку
    вернуть индекс добавленного элемента
```

#### Важное условие

Метод предполагает, что массив уже отсортирован по значению.

Если массив не отсортирован, результат работы метода не определен.

#### Когда использовать

Метод полезен для больших массивов, где нужно одновременно:

- быстро найти значение;
- добавить значение при отсутствии;
- сохранить сортировку массива.

---

### `Array.ObtainByValue()`

Ищет элемент с заданным значением в массиве.

Если элемент существует, метод возвращает индекс найденного элемента.

Если элемент не существует, метод добавляет элемент с заданным значением в массив и возвращает индекс добавленного элемента.

#### Синтаксис

```spxml
arrayObj.ObtainByValue(value)
```

#### Аргументы

| Аргумент | Тип | Обязательность | Описание |
|---|---|---:|---|
| `value` | any | да | Значение, которое необходимо найти или добавить. |

#### Результат

```text
integer
```

#### Логика работы

```text
если value уже есть в массиве:
    вернуть индекс найденного элемента
иначе:
    добавить value в массив
    вернуть индекс добавленного элемента
```

#### Отличие от `AppendUniqueValue()`

| Метод | Если значение уже есть | Если значения нет |
|---|---|---|
| `AppendUniqueValue(value)` | Возвращает `false` | Добавляет значение и возвращает `true` |
| `ObtainByValue(value)` | Возвращает индекс найденного элемента | Добавляет значение и возвращает индекс добавленного элемента |

---

### `Array.join()`

Преобразует все элементы массива в объект `String` и соединяет их в одну строку.

#### Синтаксис

```spxml
arrayObj.join()
arrayObj.join(separator)
```

#### Аргументы

| Аргумент | Тип | Обязательность | Описание |
|---|---|---:|---|
| `separator` | String | нет | Разделитель между элементами массива. |

#### Результат

```text
String
```

#### Описание

Метод проходит по элементам массива, преобразует каждый элемент в строку и соединяет элементы с использованием указанного разделителя.

#### Пример

```spxml
var arr = new Array();
arr.push('one', 'two', 'three');

var str = arr.join(', ');
// str == 'one, two, three'
```

---

### `Array.push()`

Добавляет один или несколько элементов в конец массива.

Элементы добавляются, начиная с текущей длины массива `length`.

Метод возвращает новую длину массива.

#### Синтаксис

```spxml
arrayObj.push(arg1)
arrayObj.push(arg1, arg2)
arrayObj.push(arg1, arg2, ...)
```

#### Аргументы

| Аргумент | Тип | Обязательность | Описание |
|---|---|---:|---|
| `arg1` | any | да | Первый элемент, который нужно добавить в массив. |
| `arg2` и последующие | any | нет | Дополнительные элементы, которые нужно добавить в массив. |

#### Результат

```text
integer
```

#### Пример

```spxml
var a = new Array();

var len = a.push('fff', 'ggg', 'hhh');
// len == 3
// a.length == 3
```

---

### `Array.splice()`

Удаляет часть массива.

#### Синтаксис

```spxml
arrayObj.splice(arg1, arg2)
```

#### Аргументы

| Аргумент | Тип | Обязательность | Описание |
|---|---|---:|---|
| `arg1` | integer | да | Индекс в массиве, с которого нужно начинать удаление. |
| `arg2` | integer | да | Количество элементов, которое нужно удалить, начиная с индекса `arg1`. |

#### Описание

Метод удаляет из массива `arg2` элементов, начиная с позиции `arg1`.

#### Пример

```spxml
var a = new Array();
a.push('a', 'b', 'c', 'd');

a.splice(1, 2);
// удалены элементы с индексами 1 и 2: 'b', 'c'
```

## Важные замечания для разработки

1. SP-XML серверный язык WebTutor/WebSoft HCM похож по синтаксису на JavaScript, но не является JavaScript.
2. Нельзя автоматически переносить поведение JavaScript-массивов на `Array` в SP-XML, если конкретный метод или поведение не описаны в документации.
3. Методы `FindIndexBySortedValue()` и `ObtainBySortedValue()` требуют, чтобы массив был отсортирован по возрастанию значений.
4. Если массив не отсортирован, результат работы `ObtainBySortedValue()` не определен.
5. Для обычного поиска по неотсортированному массиву следует использовать `indexOf()` или `ByValueExists()`.
6. Для добавления уникального значения с булевым результатом подходит `AppendUniqueValue()`.
7. Для получения индекса существующего или добавленного значения подходит `ObtainByValue()`.
8. Для больших отсортированных массивов предпочтительнее использовать методы с `SortedValue`, так как они рассчитаны на более быстрый поиск и вставку.

## Краткая сводка для Codex

```text
Array в SP-XML — базовый объект для хранения упорядоченного набора элементов.

Свойство:
- length: integer — количество элементов массива.

Методы:
- AppendUniqueValue(value): bool — если value уже есть, возвращает false; иначе добавляет value и возвращает true.
- ByValueExists(value): bool — проверяет наличие value в массиве.
- FindIndexBySortedValue(value[, startIndex]): integer — ищет value в отсортированном массиве, возвращает индекс или -1.
- indexOf(value[, startIndex]): integer — ищет value в массиве, возвращает индекс или -1.
- ObtainBySortedValue(value): integer — ищет value в отсортированном массиве; если нет, вставляет с сохранением сортировки; возвращает индекс.
- ObtainByValue(value): integer — ищет value; если нет, добавляет; возвращает индекс найденного или добавленного элемента.
- join([separator]): String — соединяет элементы массива в строку через separator.
- push(arg1[, arg2, ...]): integer — добавляет элементы в конец массива, возвращает новую длину.
- splice(arg1, arg2): удаляет arg2 элементов начиная с индекса arg1.

Важно:
- SP-XML не является JavaScript, даже если синтаксис похож.
- Методы SortedValue использовать только для массивов, отсортированных по возрастанию.
- Для неотсортированного массива использовать indexOf(), ByValueExists(), ObtainByValue() или AppendUniqueValue().
```
