## Назначение

`Binary` — базовый объект SP-XML, предназначенный для хранения бинарных данных, в том числе бинарных данных большого размера.

Исторически объект активно использовался в случаях, когда строки не могли содержать бинарные данные, а также для задач, где требовалась многократная запись данных небольшими фрагментами в один большой результат.

Типичный старый сценарий использования — генерация большого HTML-документа. Добавление множества небольших строковых фрагментов к обычной строке в цикле могло работать существенно медленнее, чем запись этих фрагментов в `Binary` с последующим преобразованием результата в строку.

## Статус и рекомендации по использованию

`Binary` считается устаревшим объектом.

В современной разработке вместо него предпочтительно использовать отдельный объект `WriteBuffStream`, если задача связана с буферизованной записью большого количества данных.

Для поддержки старого кода важно понимать методы `Binary`, но при написании нового кода Codex не должен автоматически выбирать `Binary`, если есть возможность использовать более актуальный механизм.

## Свойства

### `Binary.Size`

Размер содержимого бинарного объекта в байтах.

**Тип атрибута:** `Integer`

**Возвращаемое значение:**

| Тип | Описание |
|---|---|
| `Integer` | Размер текущего содержимого объекта в байтах. |

**Пример смысла свойства:**

```javascript
// Условный пример: проверка, есть ли данные в бинарном объекте
if ( binaryObj.Size > 0 )
{
    // объект содержит данные
}
```

### `Binary.WriteStream`

Поток, в который можно писать данные любыми функциями, работающими с потоками.

Данные, записанные в поток `WriteStream`, попадают в текущий объект `Binary`.

**Тип:** потоковый объект.

**Назначение:** запись данных в `Binary` через API, которые умеют работать со stream-объектами.

**Важное замечание:** если задача состоит именно в потоковой или буферизованной генерации большого результата, при новом проектировании следует проверить возможность использования `WriteBuffStream`, так как `Binary` является устаревшим.

## Методы

### `Binary.AssignBinary()`

Копирует данные из заданного бинарного объекта в текущий бинарный объект.

#### Синтаксис

```javascript
AssignBinary( obj )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `obj` | `Binary` | Да | Исходный бинарный объект, данные из которого копируются в текущий объект. |

#### Возвращаемое значение

| Тип | Описание |
|---|---|
| `undefined` | Метод изменяет текущий объект и не возвращает полезного значения. |

#### Поведение

Метод заменяет содержимое текущего объекта содержимым другого бинарного объекта.

Использовать этот метод следует, когда нужно именно скопировать данные, а не добавить их в конец. Для добавления используется [`AppendBinary()`](#binaryappendbinary).

---

### `Binary.AssignStr()`

Копирует в текущий бинарный объект данные из заданной строки.

#### Синтаксис

```javascript
AssignStr( str )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `String` | Да | Строка, данные которой копируются в текущий бинарный объект. |

#### Возвращаемое значение

| Тип | Описание |
|---|---|
| `undefined` | Метод изменяет текущий объект и не возвращает полезного значения. |

#### Поведение

Метод заменяет текущее содержимое объекта данными из строки.

Если нужно добавить строку к уже существующим данным, используется [`AppendStr()`](#binaryappendstr).

---

### `Binary.AppendBinary()`

Добавляет данные из заданного бинарного объекта в текущий бинарный объект.

#### Синтаксис

```javascript
AppendBinary( arg1 )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `arg1` | `Binary` | Да | Бинарный объект, данные из которого следует добавить в текущий объект. |

#### Возвращаемое значение

| Тип | Описание |
|---|---|
| `undefined` | Метод изменяет текущий объект и не возвращает полезного значения. |

#### Поведение

Метод не очищает текущий объект, а добавляет данные в конец уже существующего содержимого.

Отличие от [`AssignBinary()`](#binaryassignbinary):

| Метод | Поведение |
|---|---|
| `AssignBinary(obj)` | Заменяет содержимое текущего объекта данными из `obj`. |
| `AppendBinary(obj)` | Добавляет данные из `obj` в конец текущего объекта. |

---

### `Binary.AppendStr()`

Добавляет в текущий бинарный объект данные из заданной строки.

#### Синтаксис

```javascript
AppendStr( str )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `str` | `String` | Да | Строка, данные которой следует добавить в текущий бинарный объект. |

#### Возвращаемое значение

| Тип | Описание |
|---|---|
| `undefined` | Метод изменяет текущий объект и не возвращает полезного значения. |

#### Поведение

Метод добавляет строковые данные в конец текущего бинарного объекта.

Исторически этот метод мог использоваться для эффективного накопления большого текстового результата из множества маленьких фрагментов.

Отличие от [`AssignStr()`](#binaryassignstr):

| Метод | Поведение |
|---|---|
| `AssignStr(str)` | Заменяет содержимое текущего объекта строкой `str`. |
| `AppendStr(str)` | Добавляет строку `str` к уже существующему содержимому. |

---

### `Binary.GetStr()`

Возвращает бинарные данные из объекта в виде строки, содержащей массив байт.

#### Синтаксис

```javascript
GetStr()
```

#### Аргументы

Метод не принимает аргументы.

#### Возвращаемое значение

| Тип | Описание |
|---|---|
| `String` | Строка, содержащая данные бинарного объекта как массив байт. |

#### Поведение

Метод используется для получения содержимого `Binary` в строковом виде.

Это особенно важно для старого кода, где `Binary` применялся как промежуточный буфер при генерации больших строковых результатов.

---

### `Binary.LoadFromUrl()`

Загружает в текущий бинарный объект данные из определенного URL.

#### Синтаксис

```javascript
LoadFromUrl( url )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `url` | `String` | Да | URL, из которого нужно загрузить данные. |

#### Возвращаемое значение

| Тип | Описание |
|---|---|
| `undefined` | Метод изменяет текущий объект и не возвращает полезного значения. |

#### Поведение

Метод загружает данные по указанному URL и помещает их в текущий объект `Binary`.

Контекст доступности URL зависит от среды выполнения SP-XML/WebTutor. В старом коде такие URL могут указывать на локальные, серверные или платформенные ресурсы.

---

### `Binary.PutToUrl()`

Сохраняет данные из текущего бинарного объекта в заданный URL.

#### Синтаксис

```javascript
PutToUrl( url )
```

#### Аргументы

| Аргумент | Тип | Обязательный | Описание |
|---|---|---:|---|
| `url` | `String` | Да | URL, по которому нужно сохранить данные текущего бинарного объекта. |

#### Возвращаемое значение

| Тип | Описание |
|---|---|
| `undefined` | Метод выполняет сохранение и не возвращает полезного значения. |

#### Поведение

Метод записывает содержимое текущего объекта `Binary` в указанный URL.

Контекст записи зависит от типа URL и прав текущей среды выполнения.

## Краткая сводка для Codex

`Binary` — устаревший базовый объект SP-XML для хранения и накопления бинарных данных.

Основные свойства и методы:

| Элемент | Назначение |
|---|---|
| `Size` | Размер содержимого в байтах. |
| `WriteStream` | Поток для записи данных в объект. |
| `AssignBinary(obj)` | Заменить содержимое данными из другого `Binary`. |
| `AssignStr(str)` | Заменить содержимое данными из строки. |
| `AppendBinary(obj)` | Добавить данные из другого `Binary` в конец. |
| `AppendStr(str)` | Добавить строку в конец бинарного объекта. |
| `GetStr()` | Получить содержимое как строку байт. |
| `LoadFromUrl(url)` | Загрузить данные из URL. |
| `PutToUrl(url)` | Сохранить данные в URL. |

При анализе старого кода WebTutor/SP-XML `Binary` может встречаться как буфер для генерации больших строк или хранения бинарных данных.

При генерации нового кода не следует выбирать `Binary` автоматически. Если задача связана с эффективной потоковой записью или накоплением большого результата, сначала нужно проверить использование более современного объекта `WriteBuffStream`.
