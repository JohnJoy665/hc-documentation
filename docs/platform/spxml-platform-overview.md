# SP-XML: назначение, состав и данные

## Кратко

SP-XML в источниках docs.datex.ru описан как платформа быстрой разработки приложений. В контексте WebTutor / WebSoft HCM эти материалы важны как базовое описание платформенных принципов: XML-документы, базы данных, схемы `x-local://` и `x-db-obj://`, серверная часть `xHttp.exe`, конфигурационные файлы и разделение данных от интерфейса.

## Состав платформы

В состав ядра платформы входят:

- `SpXml.exe` - исполняемый файл пользовательского интерфейса;
- `xHttp.exe` - исполняемый файл серверной части;
- модуль `app` - папка с файлами, используемыми ядром;
- стандартные модули `base1` и `base2` обычно используются в приложении, но источник отмечает, что это не обязательно.

## Принципы построения приложения

Источник фиксирует два принципа:

- открытость кода: формы данных, экранные формы и код вне ядра платформы находятся в файлах `.xml`, `.htm` или `.js`; возможны исключения, когда модули поставляются в зашифрованном виде или часть кода хранится в базе данных;
- отделение данных от представления: структуры данных и экранное представление описываются разными формами, а бизнес-логика работает с данными.

## Базы данных и документы

Основные данные хранятся в XML-документах внутри баз данных.

Для клиент-серверного приложения источник называет как минимум две базы:

- `data` - основная серверная база;
- `data_local` - пользовательская локальная база.

Для корзины может использоваться отдельная база:

- `trash`.

## Статические и объектные документы

В базе есть два вида документов:

- статические документы;
- объектные документы.

Статические документы имеют заранее заданный относительный путь внутри базы и доступны через `x-local://`.

Пример из источника:

```text
x-local://data/static/global_settings.xml
```

Объектные документы содержат основной объем данных приложения. В новой модели для обращения к объекту используется схема `x-db-obj://`.

Пример из источника:

```text
x-db-obj://data/person/0x41D258E8F25C328A.xml
```

## Важный нюанс для WebTutor

Источник отдельно указывает, что WebTutor использует старую объектную модель. В старой модели объектные документы базы независимо от типа использовали единое пространство 64-битных ID и схему `x-local://` для адресации.

Пример из источника:

```text
x-local://data/objects/0x41D258E8F25C32/8A.xml
```

## Конфигурационные файлы

При запуске ядро платформы считывает конфигурационный файл рядом с исполняемым файлом:

- `SpXml.ini` - для рабочего места;
- `xHttp.ini` - для сервера.

Файл состоит из строк вида:

```text
<имя параметра>: <значение параметра>
```

Для временного комментария параметра в начало строки обычно добавляют `#`.

Источник отмечает, что `SpXml.ini` и `xHttp.ini` входят в состав дистрибутива и могут затираться при обновлении. Пользовательские параметры имеет смысл добавлять в:

- `SpXml2.ini`;
- `xHttp1.ini`.

Основной параметр:

- `APPLICATION-MODULE` - имя главного модуля приложения.

## Проверенные факты из источников

- SP-XML хранит основные данные как XML-документы внутри баз данных.
- `xHttp.exe` реализует серверную часть.
- `x-local://` используется для статических документов и, в старой модели WebTutor, для объектных документов.
- `x-db-obj://` используется для адресации объектных документов в новой модели.
- Конфигурационные файлы сервера и рабочего места отличаются.
