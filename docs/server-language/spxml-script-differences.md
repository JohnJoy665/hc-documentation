# SP-XML Script и внутренний серверный язык WebTutor

## Кратко

Источник docs.datex.ru прямо указывает, что SP-XML Script и ECMAScript-подобные среды являются разными языками, хотя похожи синтаксически и имеют отдельные похожие объекты и функции.

Для WebTutor / WebSoft HCM это подтверждает правило репозитория: серверный код WebTutor нельзя автоматически писать как код для Node.js, браузера или иной внешней среды. Нужно опираться на платформенные функции и локальную документацию.

## Типы данных

Из источника:

- SP-XML Script по идеологии типов ближе к VBScript.
- Строка и дата являются скалярными значениями, а не объектами.
- `integer` и `real` - разные типы.
- `string` и `date` также поддерживаются как объекты для облегчения миграции кода.
- Строки бинарно-совместимы: в них можно хранить текст и бинарные данные.
- Текст хранится в кодировке приложения, обычно UTF-8.

Практический нюанс по строкам:

- `StrLen()` возвращает длину в байтах;
- `StrCharCount()` возвращает длину в символах;
- для получения первого символа нужно использовать функцию уровня символов, а не байтовый срез.

## Даты

Дата и дата со временем не различаются как отдельные типы. Дата может содержать время или не содержать его.

Источник отмечает, что дата и время хранятся структурировано: год, месяц, день и другие части отдельно, а часовой пояс хранится отдельно.

## Объекты и скалярные значения

Объект может использоваться как объект или как скалярное значение, если объект это поддерживает.

Пример из источника: поле XML-документа является объектом, но имеет скалярное значение. При сравнении, арифметике и преобразовании к строке объект преобразуется к скалярному значению.

## Сравнение

В SP-XML Script используется только строгое сравнение.

Разные значения:

```text
undefined
null
''
0
false
```

## Массивы и for-in

Помимо стандартного массива существуют абстрактные массивы.

`for-in`:

- по стандартному объекту обходит имена свойств;
- по абстрактным массивам работает как foreach и обходит значения.

## Функции и библиотеки

Источник указывает, что SP-XML Script использует концепцию библиотек с разделением локальных и глобальных переменных внутри библиотеки. Использование функций внутри кода возможно, но не является рекомендуемой техникой.

## Связь с агентами WebTutor

В материале по агентам сервера указано:

- файлы агентов могут иметь расширение `.js`;
- код агента пишется без `<% ... %>`;
- расширение `.js` не означает, что можно применять Node.js API, npm-пакеты, браузерные API или современные ECMAScript-паттерны без подтверждения в документации WebTutor.
