# Агенты сервера WebTutor / WebSoft HCM

## Кратко

Агенты сервера - механизм WebTutor / WebSoft HCM для автоматического запуска процедур по расписанию. Они используются для серверных и фоновых операций, в том числе длительных процедур.

## Где находятся агенты

По исходной заметке список агентов расположен в административном интерфейсе:

```text
Дизайнер -> Бизнес-логика -> Агенты сервера
```

## Отличие от HTML-шаблонов

В серверных HTML-шаблонах код обычно помещается в блоки:

```html
<% ... %>
```

В файлах серверных агентов код пишется напрямую, без оборачивания в `<% ... %>`.

## Расширение файлов

Файлы агентов могут иметь расширение:

```text
.js
```

Это не означает, что код агента можно писать как код для Node.js или браузера. Код агента относится к внутреннему серверному языку WebTutor / SP-XML Script и должен опираться на платформенные функции.

## Пример сценария агента

Исходная заметка содержит пример, который:

1. Задает идентификатор `testId`.
2. Через `XQuery(...)` ищет заявку в каталоге `requests`.
3. Проверяет `object_id` и `status_id`.
4. Открывает документ через `OpenDoc(UrlFromDocID(iReqId))`.
5. Получает `TopElem`.
6. Меняет статус заявки на `close`.
7. Ищет запись в `workflow_log_entrys` через `ArrayOptFindByKey(...)`.
8. Создает уведомление через `tools.create_notification(...)`.
9. Сохраняет документ через `oRequestDoc.Save()`.

## Использованные функции и объекты

В примере встречаются:

- `XQuery(...)`
- `ArrayOptFirstElem(...)`
- `OptInt(...)`
- `OpenDoc(...)`
- `UrlFromDocID(...)`
- `TopElem`
- `ArrayOptFindByKey(...)`
- `tools.create_notification(...)`
- `Save()`
- `alert(...)`

`alert(...)` в контексте агента описан как диагностический вывод выполнения серверного агента, а не браузерный `window.alert`.

## Практические правила

- Не оборачивать код агента в `<% ... %>`.
- Не применять Node.js API, браузерные API, npm-пакеты, `fetch`, `Promise`, `async/await` без подтверждения в локальной документации WebTutor.
- Сохранять различие между серверным HTML-шаблоном и агентом.
- При изменении документов использовать сохранение через `Save()`, если документ был изменен и источник подтверждает такой паттерн.
- При неизвестных функциях, объектах, каталогах и полях запрашивать дополнительные материалы.
