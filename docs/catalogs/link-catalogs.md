# Каталоги-связки

## Кратко

Каталоги-связки в WebTutor предназначены для облегчения и ускорения работы со связанными каталогами.

По источнику каталог-связка - это представление (`view`), сформированное стандартными средствами путем объединения двух или более стандартных каталогов по определенному набору полей.

## Где посмотреть

Перечень доступных каталогов-связок можно увидеть на вкладке **Настройка отчета** карточки **Настраиваемый отчет**.

В исходном материале в этом месте была иллюстрация интерфейса, но она не была передана вместе с текстом.

## Состав каталогов-связок

Состав каталогов-связок постоянно пополняется. Источник фиксирует набор для версии `331 (2021.2.3.331)`.

Примеры каталогов-связок из источника:

- `blog_author`;
- `budget_object`;
- `group_collaborator`;
- `event_group`;
- `event_lector`;
- `event_object_resource`;
- `event_collaborator`;
- `event_phase`;
- `knowledge_object`;
- `object_expert`;
- `tagged_object`;

- `education_plan_collaborator`;

- `education_method_lector`;
- `func_manager`.

Полный список сохранен в raw-источнике.

## Использование в серверном коде WebTutor

Каталоги-связки используются в `XQuery` так же, как обычные каталоги.

Состав полей конкретного каталога-связки нужно проверять через панель запросов или актуальную документацию конкретной версии.

Примеры из источника:

```js
XQuery( 'for $elem in group_collaborators return $elem' );
```

```js
XQuery( 'for $elem in group_collaborators where $elem/collaborator_id='+curUserID+' and $elem/group_id='+fldProjectGroup.object_id+' return $elem' );
```

Эти примеры относятся к серверному коду WebTutor. Блоки кода в источнике были размечены как `javascript`, но это не делает их обычным JavaScript.

## Важные нюансы

- Доступный набор каталогов-связок зависит от версии WebTutor.
- В источнике код каталога в списке указан как `group_collaborator`, а в примерах XQuery используется `group_collaborators`. Это различие сохранено и требует проверки.
- В исходном материале код `submission_recipient}` был нормализован в Markdown как `submission_recipient`, потому что закрывающая фигурная скобка выглядит как опечатка.

## Проверенные факты

- Каталог-связка является `view` поверх двух или более стандартных каталогов.
- Каталог-связка используется в `XQuery` как обычный каталог.
- Источник относится к версии `331 (2021.2.3.331)` для состава каталогов-связок.
