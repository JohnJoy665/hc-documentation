# Варианты установки WebSoft HCM

Раздел «Установка» базы знаний перечисляет несколько способов развёртывания WebSoft HCM. Выбор зависит от операционной системы, корпоративных стандартов и предпочтений администратора. Ниже приведён обзор доступных вариантов.

## Установка через Docker

Для быстрого развёртывания можно воспользоваться подготовленными Docker‑образами. WebSoft предоставляет отдельные образы для:

* **Сервера WebSoft HCM** — основной портал. Разворачивается командой `docker run` с указанием порта, путей к каталогам `data` и `logs` и переменных окружения для базы данных. Подробная процедура описана в файле 03‑install‑via‑docker.md【182237728321055†screenshot】.
* **WebSoftIDM** — модуль аутентификации/авторизации. Может быть запущен отдельно или вместе с HCM. Настройка выполняется с помощью переменных окружения и `docker-compose`【687277790184942†screenshot】.
* **Сервера записи VClass** — сервис для записи вебинаров и занятий. Распространяется в виде Docker‑образа, который разворачивается на Linux‑сервере. Требует наличия FFmpeg и ресурсов CPU/GPU【363971634706873†screenshot】.
* **Inventa** — модуль интеллектуального поиска. Поставляется как контейнер и подключается к WebSoft HCM через REST‑API【848555681754747†screenshot】.
* **Telemetry (Signoz)** — платформа телеметрии для мониторинга производительности. Устанавливается с помощью `docker-compose` и позволяет собирать метрики, логи и трассировки【452867952766753†screenshot】.

Использование Docker подходит для тестовых стендов и ситуаций, когда хочется быстро развернуть систему без сложной настройки окружения. Однако для высоких нагрузок рекомендуется использовать «нативную» установку на Linux или Windows.

## Установка на Linux

Развёртывание на Linux требует установки зависимостей (Node.js, .NET Runtime, сервер базы данных) и создания системных пользователей. Поддерживаются дистрибутивы Alt Linux и Ubuntu. Процесс описан в отдельной инструкции (см. 04‑install‑on‑linux.md).

## Установка на Windows

Для Windows предоставляется инсталлятор (MSI) с графическим мастером. В процессе установки выбираются компоненты (WebSoft HCM, WebSoftIDM), настраиваются каталоги и создаются службы IIS. Дополнительные сведения приведены в файле 05‑install‑on‑windows.md【630080194694049†screenshot】.

## Создание тестового стенда

Раздел «Установка» также содержит статьи по созданию тестового стенда и его актуализации. Тестовый стенд позволяет протестировать обновления и изменения перед переносом в боевую среду. Дополнительные инструкции по развёртыванию тестовой платформы через Docker приведены в соответствующей статье (см. раздел Docker)【392378813991336†screenshot】.

## Установка демонстрационного контента

Для знакомства с функциональностью платформы можно установить демо‑контент. В архив включены примерные курсы, тесты и настройки. Установка выполняется через административный интерфейс WebSoft HCM или копированием файлов на сервер, как описано в одном из разделов «Установка»【15119285887854†screenshot】.

Выбор варианта зависит от целей и инфраструктуры. Docker‑установка предоставляет максимальную портативность, а инсталляторы для Linux/Windows — гибкость и возможность тонкой настройки.
