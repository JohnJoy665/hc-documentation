# Запуск WebSoft HCM как служба

Чтобы портал WebSoft HCM автоматически запускался при загрузке сервера и работал без участия пользователя, его можно установить как службу Windows. В официальной документации описаны два способа: через встроенный инструмент **x‑shell** и команду `sc create`.

## Создание службы через x‑shell

1. Убедитесь, что WebSoft HCM уже установлен. Запустите **x‑shell**, перейдя в каталог `WebSoftServer` и выполнив:

   ```bash
   x-shell set type service
   ```

   Эта команда создаст службу Windows под названием `WebSoftHCMCorpServer` и зарегистрирует её в системе【651732142429040†screenshot】.

2. Откройте консоль **Services (services.msc)**. Найдите созданную службу и в её свойствах измените:
   * **Startup Type** на `Automatic`, чтобы служба запускалась при старте ОС【205663700573740†screenshot】.
   * На вкладке **Recovery** (Восстановление) для **First Failure** и **Second Failure** выберите `Restart the service`, чтобы служба перезапускалась после сбоев【205663700573740†screenshot】.

## Создание службы вручную (sc create)

Альтернативный способ — создать службу вручную с помощью команды `sc` в командной строке от имени администратора:

```cmd
sc create WebSoftHCMCorpServer binPath= "C:\\WebSoftServer\\xhttp_64.exe /s" DisplayName= "WebSoftHCMCorpServer" type= own start= auto
```

После выполнения служба появится в списке служб и будет настроена на автоматический запуск. При необходимости отредактируйте параметры через `services.msc`【589690356369803†screenshot】.

## Настройка учётной записи

По умолчанию служба запускается под системной учётной записью **Local System**. Если WebSoft HCM подключается к базе данных с использованием Windows‑аутентификации, рекомендуется выбрать доменную учётную запись и предоставить ей права:

* Доступ на чтение/запись к каталогу `WebSoftServer` и подкаталогам (logs, applications, webtutor и индексы)【860668616519084†screenshot】.
* Права `db_owner` на соответствующую базу данных в SQL Server или PostgreSQL.

Измените пользователя на вкладке **Log On** в свойствах службы и введите учетные данные【860668616519084†screenshot】.

## Запуск и управление

После создания службы нажмите «Start/Запустить» в окне Services или выполните:

```cmd
net start WebSoftHCMCorpServer
```

Чтобы остановить службу, используйте `net stop WebSoftHCMCorpServer`. Любые изменения в конфигурационных файлах требуют перезапуска службы.

## Рекомендации

* Проверьте, что служба корректно запускается и слушает указанные порты (80/443). Если возникают ошибки, просмотрите журнал сервиса в папке `logs` и события Windows (Event Viewer).
* При обновлении версии WebSoft HCM временно остановите службу, замените файлы и снова запустите её. Соблюдайте инструкции из раздела 11 «Обновление».
* Настройка служб в Linux выполняется через systemd; см. раздел 04 «Установка на Linux».
