В рамках работ был подготовлен отдельный локальный учебный контур WebSoft HCM для практики работы с PostgreSQL, без изменения основной локальной установки. Исходная рабочая папка портала находится по пути C:\WebSoft\WebSoftServer. Эту папку напрямую не изменяли, чтобы не повредить текущий локальный стенд. Для экспериментов была создана отдельная копия сервера по пути C:\WebSoft\WebSoftServer_PG_LOCAL_STUDY.

Перед подключением PostgreSQL была проверена локальная среда. На компьютере уже был установлен и запущен PostgreSQL, поэтому новую СУБД устанавливать не потребовалось. Проверка показала, что порт 5432 занят процессом postgres.exe, а служба PostgreSQL работает как postgresql-x64-16. Также был найден клиент psql по пути C:\Program Files\PostgreSQL\16\bin\psql.exe, версия psql (PostgreSQL) 16.2. В списке существующих баз были обнаружены postgres, salcal, template0 и template1. Для учебного WebSoft-стенда существующие базы не использовались.

Для нового учебного контура была создана отдельная PostgreSQL-база websoft_hcm_local_study, отдельный пользователь websoft_hcm_local_study_user и пароль websoft_hcm_local_study_pass. Подключение к базе выполняется по адресу localhost, порт 5432. Эти параметры были выбраны специально так, чтобы по названию было понятно, что база относится именно к локальному учебному стенду WebSoft HCM и не связана с другими рабочими или тестовыми базами.

В копии портала C:\WebSoft\WebSoftServer_PG_LOCAL_STUDY был изменён тип базы данных через x-shell. Изначально была попытка использовать значение pgsql, однако x-shell сообщил, что допустимыми значениями являются xml, mssql, postgresql и oracle. После этого был выбран корректный тип postgresql. Команда db set type postgresql была выполнена в x-shell. На запросы системы были указаны параметры подключения: Server — localhost, Database — websoft_hcm_local_study, Username — websoft_hcm_local_study_user, Password — websoft_hcm_local_study_pass. После подтверждения изменений x-shell создал безопасную копию файла spxml_unibridge_config.xml и сообщил: Database Type postgresql set, xhttp restart recommended.

После перезапуска xhttp_64.exe из папки копии была выполнена инициализация структуры базы командой db init. В процессе появлялись предупреждения и сообщения об ошибке загрузки внутреннего документа x-local://wtv/wtv_view_main.xml, однако итоговый статус был успешным: Task 'InitializeDatabase' succeeded. После этого в DBeaver были проверены таблицы в базе websoft_hcm_local_study; структура WebSoft появилась, но таблицы были пустыми.

Так как требовались исходные демонстрационные данные, была выполнена миграция из XML-хранилища, которое было скопировано вместе с локальной папкой WebSoft. Миграция была запущена командой db migrate from xml и завершилась успешно: File System Migration completed. Overall Docs:21627. Migration Errors:0. Task 'MigrateFileSystemTables' succeeded. Таким образом, в PostgreSQL было перенесено 21 627 документов, ошибок миграции не было.

После переноса данных была выполнена перестройка каталогов и индексов командой rctg start. В процессе перестройки появились сообщения об отсутствующей библиотеке программного кода libAnalytics, а также несколько ошибок ProcessDocInCatalog() failed по ресурсам. Эти ошибки не остановили процесс. В конце задача завершилась сообщениями Indexes rebuilt, Task finished и повторным Indexes rebuilt. То есть перестройка индексов была завершена.

Итоговое состояние: создан отдельный учебный локальный портал WebSoft HCM, подключённый к PostgreSQL, с перенесёнными демонстрационными данными из исходной XML-базы. Основная локальная папка C:\WebSoft\WebSoftServer не изменялась. Все действия выполнялись в копии C:\WebSoft\WebSoftServer_PG_LOCAL_STUDY. Учебная база доступна в DBeaver со следующими параметрами: тип подключения PostgreSQL, Host — localhost, Port — 5432, Database — websoft_hcm_local_study, User — websoft_hcm_local_study_user, Password — websoft_hcm_local_study_pass.

Для запуска учебного портала нужно запускать сервер именно из папки копии: перейти в C:\WebSoft\WebSoftServer_PG_LOCAL_STUDY и запустить .\xhttp_64.exe. Для входа в портал можно использовать демонстрационную учётную запись user1 с паролем user1.

Результат работы: локальный учебный стенд WebSoft HCM переведён на отдельную PostgreSQL-базу websoft_hcm_local_study, данные из исходного демо-хранилища перенесены, таблицы доступны для изучения и написания SQL-запросов через DBeaver.
