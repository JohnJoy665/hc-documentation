# Глоссарий

- `Request` - объект Web-сервера, доступный при обработке HTTP-запроса в серверном коде WebTutor.
- `Request.Query` - параметры URL и POST-формы по материалам API routing.
- `Request.Body` - тело HTTP-запроса строкой по материалам API routing.
- `Request.AuthUserID` - идентификатор фактически определенного пользователя; использовался в диагностике авторизации.
- `Request.Session` - сессия запроса; из нее в материалах читали `Env`.
- `Request.DropSession` - метод сброса сессии, встречается в logout-файлах.
- `Response` - объект HTTP-ответа.
- `Response.Write` - запись строки в HTTP-ответ.
- `Response.AddHeader` - добавление HTTP-заголовка; использовалось для диагностического заголовка logout.
- `tools_web.user_init` - функция инициализации пользователя по `Request` и параметрам запроса.
- `tools_web.set_cookie_auth` - функция работы с cookie-авторизацией; встречается в logout-файлах.
- `Env.curUserID` - идентификатор текущего пользователя в окружении сессии.
- `Env.curUser` - текущий пользователь в окружении сессии.
- `x-auth-id` - cookie/тикет авторизации, очищается встроенным logout и настраивается через `xhttp_config.json`.
- `user_login` - cookie, очищалась встроенным logout в проверенном кейсе.
- `user_password` - cookie, очищалась встроенным logout в проверенном кейсе.
- `Authorization: Basic` - HTTP-заголовок Basic Auth; не очищается через `Set-Cookie`.
- `LogoutPath` - параметр `xhttp_config.json`, задающий путь logout.
- `RedirectLoggedOffUri` - параметр `xhttp_config.json`, задающий страницу после выхода.
- `loggedoff.html` - страница после logout; в кейсе помогла выявить Basic/NTLM тип авторизации узла.
- `portal_auth_type` - тип авторизации портала/узла, встречался как `curHost.portal_auth_type`.
- `cookie / Form-Based` - тип авторизации узла, решивший проблему повторного входа после logout в локальном кейсе.
- `Basic Auth` - HTTP Basic-аутентификация, при которой браузер может кэшировать credentials.
- `x-local://` - внутренняя схема WebTutor для обращения к файлам из серверного кода; не является клиентским URL для браузера.
- `_wt` - короткий URL-префикс портала; требует дополнительной документации.
- `router.html` - серверный HTML-хендлер, используемый как единая точка входа backend API.
- `Server.Execute` - выполнение серверного include/файла.
- `AppDirectoryPath` - функция получения пути приложения, встречается в вызовах `Server.Execute(AppDirectoryPath() + ...)`.
- `OpenCodeLib` - открытие библиотеки кода в паттерне `router.html`.
- `CallObjectMethod` - вызов метода объекта/библиотеки в паттерне `router.html`.
- `tools.object_to_text` - сериализация объекта в текст; в источнике использовалась для JSON.
- `GetOptProperty` - безопасное чтение свойства с fallback-значением.
- `xhttp_config.json` - конфигурационный файл xHTTP WebSoft HCM.
- `Authentication` - секция `xhttp_config.json` для правил аутентификации.
- `XAuthIdOptions` - секция/настройки для multi `x-auth-id`.
- `AllowedHosts` - секция ограничения доступных host names.
- `ResponseHeadersRules` - секция управления HTTP-заголовками xHTTP.
- `DefaultTicketCacheLifeTime` - время жизни локального кэша bearer-токена по PDF.
- `AspNetMappings` - обязательный с версии 2023.2 параметр обратной совместимости с `Websoft.Mapping.HCM` по PDF.
- `db set type postgresql` - команда x-shell для переключения учебного контура на PostgreSQL.
- `db init` - команда инициализации структуры базы.
- `db migrate from xml` - команда миграции из XML-хранилища.
- `rctg start` - команда перестройки каталогов и индексов.
- `internal server language` - в этом репозитории используется формулировка «внутренний серверный язык WebTutor» вместо автоматического приравнивания серверного кода к обычному JavaScript.
- `SP-XML` - платформа быстрой разработки приложений, базовая для рассматриваемых материалов docs.datex.ru.
- `SP-XML Script` - внутренний серверный язык платформы; синтаксически похож на ECMAScript-семейство, но имеет собственные типы, функции и поведение.
- `SpXml.exe` - исполняемый файл пользовательского интерфейса по docs.datex.ru.
- `xHttp.exe` - исполняемый файл серверной части по docs.datex.ru.
- `SpXml.ini` - конфигурационный файл рабочего места.
- `xHttp.ini` - конфигурационный файл сервера.
- `SpXml2.ini` - дополнительный конфигурационный файл для пользовательских параметров рабочего места.
- `xHttp1.ini` - дополнительный конфигурационный файл для пользовательских параметров сервера.
- `APPLICATION-MODULE` - основной параметр конфигурации, содержащий имя главного модуля приложения.
- `data` - основная серверная база в типовом SP-XML приложении.
- `data_local` - пользовательская локальная база.
- `trash` - база для корзины.
- `x-db-obj://` - схема URL для объектных документов в новой объектной модели.
- `XQuery` - язык/функция выборки данных, используется в примере агента.
- `ArrayOptFirstElem` - функция получения первого элемента массива или `undefined`, встречается в примере агента.
- `OptInt` - безопасное приведение к целому числу, встречается в примере агента.
- `OpenDoc` - открытие документа по URL документа, встречается в примере агента.
- `UrlFromDocID` - получение URL документа по ID.
- `TopElem` - верхний элемент открытого документа.
- `ArrayOptFindByKey` - поиск элемента массива по значению поля.
- `tools.create_notification` - создание уведомления.
- `Save()` - сохранение измененного документа.
- `alert(...)` - диагностический вывод в контексте агента WebTutor; не браузерный `window.alert`.
- `CallObjectMethod` - динамический вызов метода объекта по имени.
- `CallObjectMethodWithLock` - динамический вызов метода объекта по имени с блокировкой через `Lock`.
- `GetObjectProperty` - динамическое получение свойства объекта по имени.
- `GetClassObjectPropertyNames` - получение списка имен свойств объектов класса.
- `Lock` - объект блокировки для функций с `WithLock`; требует дополнительной проверки.
- `shared_docs` - секция модуля для регистрации общих документов в кэше; точное имя требует проверки по актуальным файлам.
- `FetchDoc` - открывает XML-документ и помещает его в кэш документов.
- `GetCachedDoc` - возвращает документ из кэша или завершает выполнение с ошибкой, если документа нет.
- `GetOptCachedDoc` - возвращает документ из кэша или `undefined`.
- `OpenNewDoc` - открытие нового XML-документа по материалам SP-XML.
- `RegisterSharedDoc` - динамическая регистрация общего документа.
- `ReplaceCachedDoc` - замена документа в кэше.
- `ArrayDirect` - приводит массив к массиву с прямым индексированием.
- `ArraySelectAll` - возвращает стандартный массив со всеми элементами исходного массива.
- `ArrayCount` - возвращает число элементов массива; для сложных массивов может требовать перебора.
- `ArrayOptSize` - возвращает размер массива, если он известен заранее.
- `StrLen` - длина строки в байтах.
- `StrCharCount` - количество символов в строке.
- `StrLeftRange` - фрагмент строки по длине в байтах.
- `StrLeftCharRange` - фрагмент строки по длине в символах.
- `StrOptScan` - необязательный разбор строки по шаблону, при несовпадении возвращает `undefined`.
- `Date` - функция создания/преобразования даты; безопасность для пользовательского ввода требует проверки.
- `OptDate` - необязательное преобразование даты.
- `ParseDate` - разбор даты.
- `StrXmlDate` - форматирование даты для XML.
- `HtmlEncode` - кодирование текста для HTML.
- `HtmlEncodeDoc` - преобразование текста в полный HTML-документ.
- `HtmlToPlainText` - преобразование HTML в простой текст.
- `EncodeJson` - преобразование значения в JSON-строку.
- `ParseJson` - разбор JSON; подробности требуют отдельной проверки.
- `HttpRequest` - функция HTTP-запроса из SP-XML; требует проверки безопасного применения.
- `SafeEval` - функция выполнения кода; требует проверки безопасного контекста.
- `EvalCodeUrl` - выполнение кода по URL; требует проверки безопасного контекста.
- `LoadUrlText` - чтение текста по URL.
- `PutUrlText` - запись текста по URL.
- `UrlExists` - проверка существования URL.
- `x-app://` - URL-схема для доступа к файлам модулей в новой объектной модели.
- `xxx_application.xml` - файл описания структуры приложения.
- `xxx_module.xml` - файл описания модуля.
- `xxx_module.js` - файл с методами инициализации модуля.
- `OnModuleLoadUiText` - метод загрузки языковых констант модуля.
- `OnModuleLoad` - метод этапа загрузки модуля.
- `OnModulePrepare` - метод этапа подготовки модуля.
- `OnModuleInit` - метод этапа инициализации модуля.
- `OnModuleStart` - метод этапа запуска модуля.
- `V1 library` - библиотека SP-XML старого типа, часто XML-документ с `INLINE-FORM`.
- `V2 library` - библиотека SP-XML нового типа с namespace, классами, типизированными параметрами и необязательными аргументами.
